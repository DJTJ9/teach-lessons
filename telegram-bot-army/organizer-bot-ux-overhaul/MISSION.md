# Mission

## Why
Dokumentiere die Architektur und Muster des Organizer Bot UX Overhauls so, dass du sie in zukünftigen Bot-Projekten direkt wiederverwenden kannst — ohne den Code nochmal zu lesen.

## Success looks like
- Du kannst erklären wann Reply-Keyboard vs Inline-Keyboard die bessere Wahl ist
- Du kannst eine In-Memory State Machine für Multi-Step-Dialoge von Grund auf skizzieren
- Du kennst die Fallstricke bei Source-Inspection-Tests (Substring-Problem, Index-Scope)

## Constraints
- Kontext: gerade implementiert — du kennst den Code bereits
- Ziel: Referenz für spätere Projekte, nicht "von Null lernen"
- Scope: Python + raw Telegram HTTP API patterns

## Out of scope
- Notion MCP Integration
- Reminder/Plan-Loop-Mechanismus
- Abend-Workflow Details
