Bot-Code auf den Pi übertragen

Git installieren, das Repository klonen — der Bot-Code landet auf dem Pi.

Warum diese Lektion?

Der Bot-Code liegt auf GitHub. Git ist der sauberste Weg, ihn auf den Pi zu bringen — und ermöglicht spätere Updates mit einem einzigen Befehl statt Dateien manuell zu kopieren.

Was ist Git?

Git ist ein Versionskontrollsystem. Es verfolgt Änderungen an Code und erlaubt die Synchronisierung zwischen Computern. Für uns: Bot-Code von GitHub auf den Pi holen.

Repository (Repo)

Ein Repository ist ein Ordner, dessen Änderungsgeschichte von Git verwaltet wird. Das Bot-Repo liegt auf GitHub — git clone kopiert es einmalig auf den Pi.

Schritt 1: Git installieren

1

Prüfen ob Git vorhanden ist

Auf dem Pi (via SSH):

git --version

Wenn eine Version erscheint: fertig. Sonst installieren:

sudo apt update && sudo apt install git -y

Schritt 2: Repository klonen

2

Arbeitsverzeichnis anlegen

mkdir -p ~/projekte
cd ~/projekte
3

Repo von GitHub klonen

git clone https://github.com/DJTJ9/claude-telegram-chat-bot.git

Ergebnis: neuer Ordner ~/projekte/claude-telegram-chat-bot/ mit dem kompletten Bot-Code.

ls claude-telegram-chat-bot/     # Prüfen ob Dateien da sind
4

In den Bot-Ordner wechseln

cd ~/projekte/claude-telegram-chat-bot
ls -la

Du solltest Dateien wie bot.py, requirements.txt, .gitignore sehen.

Schritt 3: Updates einspielen

Wenn du auf dem Windows-PC Code änderst und auf GitHub pushst, reicht auf dem Pi:

cd ~/projekte/claude-telegram-chat-bot
git pull

Git lädt nur die Änderungen herunter — kein erneutes Klonen nötig.

💡 Alias für schnelle Updates

Füge in ~/.bashrc einen Alias hinzu:

echo 'alias bot-update="cd ~/projekte/claude-telegram-chat-bot && git pull"' >> ~/.bashrc
source ~/.bashrc

Danach reicht: bot-update

Wichtig: .env-Datei gehört nicht ins Repo

⚠️ Secrets bleiben lokal

Die .env-Datei mit Telegram-Token und Notion-API-Key ist in .gitignore eingetragen und landet nie auf GitHub. Du erstellst sie manuell auf dem Pi — das kommt in der nächsten Lektion.

Warum .gitignore?

Die Datei .gitignore listet Dateien auf, die Git bewusst ignoriert. Secrets (API-Keys, Passwörter) dürfen nie in Git — sonst sind sie öffentlich auf GitHub sichtbar.

Verifikation

Prüfen ob alles da ist

ls -la ~/projekte/claude-telegram-chat-bot/
cat ~/projekte/claude-telegram-chat-bot/requirements.txt

Wenn du die Python-Dependencies siehst — Code ist erfolgreich übertragen.

🧠 Wissenscheck

1. Was ist der Unterschied zwischen git clone und git pull?
clone = erstes Herunterladen eines Repos (einmalig). pull = Änderungen vom Server holen, wenn das Repo schon lokal existiert.
2. Warum ist die .env-Datei nicht im GitHub-Repo?
Sie enthält API-Keys und Tokens. Diese Secrets auf GitHub würden jeden erlauben, deinen Bot zu übernehmen. .gitignore stellt sicher, dass sie nie committed werden.
3. Wie bekommst du später Code-Updates auf den Pi?
cd ~/projekte/claude-telegram-chat-bot && git pull — Git holt nur die Änderungen herunter.

✅ Das hast du in dieser Lektion gelernt

  • Git ist ein Versionskontrollsystem für Code-Synchronisierung
  • git clone URL kopiert ein Repository einmalig auf den Pi
  • git pull holt spätere Updates — kein erneutes Klonen nötig
  • Die .env-Datei mit Secrets gehört nie ins Repo — muss manuell erstellt werden
