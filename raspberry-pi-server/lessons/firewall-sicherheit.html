Lektion 7 von 8 · Raspberry Pi als Home-Server

Firewall und Sicherheit

ufw einrichten, SSH absichern, fail2ban als Bonus — Grundschutz für den Home-Server.

Warum diese Lektion?

Dein Pi kommuniziert nur ausgehend mit Telegram/Notion — er braucht keinen eingehenden Traffic außer SSH. Eine Firewall stellt sicher, dass nur dieser eine Port offen ist.

Was ist ufw?

ufw — Uncomplicated Firewall

ufw ist ein benutzerfreundliches Frontend für iptables (die eigentliche Linux-Firewall). Es macht Firewall-Regeln lesbar und einfach zu verwalten — ohne komplizierte Syntax.

Schritt 1: ufw konfigurieren

⚠️ SSH zuerst erlauben!

Erlaube SSH BEVOR du die Firewall aktivierst. Sonst sperrst du dich selbst aus.

1

ufw installieren

sudo apt update && sudo apt install ufw -y
2

SSH-Port erlauben (ZUERST!)

sudo ufw allow ssh

Das erlaubt Port 22. Ohne diese Regel: nach dem Aktivieren ausgesperrt.

3

Standard-Policy setzen

sudo ufw default deny incoming
sudo ufw default allow outgoing

Alles eingehend blockieren außer explizit erlaubten Ports. Ausgehend alles erlauben (Bot braucht das für Telegram/Notion).

4

Firewall aktivieren

sudo ufw enable

Bestätigung mit y. Firewall ist jetzt aktiv und startet beim Boot automatisch.

5

Status prüfen

sudo ufw status verbose

Du solltest sehen: Status: active und Port 22 als erlaubt.

Bonus: fail2ban

fail2ban

fail2ban überwacht Log-Dateien und blockiert IPs automatisch nach zu vielen fehlgeschlagenen Login-Versuchen. Schützt vor Brute-Force-Angriffen auf SSH.

6

fail2ban installieren

sudo apt install fail2ban -y
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

Standard-Konfiguration schützt SSH bereits: 5 fehlgeschlagene Versuche → IP für 10 Minuten blockiert.

sudo fail2ban-client status sshd

💡 Bot braucht keinen eingehenden Port

Der Telegram-Notion-Bot nutzt Polling (ausgehend). Kein eingehender Port nötig — nur SSH bleibt offen. Das ist die sicherste Konfiguration.

🧠 Wissenscheck

1. Warum muss man SSH erlauben BEVOR man ufw aktiviert?
Sobald ufw aktiv ist mit deny incoming, wird jede eingehende Verbindung blockiert — auch SSH. Ohne vorherige Erlaubnis von Port 22 sperrt man sich selbst aus.
2. Warum braucht der Bot keinen eingehenden Port?
Der Bot nutzt Telegram-Polling: Er fragt selbst bei Telegram ab ob neue Nachrichten da sind (ausgehende Verbindung). Telegram "pusht" nichts an den Bot — keine eingehenden Ports nötig.
3. Was macht fail2ban bei 5 fehlgeschlagenen SSH-Logins?
Es blockiert die angreifende IP-Adresse automatisch für 10 Minuten. Brute-Force-Angriffe werden so wirkungslos — zu viele gesperrte IPs, zu wenig Zeit.

✅ Das hast du in dieser Lektion gelernt

  • ufw macht Linux-Firewall einfach verwaltbar
  • Reihenfolge: SSH erlauben → Defaults setzen → Firewall aktivieren
  • deny incoming + allow outgoing: Bot funktioniert, niemand kommt rein außer SSH
  • fail2ban schützt automatisch vor Brute-Force-Angriffen
  • Letzte Lektion: Logs lesen und Bot warten
