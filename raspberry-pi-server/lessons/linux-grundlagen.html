Linux-Grundlagen

Dateisystem navigieren, Pakete installieren, Dateien bearbeiten — das Handwerkszeug für den Alltag auf dem Pi.

Warum diese Lektion?

Alles auf dem Pi passiert über das Terminal. Wer Befehle kennt, kann den Bot installieren, starten, debuggen und warten. Diese Lektion gibt dir die 20% der Befehle, die du 80% der Zeit brauchst.

Das Dateisystem verstehen

Linux hat eine andere Struktur als Windows. Statt C:\Users\... gibt es /home/pi/.... Alles startet von / (Root-Verzeichnis).

PfadWas ist das?
/Root — Wurzel des ganzen Systems
/home/pi/Dein Heimverzeichnis (wie C:\Users\pi)
/etc/Konfigurationsdateien
/var/log/Log-Dateien
/usr/bin/Installierte Programme
~Abkürzung für /home/pi (dein Heimverzeichnis)

Pakete installieren mit apt

apt ist der Paketmanager von Debian. Damit installierst du Software — wie ein App Store in der Kommandozeile.

BefehlBedeutung
sudo apt updatePaketliste aktualisieren (immer zuerst)
sudo apt upgradeInstallierte Pakete aktualisieren
sudo apt install paketPaket installieren
sudo apt remove paketPaket entfernen
💡 Was ist sudo?

sudo = "super user do" — Admin-Rechte für einen Befehl. Systemänderungen (Pakete, Dienste) brauchen sudo. Eigene Dateien bearbeiten nicht.

🖥️ Übung 3: System aktualisieren
sudo apt update && sudo apt upgrade -y

&& = "nur ausführen wenn der erste Befehl erfolgreich war". -y = alle Fragen automatisch bestätigen.

🧠 Wissenscheck

1. Was macht sudo apt update und warum ist es wichtig?
Es aktualisiert die Paketliste — lädt aber noch nichts herunter. Wichtig, weil apt sonst veraltete Versionen findet. Immer vor apt install ausführen.
2. Was ist der Unterschied zwischen ~ und /?
~ = dein Heimverzeichnis (/home/pi). / = Root des gesamten Systems. Als normaler Nutzer arbeitest du immer in ~.
3. Wie speichert man in nano und verlässt den Editor?
Strg+X → Y (Ja, speichern) → Enter (Dateiname bestätigen).
