Lektion 5 von 8 · Raspberry Pi als Home-Server

Python-Umgebung einrichten

venv erstellen, Dependencies installieren, .env-Datei für Secrets — dann den Bot testweise starten.

Warum diese Lektion?

Python-Projekte brauchen isolierte Umgebungen (venv), damit Pakete nicht mit anderen Projekten kollidieren. API-Keys müssen sicher konfiguriert werden — nie im Code, immer in einer .env-Datei.

Was ist ein venv?

Virtual Environment (venv)

Ein venv ist ein isolierter Python-Ordner mit eigenen installierten Paketen. Pakete im venv beeinflussen nicht das System-Python und andere Projekte. Standard-Praxis für alle Python-Projekte.

Schritt 1: Python prüfen und venv erstellen

1

Python-Version prüfen

Auf dem Pi (via SSH):

python3 --version

Raspberry Pi OS Lite (64-bit) kommt mit Python 3.11+. Falls venv fehlt:

sudo apt update && sudo apt install python3 python3-venv python3-pip -y
2

In den Bot-Ordner wechseln

cd ~/projekte/claude-telegram-chat-bot
3

Virtuelles Environment erstellen

python3 -m venv venv

Das erstellt einen Ordner venv/ mit einer isolierten Python-Installation.

4

venv aktivieren

source venv/bin/activate

Die Eingabeaufforderung ändert sich zu (venv) pi@raspberry-pi:... — das zeigt, dass das venv aktiv ist.

💡 venv immer aktivieren

Vor jedem manuellen Start muss source venv/bin/activate ausgeführt werden. Der systemd-Service (Lektion 6) macht das automatisch im Service-Setup.

Schritt 2: Abhängigkeiten installieren

5

Requirements installieren

Im aktiven venv:

pip install -r requirements.txt

pip installiert alle in requirements.txt gelisteten Pakete. Das dauert einige Minuten — der Pi kompiliert manche Pakete.

6

Installation prüfen

pip list

Du solltest python-telegram-bot, notion-client und andere Bot-Abhängigkeiten sehen.

Schritt 3: .env-Datei mit Secrets anlegen

⚠️ Secrets sicher konfigurieren

Die .env-Datei enthält API-Keys — niemals per git commiten. Sie muss manuell auf dem Pi erstellt werden.

7

.env-Datei erstellen

nano ~/projekte/claude-telegram-chat-bot/.env

Inhalt (ersetze die Platzhalter mit deinen echten Werten):

TELEGRAM_TOKEN=123456789:ABCdef...
NOTION_TOKEN=secret_abc123...
TELEGRAM_CHAT_ID=8896609541

Speichern: Strg+X → Y → Enter

8

Dateiberechtigungen sichern

chmod 600 ~/projekte/claude-telegram-chat-bot/.env

chmod 600 = nur du (pi) kannst lesen und schreiben — kein anderer Nutzer auf dem System.

Schritt 4: Bot testweise starten

9

Bot manuell starten

cd ~/projekte/claude-telegram-chat-bot
source venv/bin/activate
python3 bot.py

Der Bot startet und gibt Logs aus. Schicke eine Nachricht auf Telegram — der Bot sollte antworten.

Stoppen mit Strg+C

✅ Bot läuft!

Wenn der Bot auf Telegram-Nachrichten antwortet — alles korrekt konfiguriert. Jetzt kommt Lektion 6: den Bot als systemd-Service einrichten, damit er permanent läuft.

Warum nicht einfach python3 bot.py dauerhaft laufen lassen?

Ein manuell gestarteter Prozess stoppt, wenn du die SSH-Verbindung trennst. systemd (Lektion 6) startet den Bot automatisch beim Booten und beim Absturz — das ist echtes 24/7-Hosting.

🧠 Wissenscheck

1. Warum braucht man ein venv statt das System-Python zu nutzen?
Isolation: Pakete im venv beeinflussen nicht das System-Python und andere Projekte. Verschiedene Versionen desselben Pakets für verschiedene Projekte sind so kein Problem.
2. Was bedeutet chmod 600 .env?
600 = Lesen + Schreiben nur für den Eigentümer, kein Zugriff für andere Nutzer. Wichtig für Secrets — kein anderer Systemnutzer soll die API-Keys lesen können.
3. Warum stoppt der Bot wenn man die SSH-Verbindung schließt?
Ein im Terminal gestarteter Prozess ist an die Terminal-Session gebunden. Endet die Session, wird der Prozess beendet. systemd (nächste Lektion) löst das dauerhaft.

✅ Das hast du in dieser Lektion gelernt

  • venv isoliert Python-Pakete vom System — Standard-Praxis für Python-Projekte
  • python3 -m venv venv erstellt, source venv/bin/activate aktiviert
  • pip install -r requirements.txt installiert alle Bot-Abhängigkeiten
  • .env-Datei speichert Secrets, chmod 600 schützt sie
  • Bot läuft — aber nur wenn Terminal offen ist. Nächster Schritt: systemd-Service
