Lektion 2 von 8 · Raspberry Pi als Home-Server

SSH-Zugang einrichten

Sicher vom Windows-PC auf den Pi verbinden — mit SSH-Keys statt Passwort.

Warum diese Lektion?

SSH ist deine einzige Verbindung zum Pi — kein Bildschirm, keine Tastatur, nur ein Terminal-Fenster auf dem PC. SSH-Keys machen diese Verbindung sicher und komfortabel (kein Passwort-Tippen mehr).

Was ist SSH?

SSH (Secure Shell) ist ein Protokoll für verschlüsselte Verbindungen zu entfernten Computern. Du tippst Befehle auf deinem PC, sie werden verschlüsselt zum Pi gesendet, dort ausgeführt, und das Ergebnis kommt verschlüsselt zurück.

Schlüsselpaar-Konzept

SSH-Keys funktionieren wie Schloss (öffentlicher Schlüssel) und Schlüssel (privater Schlüssel). Der öffentliche Schlüssel liegt auf dem Pi. Nur wer den passenden privaten Schlüssel auf dem PC hat, kommt rein — ohne Passwort.

Schritt 1: Erstes Login mit Passwort

1

PowerShell öffnen und verbinden

SSH ist ab Windows 10 eingebaut. In PowerShell:

ssh pi@raspberry-pi.local

Beim ersten Verbinden fragt SSH nach Fingerprint-Bestätigung:

The authenticity of host 'raspberry-pi.local' can't be established.
Are you sure you want to continue connecting? yes

Tippe yes und Enter. Dann das Passwort aus Lektion 1 eingeben.

💡 Du bist drin wenn du siehst:

pi@raspberry-pi:~ $ — das ist die Eingabeaufforderung des Pis.

Schritt 2: SSH-Key generieren (auf dem PC)

2

SSH-Verbindung trennen

Erst zurück zum PC:

exit
3

Schlüsselpaar erstellen (auf dem PC, nicht auf dem Pi)

ssh-keygen -t ed25519 -C "raspberry-pi-server"

Bei den Fragen einfach Enter drücken (Standard-Pfad, keine Passphrase):

Enter file: [Enter]
Enter passphrase: [Enter]

Ergebnis in C:\Users\DeinName\.ssh\:

id_ed25519       ← privater Schlüssel (NIEMALS teilen!)
id_ed25519.pub   ← öffentlicher Schlüssel (kommt auf den Pi)

Schritt 3: Öffentlichen Schlüssel auf den Pi übertragen

4

Schlüssel auf Pi kopieren

In PowerShell auf dem PC:

type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh pi@raspberry-pi.local "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"

Einmal das Passwort eingeben. Danach testen:

ssh pi@raspberry-pi.local

Kein Passwort mehr — direkter Login mit SSH-Key. ✅

Schritt 4: Passwort-Login deaktivieren

5

SSH-Konfiguration auf dem Pi bearbeiten

Auf dem Pi (nach SSH-Login):

sudo nano /etc/ssh/sshd_config

Diese Zeilen suchen und setzen:

PasswordAuthentication no
PubkeyAuthentication yes
PermitRootLogin no

Speichern: Strg+X, dann Y, dann Enter.

6

SSH-Dienst neu starten

sudo systemctl restart ssh
⚠️ Wichtig: Nicht ausloggen!

Öffne ein zweites PowerShell-Fenster und teste den Login bevor du die aktuelle Verbindung trennst. So bist du nicht ausgesperrt wenn etwas schiefgeht.

Bonus: SSH-Shortcut einrichten

~/.ssh/config auf dem PC erstellen

Erstelle die Datei C:\Users\DeinName\.ssh\config mit diesem Inhalt:

Host pi
    HostName raspberry-pi.local
    User pi
    IdentityFile ~/.ssh/id_ed25519

Danach reicht nur noch: ssh pi

🧠 Wissenscheck

1. Wozu braucht man SSH-Keys, wenn ein Passwort doch auch funktioniert?
Keys sind sicherer: kein Brute-Force-Angriff möglich, kein Passwort das gestohlen werden kann. Außerdem komfortabler — kein Tippen beim Login.
2. Was ist der Unterschied zwischen privatem und öffentlichem Schlüssel?
Der öffentliche Schlüssel (.pub) liegt auf dem Server — darf geteilt werden. Der private Schlüssel bleibt nur auf dem PC. Wer den privaten Schlüssel hat, kommt rein. Niemals den privaten Schlüssel teilen.
3. Warum deaktivieren wir den Passwort-Login?
Ohne Passwort-Login sind automatisierte Angriffe (Bots, die ständig Passwörter probieren) wirkungslos. Selbst wenn jemand deine IP und Port 22 kennt, kommt er ohne deinen privaten Schlüssel nicht rein.

✅ Das hast du in dieser Lektion gelernt

  • SSH verbindet sicher und verschlüsselt deinen PC mit dem Pi
  • SSH-Keys: privater Schlüssel (PC) + öffentlicher Schlüssel (Pi)
  • ssh-keygen -t ed25519 erzeugt ein modernes Schlüsselpaar
  • Passwort-Login deaktivieren schützt vor automatisierten Angriffen
