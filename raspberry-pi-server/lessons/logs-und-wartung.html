Logs und Wartung

journalctl, Bot-Logs lesen, System aktualisieren, Fehlersuche wenn der Bot abstürzt.

Warum diese Lektion?

Ein laufender Server braucht gelegentlich Aufmerksamkeit. Diese Lektion gibt dir das Handwerkszeug, um zu verstehen was der Bot macht, Fehler zu diagnostizieren und das System gesund zu halten.

Logs lesen mit journalctl

journalctl

journalctl ist der Log-Reader für systemd-Services. Da der Bot seine Ausgabe an das systemd-Journal sendet (StandardOutput=journal), sind alle Bot-Logs hier zu finden.

Die wichtigsten journalctl-Befehle

# Alle Logs des Bot-Service
journalctl -u telegram-notion-bot

# Letzte 50 Zeilen
journalctl -u telegram-notion-bot -n 50

# Live-Logs (wie tail -f)
journalctl -u telegram-notion-bot -f

# Logs seit dem letzten Boot
journalctl -u telegram-notion-bot -b

# Letzte Stunde
journalctl -u telegram-notion-bot --since "1 hour ago"

# Nur Fehler
journalctl -u telegram-notion-bot -p err
💡 Live-Logs für Debugging

journalctl -u telegram-notion-bot -f zeigt Logs in Echtzeit. Schick eine Telegram-Nachricht und sieh sofort was der Bot macht. Beenden mit Strg+C.

System aktuell halten

Regelmäßiges Update (empfohlen: monatlich)

sudo apt update && sudo apt upgrade -y

Installiert Sicherheitsupdates. Der Bot läuft währenddessen weiter.

Bot-Code aktualisieren

cd ~/projekte/telegram-notion-bot
git pull
source venv/bin/activate
pip install -r requirements.txt   # nur wenn requirements.txt geändert
sudo systemctl restart telegram-notion-bot
sudo systemctl status telegram-notion-bot

Fehlersuche: Bot startet nicht

Diagnose-Checkliste

1
Status prüfen: sudo systemctl status telegram-notion-bot
2
Logs lesen: journalctl -u telegram-notion-bot -n 30 — Fehlermeldung suchen
3
.env prüfen: cat ~/projekte/telegram-notion-bot/.env — Tokens korrekt?
4
Manuell testen: cd ~/projekte/telegram-notion-bot && source venv/bin/activate && python3 bot.py
5
Internet: ping -c 3 api.telegram.org — Ist Telegram erreichbar?

Logs aufräumen

Journalctl-Logs auf 100 MB begrenzen

sudo journalctl --vacuum-size=100M

Dauerhaft konfigurieren:

sudo nano /etc/systemd/journald.conf
# Zeile setzen: SystemMaxUse=100M
sudo systemctl restart systemd-journald

🧠 Wissenscheck

1. Wie siehst du Bot-Logs in Echtzeit?
journalctl -u telegram-notion-bot -f — das -f steht für "follow". Beenden mit Strg+C.
2. Der Bot startet nicht. Was ist dein erster Diagnose-Schritt?
sudo systemctl status telegram-notion-bot zeigt ob der Service läuft und die letzten Log-Zeilen. Oft steht der Fehler direkt dort. Dann journalctl -n 50 für mehr Details.
3. Warum sollte man Logs auf einer SD-Karte begrenzen?
SD-Karten haben begrenzte Kapazität und leiden unter häufigen Schreibvorgängen. Unbegrenzte Logs können die Karte füllen oder beschleunigt abnutzen. 100 MB ist für Bot-Logs mehr als genug.
