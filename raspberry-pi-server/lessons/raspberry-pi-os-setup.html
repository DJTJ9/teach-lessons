Lektion 1 von 8 · Raspberry Pi als Home-Server

Raspberry Pi OS einrichten

OS flashen, SSH aktivieren, Pi zum ersten Mal hochfahren — in 30 Minuten einsatzbereit.

Warum diese Lektion?

Ohne laufendes Betriebssystem kann der Raspberry Pi gar nichts tun. Das OS ist die Grundlage für alles Weitere — SSH-Zugang, Bot-Code, Service-Start. Diese Lektion legt das Fundament.

Was ist Raspberry Pi OS?

Raspberry Pi OS (früher Raspbian) ist ein Linux-Betriebssystem, das speziell für Raspberry-Pi-Hardware optimiert wurde. Es basiert auf Debian — einer der am weitesten verbreiteten Linux-Distributionen. Das OS läuft von einer MicroSD-Karte.

Für einen Server ohne Bildschirm nutzen wir die Lite-Version (ohne Desktop-Oberfläche) — kleiner, schneller, stabiler.

Was du brauchst

Hardware-Liste

  • Raspberry Pi 3 Model B oder B+ (1 GB RAM)
  • MicroSD-Karte, mind. 16 GB (Klasse 10 / A1/A2)
  • Micro-USB Netzteil, mind. 2,5A (offiziell: Raspberry Pi 12,5W Micro-USB)
  • Netzwerkkabel (Ethernet) — stabiler als WLAN für einen Server
  • SD-Kartenleser (für den Windows-PC)

Schritt-für-Schritt: OS flashen

1

Raspberry Pi Imager herunterladen

Lade den offiziellen Imager von raspberrypi.com/software herunter und installiere ihn auf deinem Windows-PC. Das Tool flasht das OS auf die SD-Karte und konfiguriert es gleichzeitig.

2

OS auswählen: Raspberry Pi OS Lite (64-bit)

Öffne den Imager. Klicke auf "Raspberry Pi OS auswählen". Wähle:

Raspberry Pi OS (other)
  → Raspberry Pi OS Lite (64-bit)

Kein Desktop, kein grafisches Interface — für einen Server nicht nötig und nur Ballast.

3

SD-Karte auswählen

Klicke auf "Speicher auswählen" und wähle deine MicroSD-Karte. Achtung: Alle Daten auf der Karte werden gelöscht.

4

Einstellungen konfigurieren (Zahnrad-Icon)

Klicke auf das Zahnrad-Symbol (Erweiterte Optionen). Hier passiert das Wichtigste:

✅ Hostname setzen: raspberry-pi
✅ SSH aktivieren → "Passwort-Authentifizierung verwenden"
✅ Benutzername: pi
✅ Passwort: [sicheres Passwort wählen — notiere es!]
✅ Zeitzone: Europe/Berlin
✅ Tastaturlayout: de
💡 Tipp

SSH hier zu aktivieren erspart dir, Bildschirm und Tastatur an den Pi anzuschließen. Du kannst direkt vom Windows-PC verbinden.

5

Flashen starten

Klicke auf "Schreiben". Der Imager lädt das OS herunter, schreibt es auf die Karte und verifiziert es. Das dauert 5–15 Minuten.

6

Pi zum ersten Mal starten

SD-Karte in den Pi einsetzen (Slot unterhalb des Pi, goldene Pins nach oben). Ethernet-Kabel einstecken. Dann Netzteil anschließen. Der Pi bootet automatisch — kein Ein-Schalter.

Warte 60–90 Sekunden bis der Pi vollständig hochgefahren ist.

7

IP-Adresse herausfinden

Öffne auf dem Windows-PC eine PowerShell und führe aus:

ping raspberry-pi.local

Oder schau in deinem Router unter "Verbundene Geräte" nach dem Hostnamen raspberry-pi. Die IP sieht typischerweise so aus: 192.168.1.XXX

⚠️ Wichtig

Notiere die IP-Adresse deines Pis. Du brauchst sie in der nächsten Lektion für den SSH-Zugang. Tipp: Vergib dem Pi im Router eine feste IP (DHCP-Reservierung), damit sie sich nie ändert.

🧠 Wissenscheck

1. Warum verwenden wir Raspberry Pi OS Lite statt der Vollversion?
Kein Desktop nötig — ein Server wird ausschließlich über die Kommandozeile (SSH) bedient. Lite ist kleiner, schneller und hat weniger Angriffsfläche.
2. Was macht das SSH-Aktivieren im Imager?
Es erstellt eine leere Datei namens ssh auf der Boot-Partition. Der Pi erkennt sie beim ersten Start und aktiviert den SSH-Dienst. Ohne diese Datei wäre kein Fernzugang möglich.
3. Warum ist Ethernet besser als WLAN für einen Server?
Kabelverbindungen sind stabiler (keine Verbindungsabbrüche), schneller und funktionieren auch wenn das WLAN-Passwort geändert wird. Für einen 24/7-Bot ist Zuverlässigkeit wichtiger als Komfort.

✅ Das hast du in dieser Lektion gelernt

  • Raspberry Pi OS Lite (64-bit) ist die richtige Wahl für Server-Betrieb
  • Raspberry Pi Imager konfiguriert OS, SSH und Benutzer in einem Schritt
  • SSH muss vor dem ersten Boot aktiviert werden — danach ist kein Bildschirm am Pi nötig
  • Der Pi ist über ping raspberry-pi.local erreichbar
  • Nächster Schritt: SSH-Verbindung vom Windows-PC aufbauen
