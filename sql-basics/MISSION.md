# Mission

## Why
SQL und PostgreSQL so gut beherrschen, dass die Dart-App-Daten in eine echte Datenbank migriert werden koennen und zukuenftige Spiele- und App-Projekte sauber datenbankgestuetzt entwickelt werden koennen.

## Success looks like
- Kann Tabellen designen, anlegen und verwalten (DDL)
- Kann Daten einfuegen, abfragen, filtern, sortieren (DML)
- Kann JOINs, GROUP BY, Aggregatfunktionen korrekt einsetzen
- Hat Dart-App-Daten erfolgreich in PostgreSQL migriert
- Versteht wann welche Datenbankstruktur fuer Spiele/Apps sinnvoll ist

## Constraints
- Vorkenntnisse: 0 (noch nie SQL gearbeitet)
- DB-Engine: PostgreSQL (in Deutschland weit verbreitet, ideal fuer Analyse + App/Game-Dev)
- Konkreter Anwendungsfall: Dart-Daten-Migration zu PostgreSQL
- Zielniveau: Junior Developer

## Out of scope
- NoSQL (MongoDB, Redis)
- Stored Procedures / Trigger (fortgeschrittene DB-Programmierung)
- DBA-Themen (Replikation, Sharding, Tuning auf Systemebene)
- MySQL/MariaDB spezifische Syntax
