JOINs

Tabellen zusammenfuehren — INNER JOIN, LEFT JOIN, RIGHT JOIN

Warum JOINs?

Daten in relationalen Datenbanken sind auf mehrere Tabellen aufgeteilt. JOINs bringen sie fuer eine Abfrage zusammen — ohne Daten zu kopieren.

Carol (id=3) hat keine Scores. Das ist wichtig fuer den Unterschied zwischen INNER und LEFT JOIN.

INNER JOIN: nur Treffer auf beiden Seiten

SELECT s.username, sc.punkte FROM spieler s INNER JOIN scores sc ON s.id = sc.spieler_id; username | punkte ----------+-------- Alice | 4200 Alice | 3800 Bob | 1850 -- Carol taucht NICHT auf -- kein Score vorhanden

INNER JOIN gibt nur Zeilen zurueck die in BEIDEN Tabellen einen Treffer haben. Carol fehlt, weil sie keinen Score hat.

LEFT JOIN: alle aus der linken Tabelle

SELECT s.username, sc.punkte FROM spieler s LEFT JOIN scores sc ON s.id = sc.spieler_id; username | punkte ----------+-------- Alice | 4200 Alice | 3800 Bob | 1850 Carol | NULL -- Carol ist dabei, punkte ist NULL

LEFT JOIN gibt alle Zeilen aus der linken Tabelle (spieler) zurueck — auch wenn kein Treffer in der rechten Tabelle (scores) existiert. Fehlende Werte werden als NULL gezeigt.

Alias: Tabellen abkuerzen

-- s = spieler, sc = scores (Alias) SELECT s.username, s.ist_aktiv, sc.punkte, sc.erreicht_am FROM spieler s LEFT JOIN scores sc ON s.id = sc.spieler_id WHERE sc.punkte > 3000 ORDER BY sc.punkte DESC;

Mehrere JOINs verketten

-- Spieler + Scores + Items (3 Tabellen) SELECT s.username, sc.punkte, i.name AS item_name FROM spieler s INNER JOIN scores sc ON s.id = sc.spieler_id LEFT JOIN spieler_items si ON s.id = si.spieler_id LEFT JOIN items i ON si.item_id = i.id;

Praxisbeispiel: Spieler-Highscore-Liste

-- Bester Score je Spieler mit Spieler-Name SELECT s.username, MAX(sc.punkte) AS highscore, COUNT(sc.id) AS anzahl_versuche FROM spieler s LEFT JOIN scores sc ON s.id = sc.spieler_id GROUP BY s.id, s.username ORDER BY highscore DESC NULLS LAST; username | highscore | anzahl_versuche ----------+-----------+---------------- Carol | NULL | 0 Alice | 4200 | 2 Bob | 1850 | 1
NULLS LAST schiebt NULL-Werte ans Ende der Sortierung. Ohne NULLS LAST wuerden NULL-Werte bei DESC ganz oben erscheinen.

RIGHT JOIN und FULL OUTER JOIN

-- RIGHT JOIN: alle aus der rechten Tabelle (selten genutzt) -- Kann immer als LEFT JOIN umgeschrieben werden -- FULL OUTER JOIN: alle aus beiden Tabellen SELECT s.username, sc.punkte FROM spieler s FULL OUTER JOIN scores sc ON s.id = sc.spieler_id; -- Spieler ohne Score UND Scores ohne Spieler (Datenintegritaetsproblem)

Quiz

1. Du willst ALLE Spieler anzeigen, auch die ohne Scores. Welcher JOIN?

B — LEFT JOIN mit spieler als linker Tabelle gibt ALLE Spieler zurueck. Spieler ohne Score bekommen NULL in den Score-Feldern. INNER JOIN wuerde Spieler ohne Score weglassen.

2. Wozu dienen Tabellen-Aliases (s, sc) in JOINs?

B — Ohne Alias: spieler.id vs scores.spieler_id — lesbar aber laenger. Mit Alias s.id vs sc.spieler_id — kuerzer. Ohne Alias sind Spalten mit gleichem Namen (z.B. beide haben "name") nicht unterscheidbar.

3. Was ist der Unterschied zwischen INNER JOIN und LEFT JOIN wenn alle Zeilen Treffer haben?

B — Wenn jede Zeile links einen Treffer rechts hat, ist das Ergebnis identisch. Der Unterschied zeigt sich nur wenn KEINE Treffer vorhanden sind: LEFT JOIN behaelt die linke Zeile mit NULL, INNER JOIN wirft sie weg.
