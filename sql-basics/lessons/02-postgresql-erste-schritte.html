Installation (Windows mit WSL)

Da du Windows nutzt, empfehle ich PostgreSQL via WSL (Windows Subsystem for Linux). Das ist die sauberste Entwicklungsumgebung.

1
WSL und Ubuntu installieren (falls nicht vorhanden)
# In PowerShell als Administrator: wsl --install # System neu starten, dann Ubuntu-Setup abschliessen
2
PostgreSQL in WSL installieren
$ sudo apt update && sudo apt install postgresql postgresql-contrib -y $ sudo service postgresql start * Starting PostgreSQL 14 database server
3
Als postgres-User einloggen
$ sudo -u postgres psql psql (14.x) Type "help" for help. postgres=# # Du bist jetzt in der PostgreSQL-Shell (psql)
Alternative: pgAdmin (GUI) installieren falls du lieber mit einer grafischen Oberflaeche arbeitest. Aber psql lernen lohnt sich — du kannst damit von ueberall arbeiten.

psql: die wichtigsten Befehle

psql hat zwei Arten von Befehlen: Backslash-Befehle (Meta-Befehle) und SQL-Befehle. Meta-Befehle enden nicht mit Semikolon, SQL schon.

BefehlWas er tut
\lAlle Datenbanken auflisten
\c datenbanknameZu einer Datenbank wechseln
\dtAlle Tabellen in aktueller Datenbank
\d tabellennameStruktur einer Tabelle anzeigen
\qpsql beenden
\h SELECTHilfe zu SQL-Befehl anzeigen

Erste Datenbank erstellen

-- In psql: SQL-Befehle enden immer mit Semikolon CREATE DATABASE spieledb; CREATE DATABASE -- Zur neuen Datenbank wechseln \c spieledb You are now connected to database "spieledb" as user "postgres". -- Tabellen anzeigen (noch leer) \dt Did not find any relations.

Erste Tabelle und erste Daten

-- Tabelle erstellen (mehr Details in Lektion 03) CREATE TABLE spieler ( id SERIAL PRIMARY KEY, name TEXT NOT NULL, punkte INTEGER DEFAULT 0 ); CREATE TABLE -- Daten einfuegen INSERT INTO spieler (name, punkte) VALUES ('Alice', 4200); INSERT INTO spieler (name, punkte) VALUES ('Bob', 1850); INSERT 0 1 -- Daten abfragen SELECT * FROM spieler; id | name | punkte ----+-------+-------- 1 | Alice | 4200 2 | Bob | 1850 (2 rows)
SERIAL ist eine PostgreSQL-Kurzschreibweise fuer "auto-inkrementierende Ganzzahl". Die Datenbank vergibt die ID automatisch — du musst sie nicht mitangeben beim INSERT.

Datenbank-Hierarchie verstehen

PostgreSQL-Hierarchie: Server (PostgreSQL-Instanz) └─ Datenbank: spieledb └─ Schema: public (Standard) ├─ Tabelle: spieler ├─ Tabelle: scores └─ Tabelle: items

Eine PostgreSQL-Instanz kann viele Datenbanken haben. Deine Dart-App, dein Spiel und andere Projekte bekommen jeweils eigene Datenbanken. Tabellen existieren innerhalb einer Datenbank.

Quiz

1. Was bewirkt \c spieledb in psql?

B — \c steht fuer "connect". Danach werden alle SQL-Befehle in dieser Datenbank ausgefuehrt. Ohne \c arbeitet man in der zuletzt verbundenen Datenbank.

2. Warum muss beim INSERT die id nicht angegeben werden?

B — SERIAL ist eine Sequenz die automatisch 1, 2, 3... vergibt. Erster INSERT bekommt id=1, zweiter id=2 usw. NULL waere ein Fehler da PRIMARY KEY NOT NULL impliziert.

3. Welcher Befehl zeigt alle Tabellen in der aktuellen Datenbank?

\dt — "l" listet Datenbanken, "dt" listet Tabellen (d = describe, t = tables). \d ohne t zeigt Tabellen UND andere Objekte wie Sequenzen.
