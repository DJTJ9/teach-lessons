SQL BASICS · LEKTION 04

DML: INSERT und SELECT

Daten einfuegen, abfragen, filtern und sortieren — die meistgenutzten SQL-Befehle

DML — Data Manipulation Language

DML bearbeitet Daten (nicht Strukturen): INSERT (einfuegen), SELECT (lesen), UPDATE (aendern), DELETE (loeschen). Diese Lektion fokussiert auf INSERT und SELECT — die taeglich genutzten 80%.

INSERT: Daten einfuegen

-- Einzelner Datensatz INSERT INTO spieler (username, punkte) VALUES ('Alice', 4200); -- Mehrere Datensaetze auf einmal INSERT INTO spieler (username, punkte) VALUES ('Alice', 4200), ('Bob', 1850), ('Carol', 7340); -- ID des eingefuegten Datensatzes zurueckbekommen INSERT INTO spieler (username) VALUES ('Dave') RETURNING id; id ---- 4
RETURNING id ist PostgreSQL-spezifisch und sehr nuetzlich: Du bekommst die generierte ID direkt zurueck. In Python/Dart kannst du sie direkt fuer Foreign Keys verwenden.

SELECT: Daten abfragen

-- Alle Spalten, alle Zeilen SELECT * FROM spieler; id | username | punkte | erstellt_am | ist_aktiv ----+----------+--------+----------------------+----------- 1 | Alice | 4200 | 2024-01-15 10:30:00 | t 2 | Bob | 1850 | 2024-02-03 14:22:00 | t 3 | Carol | 7340 | 2024-01-28 09:15:00 | t -- Nur bestimmte Spalten SELECT username, punkte FROM spieler; username | punkte ----------+-------- Alice | 4200 Bob | 1850 Carol | 7340 -- Spalte umbenennen in der Ausgabe (ALIAS) SELECT username AS name, punkte AS score FROM spieler;

WHERE: Filtern

-- Vergleichsoperatoren: = != < > <= >= SELECT * FROM spieler WHERE punkte > 3000; id | username | punkte ----+----------+-------- 1 | Alice | 4200 3 | Carol | 7340 -- Text-Vergleich: immer einfache Anfuehrungszeichen SELECT * FROM spieler WHERE username = 'Alice'; -- LIKE: Muster-Suche (% = beliebig viele Zeichen) SELECT * FROM spieler WHERE username LIKE 'A%'; -- Findet: Alice, Anna, Andreas... (beginnt mit A) -- ILIKE: wie LIKE aber case-insensitive SELECT * FROM spieler WHERE username ILIKE 'alice'; -- Mehrere Bedingungen SELECT * FROM spieler WHERE punkte > 2000 AND ist_aktiv = TRUE; -- NULL-Pruefung: IS NULL / IS NOT NULL SELECT * FROM spieler WHERE email IS NOT NULL;

ORDER BY und LIMIT: Sortieren und begrenzen

-- Absteigend sortieren (hoechste zuerst) SELECT username, punkte FROM spieler ORDER BY punkte DESC; username | punkte ----------+-------- Carol | 7340 Alice | 4200 Bob | 1850 -- Aufsteigend (Standard, ASC kann weggelassen werden) SELECT * FROM spieler ORDER BY username ASC; -- Top 3 Spieler SELECT username, punkte FROM spieler ORDER BY punkte DESC LIMIT 3; -- Paginierung: Seite 2, je 10 Ergebnisse SELECT * FROM spieler ORDER BY id LIMIT 10 OFFSET 10;

UPDATE und DELETE

-- Punkte von Alice aktualisieren UPDATE spieler SET punkte = 5000 WHERE username = 'Alice'; -- Vorsicht: UPDATE ohne WHERE aktualisiert ALLE Zeilen! UPDATE spieler SET punkte = 0; -- Alle Punkte auf 0 gesetzt! -- Inaktive Spieler loeschen DELETE FROM spieler WHERE ist_aktiv = FALSE; -- Vorsicht: DELETE ohne WHERE loescht ALLE Zeilen!
UPDATE und DELETE ohne WHERE sind die haeufigsten Katastrophen in Datenbanken. Gewoehn dir an: erst SELECT * FROM tabelle WHERE bedingung ausfuehren, pruefen ob die richtigen Zeilen gezeigt werden, dann UPDATE/DELETE mit derselben WHERE-Klausel.

Quiz

1. Was gibt SELECT username FROM spieler WHERE punkte > 5000 ORDER BY punkte DESC LIMIT 1 zurueck?

B — WHERE filtert auf > 5000, ORDER BY DESC sortiert hoechste zuerst, LIMIT 1 gibt nur den ersten Eintrag zurueck. Ergebnis: "Carol" mit 7340 Punkten (nur die username-Spalte).

2. Du willst alle Spieler finden deren Name mit "B" beginnt. Welches WHERE?

LIKE 'B%' — % ist Platzhalter fuer beliebig viele Zeichen. 'B%' bedeutet "beginnt mit B". 'B' allein wuerde nur exakte Treffer finden. CONTAINS existiert nicht in SQL.

3. Du fuerst aus: UPDATE spieler SET ist_aktiv = FALSE. Was passiert?

B — SQL verlangt kein WHERE. Ohne WHERE wird JEDE Zeile geaendert. Das ist kein Fehler — aber wahrscheinlich ein Katastrophe. Immer erst mit SELECT pruefen!
