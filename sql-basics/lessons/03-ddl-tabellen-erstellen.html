DDL: Tabellen erstellen

CREATE TABLE, Datentypen, Constraints, ALTER TABLE — die Sprache der Datenbankstruktur

DDL — Data Definition Language

DDL ist der Teil von SQL der Strukturen definiert: Tabellen erstellen, aendern, loeschen. Gegensatz: DML (Data Manipulation Language) — Daten einfuegen, lesen, aendern.

DDL-Befehle: CREATE, ALTER, DROP, TRUNCATE

CREATE TABLE — Syntax

CREATE TABLE tabellenname ( spalte1 DATENTYP CONSTRAINT, spalte2 DATENTYP CONSTRAINT, ... );

Die wichtigsten Datentypen

INTEGER
Ganzzahl (-2 Mrd bis +2 Mrd). Fuer IDs, Punkte, Zaehler.
BIGINT
Grosse Ganzzahl. Fuer sehr grosse Werte (Timestamps als Unix-Zeit, Telegram-Chat-IDs).
SERIAL
Auto-inkrementierender INTEGER. Perfekt fuer Primary Keys — Datenbank vergibt automatisch.
TEXT
Zeichenkette beliebiger Laenge. Standard fuer Namen, Beschreibungen, JSON-Strings.
VARCHAR(n)
Text mit maximaler Laenge n. Nur nutzen wenn Laenge semantisch wichtig (z.B. Laendercodes).
BOOLEAN
true / false. Fuer Flags wie "ist_aktiv", "hat_gewonnen".
DATE
Datum ohne Uhrzeit (YYYY-MM-DD). Fuer Geburtstage, Deadlines.
TIMESTAMP
Datum mit Uhrzeit. Fuer Ereignisse, Logs. Empfehlung: TIMESTAMPTZ (mit Zeitzone).
NUMERIC(p,s)
Praezise Dezimalzahl. Fuer Geldbetraege (niemals FLOAT fuer Geld!).
JSONB
JSON-Daten binaer gespeichert (abfragbar). PostgreSQL-Spezialtaet, sehr maechtig.

Constraints: Regeln fuer Daten

PRIMARY KEY
Eindeutig + NOT NULL. Jede Tabelle braucht einen. Meist: id SERIAL PRIMARY KEY.
NOT NULL
Spalte darf kein NULL enthalten. Fuer Pflichtfelder wie name, email.
UNIQUE
Kein doppelter Wert. Fuer email-Adressen, Nutzernamen.
DEFAULT wert
Standardwert wenn kein Wert angegeben. z.B. DEFAULT 0 fuer punkte, DEFAULT NOW() fuer timestamps.
CHECK (bedingung)
Bedingung muss erfuellt sein. z.B. CHECK (punkte >= 0) verhindert negative Punkte.

Vollstaendiges Beispiel: Spiel-Datenbank

-- Spieler-Tabelle CREATE TABLE spieler ( id SERIAL PRIMARY KEY, username TEXT NOT NULL UNIQUE, email TEXT UNIQUE, erstellt_am TIMESTAMPTZ NOT NULL DEFAULT NOW(), ist_aktiv BOOLEAN NOT NULL DEFAULT TRUE ); -- Score-Tabelle CREATE TABLE scores ( id SERIAL PRIMARY KEY, spieler_id INTEGER NOT NULL, punkte INTEGER NOT NULL CHECK (punkte >= 0), erreicht_am TIMESTAMPTZ NOT NULL DEFAULT NOW(), level INTEGER DEFAULT 1 );

ALTER TABLE: Tabellen nachtraeglich aendern

-- Neue Spalte hinzufuegen ALTER TABLE spieler ADD COLUMN avatar_url TEXT; -- Spalte umbenennen ALTER TABLE spieler RENAME COLUMN username TO name; -- Spalte loeschen ALTER TABLE spieler DROP COLUMN avatar_url; -- Tabelle komplett loeschen DROP TABLE scores; -- Sicher loeschen (kein Fehler wenn nicht vorhanden) DROP TABLE IF EXISTS scores;
DROP TABLE loescht die Tabelle und alle Daten unwiderruflich. In Produktionsdatenbanken immer zuerst ein Backup machen.

Tabellenstruktur pruefen

Quiz

1. Welcher Datentyp ist richtig fuer eine Geldbetrag-Spalte?

NUMERIC(10,2) — FLOAT hat Rundungsfehler (0.1 + 0.2 = 0.30000000000000004). Fuer Geld immer NUMERIC. INTEGER verliert Dezimalstellen.

2. Was macht DEFAULT NOW() bei einer TIMESTAMPTZ-Spalte?

B — NOW() wird bei jedem INSERT neu ausgewertet. Zeile eingefuegt um 14:30 bekommt automatisch 14:30 als Zeitstempel. Kein manuelles Mitgeben noetig.

3. Unterschied zwischen NOT NULL und UNIQUE?

B — NOT NULL: Wert muss vorhanden sein. UNIQUE: Wert darf nicht mehrfach vorkommen. Emailadressen sollten UNIQUE und NOT NULL sein (jede Email einmalig und muss angegeben werden).
