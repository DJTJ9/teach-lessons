SQL BASICS · LEKTION 05

Filtern und Aggregieren

AND/OR/IN/BETWEEN, Aggregatfunktionen, GROUP BY und HAVING

Erweiterte WHERE-Bedingungen

-- AND / OR / NOT SELECT * FROM spieler WHERE punkte > 2000 AND ist_aktiv = TRUE; SELECT * FROM spieler WHERE punkte < 1000 OR punkte > 7000; -- IN: Einer aus einer Liste SELECT * FROM spieler WHERE username IN ('Alice', 'Bob', 'Carol'); -- NOT IN: Keiner aus der Liste SELECT * FROM spieler WHERE username NOT IN ('Alice', 'Bob'); -- BETWEEN: inklusiver Bereich SELECT * FROM spieler WHERE punkte BETWEEN 2000 AND 5000; -- Datum-Bereich SELECT * FROM scores WHERE erreicht_am BETWEEN '2024-01-01' AND '2024-12-31';

Aggregatfunktionen

Aggregatfunktionen berechnen einen Wert ueber mehrere Zeilen. Ohne GROUP BY aggregieren sie die gesamte Tabelle.

COUNT(*)
Anzahl der Zeilen. COUNT(spalte) zaehlt Nicht-NULL-Werte.
SUM(spalte)
Summe aller Werte. SUM(punkte) = Gesamtpunkte.
AVG(spalte)
Durchschnitt. AVG(punkte) = mittlere Punktzahl.
MAX(spalte)
Groesster Wert. MAX(punkte) = Highscore.
MIN(spalte)
Kleinster Wert. MIN(punkte) = niedrigste Punktzahl.
-- Gesamte Tabelle aggregieren SELECT COUNT(*) AS spieler_anzahl, AVG(punkte) AS durchschnitt, MAX(punkte) AS highscore, SUM(punkte) AS gesamt_punkte FROM spieler; spieler_anzahl | durchschnitt | highscore | gesamt_punkte ----------------+--------------+-----------+-------------- 3 | 4463.33 | 7340 | 13390

GROUP BY: Aggregieren nach Gruppen

GROUP BY teilt die Tabelle in Gruppen auf und wendet Aggregatfunktionen auf jede Gruppe an.

-- Durchschnittliche Punkte pro Level SELECT level, AVG(punkte) AS avg_punkte, COUNT(*) AS anzahl FROM scores GROUP BY level ORDER BY level; level | avg_punkte | anzahl -------+------------+-------- 1 | 2340.50 | 8 2 | 4120.00 | 5 3 | 6890.25 | 4 -- Wie viele Scores hat jeder Spieler? SELECT spieler_id, COUNT(*) AS anzahl_scores, MAX(punkte) AS bester_score FROM scores GROUP BY spieler_id ORDER BY bester_score DESC;
Wichtige Regel: Jede Spalte im SELECT muss entweder in GROUP BY stehen ODER eine Aggregatfunktion sein. SELECT name, AVG(punkte) FROM scores GROUP BY level ist ein Fehler — name ist weder aggregiert noch in GROUP BY.

HAVING: GROUP BY filtern

WHERE filtert einzelne Zeilen (vor Aggregation). HAVING filtert Gruppen (nach Aggregation).

-- Nur Spieler mit mehr als 3 Scores anzeigen SELECT spieler_id, COUNT(*) AS anzahl_scores FROM scores GROUP BY spieler_id HAVING COUNT(*) > 3; -- WHERE (vor GROUP BY) + HAVING (nach GROUP BY) SELECT spieler_id, AVG(punkte) AS avg FROM scores WHERE erreicht_am >= '2024-01-01' -- Filter auf Zeilen GROUP BY spieler_id HAVING AVG(punkte) > 3000; -- Filter auf Gruppen

Ausfuehrungs-Reihenfolge von SQL

SQL wird nicht in der Reihenfolge ausgefuehrt wie es geschrieben wird:

1
FROM
Welche Tabelle?
2
WHERE
Zeilen filtern
3
GROUP BY
Gruppen bilden
4
HAVING
Gruppen filtern
5
SELECT
Spalten waehlen / berechnen
6
ORDER BY
Sortieren
7
LIMIT
Ergebnis begrenzen

Das erklaert warum du in WHERE keine Aggregatfunktionen verwenden kannst (Schritt 2 kommt vor Schritt 3) — dafuer gibt es HAVING.

Quiz

1. Du willst alle Level anzeigen, in denen mehr als 5 Scores erreicht wurden. Welche Klausel?

HAVING — WHERE filtert vor der Aggregation (einzelne Zeilen), HAVING filtert nach der Aggregation (Gruppen). Aggregatfunktionen wie COUNT() koennen nur in HAVING stehen, nicht in WHERE.

2. Was ist falsch an: SELECT level, name, COUNT(*) FROM scores GROUP BY level?

B — name ist nicht in GROUP BY und hat keine Aggregatfunktion. PostgreSQL weiss nicht welchen Namen er zeigen soll wenn mehrere Zeilen im selben Level sind. Loesung: name in GROUP BY ergaenzen oder entfernen.

3. BETWEEN 100 AND 500 — werden 100 und 500 selbst eingeschlossen?

C — BETWEEN ist immer inklusiv. BETWEEN 100 AND 500 entspricht >= 100 AND <= 500. Werte von 100 bis 500 werden alle zurueckgegeben.
