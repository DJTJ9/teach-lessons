Dart-Daten migrieren

Strategie, Schema-Design und Migration von App-Daten zu PostgreSQL

Diese Lektion ist kein Quiz-Kurs sondern ein Praxis-Leitfaden. Folge dem Prozess wenn deine Dart-Daten bereit fuer die Migration sind.

Schritt 1: Daten analysieren

Bevor du ein Schema designst, musst du verstehen welche Daten du hast.

1
Dart-Datenstruktur exportieren

Welche Klassen/Models hat die App? Welche Felder? Welche Typen? Welche Beziehungen existieren?

// Beispiel: typisches Dart-Model (SharedPreferences oder Hive) class Messung { final int id; final DateTime zeitpunkt; final double wert; final String einheit; final String? notiz; }
2
Daten-Export aus Dart-App

Daten als JSON oder CSV exportieren. Dart/Flutter kann direkt JSON serialisieren.

// Dart: Daten als JSON exportieren final json = messungen.map((m) => { 'id': m.id, 'zeitpunkt': m.zeitpunkt.toIso8601String(), 'wert': m.wert, 'einheit': m.einheit, 'notiz': m.notiz, }).toList(); // JSON in Datei schreiben: await File('export.json').writeAsString(jsonEncode(json));

Schritt 2: Schema designen

Dart-Typen zu PostgreSQL-Typen uebersetzen:

Dart --> PostgreSQL int --> INTEGER oder BIGINT double --> NUMERIC(12,4) oder FLOAT8 String --> TEXT bool --> BOOLEAN DateTime --> TIMESTAMPTZ String? --> TEXT (erlaubt NULL) List<T> --> Eigene Tabelle mit Foreign Key (1:N)
-- Schema fuer Messung-Klasse CREATE TABLE messungen ( id BIGINT PRIMARY KEY, -- aus Dart uebernehmen zeitpunkt TIMESTAMPTZ NOT NULL, wert NUMERIC(12,4) NOT NULL, einheit TEXT NOT NULL, notiz TEXT -- nullable (String?) ); -- Indexe fuer haeufige Abfragen CREATE INDEX idx_messungen_zeitpunkt ON messungen (zeitpunkt DESC); CREATE INDEX idx_messungen_einheit ON messungen (einheit);

Schritt 3: Daten importieren

Option A: COPY (schnellste Methode fuer grosse Datenmengen)

-- CSV aus Dart exportieren, dann in PostgreSQL importieren -- In psql: \COPY messungen (id, zeitpunkt, wert, einheit, notiz) FROM '/pfad/zu/export.csv' WITH (FORMAT csv, HEADER true, NULL '');

Option B: Python-Script fuer JSON-Import

# Python-Script: JSON -> PostgreSQL import json, psycopg2 from datetime import datetime conn = psycopg2.connect( dbname="meine_db", host="localhost", user="postgres" ) cur = conn.cursor() with open('export.json') as f: daten = json.load(f) for m in daten: cur.execute( """INSERT INTO messungen (id, zeitpunkt, wert, einheit, notiz) VALUES (%s, %s, %s, %s, %s) ON CONFLICT (id) DO NOTHING""", # Duplikate ignorieren (m['id'], m['zeitpunkt'], m['wert'], m['einheit'], m.get('notiz')) ) conn.commit() print(f"{cur.rowcount} Zeilen importiert")
ON CONFLICT (id) DO NOTHING macht den Import idempotent — du kannst das Script mehrfach ausfuehren ohne Fehler. Duplikate werden einfach uebersprungen.

Schritt 4: Daten verifizieren

-- Anzahl importierter Zeilen SELECT COUNT(*) FROM messungen; -- Zeitraum der Daten SELECT MIN(zeitpunkt), MAX(zeitpunkt) FROM messungen; -- Erste 10 Zeilen pruefen SELECT * FROM messungen ORDER BY zeitpunkt LIMIT 10; -- NULL-Check: gibt es unerwartete NULLs? SELECT COUNT(*) FROM messungen WHERE wert IS NULL; -- Doppelte IDs (sollten 0 sein) SELECT id, COUNT(*) FROM messungen GROUP BY id HAVING COUNT(*) > 1;

Schritt 5: Dart-App mit PostgreSQL verbinden

// Dart: postgres Package (pub.dev) // pubspec.yaml: postgres: ^3.0.0 import 'package:postgres/postgres.dart'; final conn = await Connection.open( Endpoint(host: 'localhost', database: 'meine_db'), settings: ConnectionSettings(sslMode: SslMode.disable), ); final result = await conn.execute( 'SELECT * FROM messungen WHERE zeitpunkt > @von ORDER BY zeitpunkt DESC LIMIT 100', parameters: {'von': DateTime.now().subtract(Duration(days: 30))}, ); for (final row in result) { print(row[0]); // Wert der ersten Spalte }

Checkliste: Migration abgeschlossen

Abnahmekriterien

Dart-Datenstruktur analysiert und PostgreSQL-Schema designed
Schema erstellt mit korrekten Typen, Constraints und Foreign Keys
Alle Daten importiert (COUNT() stimmt mit Dart-Export ueberein)
Keine unerwarteten NULLs oder Duplikate
Indexe fuer haeufige Abfragen erstellt
Dart-App kann Daten aus PostgreSQL lesen
Mindestens eine Analyse-Abfrage (z.B. Durchschnitt, Zeitreihe) funktioniert

Quiz

1. Dart-Typ DateTime wird zu welchem PostgreSQL-Typ?

TIMESTAMPTZ — mit Zeitzone (TZ). DateTime in Dart enthaelt immer Zeitzone-Information. TIMESTAMP ohne TZ verliert die Zeitzone-Info und kann zu Fehlern fuehren.

2. Was bewirkt ON CONFLICT (id) DO NOTHING beim Import?

B — Idempotenter Import. Das Script kann mehrfach ausgefuehrt werden. Bereits importierte Zeilen werden still uebersprungen. Besonders nuetzlich wenn der Import unterbrochen und neu gestartet wird.

3. Wann solltest du COPY statt INSERT fuer den Import verwenden?

B — COPY ist massiv schneller fuer Bulk-Imports da es den Parser und Transaktions-Overhead umgeht. Fuer wenige hundert Zeilen ist INSERT einfacher. Fuer Tausende oder Millionen Zeilen: COPY.
