Subqueries und CTEs

Abfragen in Abfragen und WITH-Klauseln fuer lesbare komplexe Queries

Was ist eine Subquery?

Eine Subquery (Unterabfrage) ist ein SELECT innerhalb eines anderen SELECT. Sie wird in runden Klammern geschrieben und kann an drei Stellen stehen: im WHERE, im FROM oder im SELECT.

Subquery im WHERE

-- Spieler die besser als der Durchschnitt sind SELECT username, punkte FROM spieler WHERE punkte > (SELECT AVG(punkte) FROM spieler); -- Erklaerung: Die innere Subquery berechnet AVG(punkte) = 4463 -- Die aeussere Query filtert dann: WHERE punkte > 4463 -- Mit IN: Spieler die mindestens einen Score > 5000 haben SELECT username FROM spieler WHERE id IN (SELECT spieler_id FROM scores WHERE punkte > 5000);

Subquery im FROM (Derived Table)

-- Spieler-Rang berechnen (ohne Window Functions) SELECT username, gesamt, rang FROM ( SELECT s.username, SUM(sc.punkte) AS gesamt, RANK() OVER (ORDER BY SUM(sc.punkte) DESC) AS rang FROM spieler s LEFT JOIN scores sc ON s.id = sc.spieler_id GROUP BY s.id, s.username ) AS rangliste WHERE rang <= 3; username | gesamt | rang ----------+--------+------ Carol | 7340 | 1 Alice | 8000 | 2 Bob | 1850 | 3

CTEs: WITH-Klausel

CTEs (Common Table Expressions) mit WITH machen komplexe Queries lesbar — die Subquery bekommt einen Namen und kann mehrfach referenziert werden.

Subquery (unleserlich)
SELECT * FROM ( SELECT s.username, MAX(sc.punkte) best FROM spieler s JOIN scores sc ON s.id=sc.spieler_id GROUP BY s.id,s.username ) t WHERE t.best > 4000;
CTE (lesbar)
WITH bester_score AS ( SELECT s.username, MAX(sc.punkte) AS best FROM spieler s JOIN scores sc ON s.id = sc.spieler_id GROUP BY s.id, s.username ) SELECT * FROM bester_score WHERE best > 4000;

Mehrere CTEs verketten

WITH -- CTE 1: Bester Score je Spieler bester_score AS ( SELECT spieler_id, MAX(punkte) AS highscore FROM scores GROUP BY spieler_id ), -- CTE 2: Nur aktive Spieler aktive_spieler AS ( SELECT id, username FROM spieler WHERE ist_aktiv = TRUE ) -- Finale Abfrage nutzt beide CTEs SELECT a.username, b.highscore FROM aktive_spieler a JOIN bester_score b ON a.id = b.spieler_id ORDER BY b.highscore DESC;
Faustregel: Wenn eine Subquery mehr als 2 Zeilen hat oder mehr als einmal referenziert wird — CTE verwenden. CTEs sind kostenlos (kein Performance-Overhead) und viel einfacher zu debuggen.

Wann Subquery, wann CTE, wann JOIN?

JOIN: Wenn du Zeilen aus zwei Tabellen kombinieren willst Subquery: Wenn du einen einzigen Wert berechnen willst (z.B. Durchschnitt) CTE: Wenn die Subquery laenger als 3 Zeilen ist oder mehrfach gebraucht wird

Quiz

1. Was berechnet WHERE punkte > (SELECT AVG(punkte) FROM spieler)?

B — Die Subquery berechnet den Durchschnitt aller Punkte als eine Zahl. Die aeussere Query vergleicht dann jeden Spieler mit diesem Durchschnitt.

2. Wann ist ein CTE einer Subquery vorzuziehen?

B — Performance ist meist gleich. CTEs sind ein Lesbarkeitstool. Subquery-in-Klammern ist schnell fuer einfache Faelle, CTE ist besser fuer alles was groesser ist.

3. Kannst du in einem CTE einen anderen CTE referenzieren?

B — Im Beispiel oben nutzt die finale SELECT-Abfrage beide CTEs gleichzeitig. Spaetere CTEs koennen auch fruehere CTEs als Eingabe nutzen — das ist eines der maechtigen Features von CTEs.
