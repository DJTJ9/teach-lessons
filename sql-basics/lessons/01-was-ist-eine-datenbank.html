Was ist eine Datenbank?

Tabellen, Zeilen, Schluessel — die Grundbausteine relationaler Datenbanken

Das Kern-Konzept: Tabellen

Eine relationale Datenbank speichert Daten in Tabellen. Eine Tabelle ist wie ein Excel-Sheet mit einer wichtigen Einschraenkung: jede Spalte hat einen festen Datentyp und jede Zeile repraesentiert genau einen Datensatz.

Tabelle: spieler
idnamepunkteregistriert_am
1Alice42002024-01-15
2Bob18502024-02-03
3Carol73402024-01-28
Tabelle (Table)
Sammlung gleichartiger Datensaetze. Entspricht einer Klasse in OOP — jede Zeile ist eine Instanz.
Zeile (Row / Record)
Ein einzelner Datensatz. "Alice hat 4200 Punkte" = eine Zeile.
Spalte (Column / Field)
Eine Eigenschaft aller Datensaetze. "punkte" ist eine Spalte — alle Spieler haben Punkte.
Primary Key (PK)
Eindeutige ID jedes Datensatzes. In der Tabelle oben: id. Kein Datensatz darf dieselbe ID haben.

Datenbank vs. Excel: der Unterschied

Excel / Tabellenkalkulation
  • Spalten koennen gemischte Typen haben
  • Keine erzwungene Eindeutigkeit
  • Abfragen per Formel, schwer zu automatisieren
  • Gut fuer: kleine Datasets, Visualisierung
  • Schlecht fuer: mehrere Benutzer gleichzeitig
Relationale Datenbank
  • Jede Spalte hat exakten Typ (INTEGER, TEXT...)
  • Primary Key erzwingt Eindeutigkeit
  • Abfragen per SQL, vollstaendig automatisierbar
  • Gut fuer: grosse Datasets, mehrere Apps
  • Gut fuer: gleichzeitige Schreibzugriffe

Mehrere Tabellen: das relationale Prinzip

Eine Datenbank hat meistens mehrere Tabellen, die miteinander verknuepft sind. Das ist das "relationale" in "relational".

Tabelle: spieler
idname
1Alice
2Bob
Tabelle: scores
idspieler_idpunkte
114200
213800
321850

spieler_id in der scores-Tabelle verweist auf id in der spieler-Tabelle. Alice (id=1) hat zwei Score-Eintraege. Das ist ein Foreign Key — mehr dazu in Lektion 08.

Datenbanken trennen Daten von Logik. Deine Dart-App, ein Python-Script und ein Web-Frontend koennen alle auf dieselbe PostgreSQL-Datenbank zugreifen — jeder liest und schreibt dieselben Daten.

Quiz

1. Was ist der Unterschied zwischen einer Zeile und einer Spalte?

B — Zeile: "Alice, 4200, 2024-01-15" = ein Spieler. Spalte: "punkte" = die Punkte-Eigenschaft aller Spieler.

2. Warum braucht jede Tabelle einen Primary Key?

B — Zwei Spieler koennen "Alice" heissen. Mit PK-ID 1 und 2 sind beide trotzdem eindeutig. Ohne PK kann man keinen bestimmten Datensatz gezielt ansprechen oder veraendern.

3. Alice hat zwei Eintraege in der scores-Tabelle (4200 und 3800 Punkte). Wie ist das moeglich?

B — Alice bleibt eine Zeile in spieler (id=1). Aber scores kann beliebig viele Zeilen mit spieler_id=1 haben. Das ist eine 1:N-Beziehung — ein Spieler, viele Scores.
