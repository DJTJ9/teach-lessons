Termux — Claude Code + SSH auf Android

~14 Min · Baustein 9/11 · baut auf Tailscale (L2) + SSH (L1) auf

Das Problem

Android-Handys sind normalerweise eine "Konsumenten-Sandbox" — kein echtes Terminal, keine Möglichkeit, Linux-Kommandozeilen-Tools (git, python, ssh, node) direkt auszuführen. Willst du unterwegs (z.B. vom Z-Flip5) auf deinen Server zugreifen oder sogar Claude Code direkt auf dem Handy nutzen, brauchst du eine echte Linux-Umgebung auf einem Gerät, das eigentlich keine hat.

Die Lösung: Ein Linux-Terminal-Emulator ohne Root

Termux ist eine App, die eine minimale Linux-Umgebung (Terminal + Paketmanager) innerhalb der normalen Android-App-Sandbox bereitstellt — ohne Root-Zugriff auf das Telefon zu benötigen. Über pkg install (Termux' eigenes apt-ähnliches Paketsystem) installierst du z.B. openssh, python, nodejs, git — echte Linux-Binaries, kompiliert für Android's Prozessor-Architektur.

pkg install openssh nodejs git python
npm install -g @anthropic-ai/claude-code   # Claude Code CLI direkt im Termux-Terminal
ssh dev   # SSH-Verbindung zum Server über Tailscale-Hostname/IP
Warum das ohne Root funktioniert: Termux nutzt Androids offizielle App-Sandbox-Mechanismen (jede App bekommt ihr eigenes, isoliertes Dateisystem) und baut Linux-Pakete gegen Androids Bionic-libc statt gegen glibc. Das Ergebnis: ein vollwertiges Linux-Terminal, das komplett innerhalb der normalen App-Berechtigungen läuft — kein Sicherheitsrisiko, keine Root-Voraussetzung, direkt aus dem Play-Store/F-Droid installierbar.

Wie das in deine Kette passt

Dein Z-Flip5 (z-flip5-von-tjark, 100.67.54.96) ist ein vollwertiger Knoten in deinem Tailscale-Tailnet (Lesson 2). Sobald Termux + SSH-Client installiert sind, kannst du vom Handy aus genau wie von deinem PC per ssh root@100.64.150.74 auf den Server — Key-Auth (Lesson 1) funktioniert identisch, weil Termux ein echtes ~/.ssh-Verzeichnis mit Private Key verwaltet.

Das macht dein Handy zu einem dritten, gleichwertigen Entwickler-Client neben PC und iPad: Code-Änderungen pushen, Bot-Logs live mitlesen (journalctl -u bot-teach -f), sogar Claude Code direkt im Termux-Terminal starten und mit dem Server-Code arbeiten — alles unterwegs, ohne dass der Server dafür zusätzliche Ports öffnen muss.

Quiz: Warum reicht die normale Android-Terminal-App (falls vorhanden) nicht für SSH/git/Claude Code?

Standard-Android-Terminal-Apps geben dir bestenfalls Zugriff auf eine sehr eingeschränkte Shell ohne Paketmanager — du kannst keine echten Linux-Binaries wie openssh, node oder git nachinstallieren. Termux bringt einen eigenen Paketmanager (pkg) mit, der Pakete speziell für Androids Architektur bereitstellt — das ist der entscheidende Unterschied zwischen "ein Terminal-Fenster" und "eine tatsächlich nutzbare Linux-Entwicklungsumgebung".

Quiz: Was bräuchtest du zusätzlich, damit ssh dev (statt der vollen IP) auf dem Handy funktioniert?

Einen SSH-Client-Config-Eintrag in ~/.ssh/config innerhalb von Termux, der den Kurznamen dev auf die Tailscale-IP/den Tailscale-Hostnamen (100.64.150.74 bzw. den MagicDNS-Namen) sowie den Private-Key-Pfad mappt — genau das gleiche Prinzip wie die ~/.ssh/config auf deinem PC (Lesson 1), nur eben in Termux' eigenem Dateisystem eingerichtet.

Zusammenfassung

Termux macht aus einem normalen, ungerooteten Android-Handy einen vollwertigen Linux-Client — mit echtem Paketmanager, SSH, git und sogar Claude Code. Zusammen mit Tailscale (Netzwerk-Zugang) und SSH-Key-Auth (Authentifizierung) wird dein Handy so zu einem gleichberechtigten Entwickler-Gerät neben PC und iPad, ohne dass am Server irgendetwas Handy-spezifisches konfiguriert werden muss.
