🎮 Lektion 07 — Unity Test Framework

Ziel: Das Unity Test Framework (NUnit) einrichten, EditMode- und PlayMode-Tests unterscheiden, und erste C#-Tests schreiben.

Testing in Unity — Übersicht

Unity verwendet NUnit als Testbibliothek — genau wie man in C# sonst xUnit oder MSTest nutzt. Unity baut eine eigene UI drum herum: den Test Runner.

Öffnen: Window → General → Test Runner

EditMode vs. PlayMode Tests

EditModePlayMode
Läuft ohne die Unity-Engine zu startenStartet die Engine, läuft im echten Spielkontext
Schnell (~Millisekunden)Langsam (~Sekunden)
Für reine Logik: Berechnungen, Datenstrukturen, AlgorithmenFür Physics, Coroutines, MonoBehaviour-Lifecycle
Kein MonoBehaviour möglichVollständiger Unity-Lifecycle verfügbar
Entspricht Unit TestsEntspricht Integration Tests
Faustregel: Starte immer mit EditMode Tests. Nur wenn du MonoBehaviour, Physics, oder Coroutines testen musst, wechsle zu PlayMode.

Dateistruktur in Unity

Assets/
  Scripts/
    PlayerHealth.cs        ← Production Code
    Inventory.cs
  Tests/
    EditMode/
      PlayerHealthTests.cs  ← EditMode Tests
      InventoryTests.cs
    PlayMode/
      GameFlowTests.cs      ← PlayMode Tests

Tests-Ordner benötigen eine asmdef-Datei (Assembly Definition). Unity erstellt diese automatisch wenn du über den Test Runner einen neuen Test-Ordner anlegst.

NUnit Syntax — Vergleich Python/pytest

🐍 pytest (Python)

def test_health_nicht_negativ():
    health = PlayerHealth(100)
    health.TakeDamage(200)
    assert health.Current == 0

💜 NUnit (C#)

[Test]
public void Health_NichtNegativ()
{
    var health = new PlayerHealth(100);
    health.TakeDamage(200);
    Assert.AreEqual(0, health.Current);
}

NUnit Attribute

AttributBedeutung
[Test]Markiert eine Test-Methode (wie def test_ in pytest)
[TestFixture]Markiert eine Test-Klasse (optional in Unity)
[SetUp]Läuft vor jedem Test (wie pytest setup_method)
[TearDown]Läuft nach jedem Test (Aufräumen)
[TestCase(1, 2, 3)]Parametrisierter Test (wie pytest @parametrize)

NUnit Assertions

// Grundlegende Assertions:
Assert.AreEqual(expected, actual);           // ==
Assert.AreNotEqual(unexpected, actual);      // !=
Assert.IsTrue(condition);                    // assert x
Assert.IsFalse(condition);                   // assert not x
Assert.IsNull(obj);                          // assert x is None
Assert.IsNotNull(obj);                       // assert x is not None
Assert.That(actual, Is.EqualTo(expected));   // moderner Stil

// Exception erwarten:
Assert.Throws(() => {
    new PlayerHealth(-1);                    // soll Exception werfen
});

Erster Test — PlayerHealth Beispiel

// Assets/Tests/EditMode/PlayerHealthTests.cs
using NUnit.Framework;

public class PlayerHealthTests
{
    private PlayerHealth health;

    [SetUp]
    public void Setup()
    {
        health = new PlayerHealth(100);  // vor jedem Test
    }

    [Test]
    public void StartHealth_IstMaxHealth()
    {
        Assert.AreEqual(100, health.Current);
    }

    [Test]
    public void TakeDamage_ReduziertHealth()
    {
        health.TakeDamage(30);
        Assert.AreEqual(70, health.Current);
    }

    [Test]
    public void TakeDamage_UnterNull_BleibtBeiNull()
    {
        health.TakeDamage(200);
        Assert.AreEqual(0, health.Current);
    }

    [Test]
    public void Heal_ErhoehtHealth()
    {
        health.TakeDamage(50);
        health.Heal(20);
        Assert.AreEqual(70, health.Current);
    }
}

Tests ausführen

Im Unity Test Runner: Run All oder Run Selected. Grüner Haken = bestanden, rotes X = fehlgeschlagen.

🧠 Quiz

Frage 1: Du willst Physics-Kollisionen testen. EditMode oder PlayMode?

A) EditMode — schneller
❌ Falsch. Physics braucht den Unity-Runtime, der nur in PlayMode läuft.
B) PlayMode — Physics braucht den Unity-Runtime
✅ Richtig! Kollisionen, Rigidbody, Coroutines → PlayMode.

Frage 2: Was ist das C#-Äquivalent zu pytest's def test_name():?

A) Eine Methode die mit Test anfängt
❌ Falsch. In NUnit wird der Name nicht automatisch erkannt — du brauchst das Attribut.
B) Eine Methode mit dem [Test] Attribut
✅ Richtig! [Test] markiert die Methode für den Test Runner.

Aufgabe: Schreibe den Test-Stub für: "Wenn Spieler geheilt wird über MaxHealth hinaus, bleibt Current bei MaxHealth."

Zeig mir die Lösung
[Test]
public void Heal_UeberMaxHealth_BleibtBeiMaxHealth()
{
    health.Heal(200);  // maxHealth = 100, Heal soll bei 100 deckeln
    Assert.AreEqual(100, health.Current);
}

📋 Zusammenfassung

  • Unity nutzt NUnit als Testbibliothek
  • EditMode: schnelle Unit Tests ohne Engine-Start, für reine Logik
  • PlayMode: langsame Integration Tests, für Physics/Coroutines/Lifecycle
  • [Test] markiert Tests, [SetUp] initialisiert vor jedem Test
  • Assert.AreEqual(expected, actual) — beachte: expected kommt zuerst!
  • Test Runner: Window → General → Test Runner
