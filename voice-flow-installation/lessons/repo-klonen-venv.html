Repo klonen + venv anlegen

Ziel: der Code liegt lokal, eine isolierte Python-Umgebung (venv) ist angelegt — inklusive Fix für einen Fehler, der dabei fast garantiert auftritt.

1

Git installieren (falls noch nicht vorhanden)

Terminal öffnen, prüfen:

PowerShell
PS> git --version git version 2.44.0.windows.1

Kommt 'git' is not recognized: git-scm.com/download/win herunterladen, Installer mit Standardeinstellungen durchklicken.

2

Repo klonen

In den Ordner wechseln, wo die App leben soll (z.B. Benutzerordner), dann:

PowerShell
PS> git clone https://github.com/DJTJ9/voice-recorder.git voice-flow PS> cd voice-flow

Repo ist privat — Git fragt nach GitHub-Login (Browser-Popup oder Personal Access Token). Mit dem Account einloggen, der Zugriff auf DJTJ9/voice-recorder hat.

3

venv anlegen

PowerShell
PS> py -3.12 -m venv .venv

Erzeugt den Ordner .venv — eine von Windows isolierte Python-Installation nur für dieses Projekt.

4

Der Stolperstein: venv aktivieren

Der übliche Befehl .venv\Scripts\activate schlägt auf vielen Windows-Rechnern so fehl:

PowerShell
PS> .venv\Scripts\activate File ...\Activate.ps1 cannot be loaded because running scripts is disabled on this system. CategoryInfo : SecurityError FullyQualifiedErrorId : UnauthorizedAccess

Grund: Windows blockiert PowerShell-Skripte (.ps1) standardmäßig aus Sicherheitsgründen — activate ist so ein Skript.

Fix — zwei Wege, einen wählen:

A) Aktivieren umgehen (kein Eingriff ins System): Statt python immer .venv\Scripts\python.exe direkt aufrufen.

B) Skripte dauerhaft erlauben (einmalig, nur für deinen Windows-Account, kein Admin nötig):
PowerShell
PS> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
Danach funktioniert .venv\Scripts\activate normal.

Warum überhaupt ein venv?

Voice Flow braucht bestimmte Paketversionen (z.B. faster-whisper). Ohne venv landen die in der globalen Python-Installation und können mit anderen Projekten kollidieren. Das venv ist ein Ordner (.venv) mit eigener, isolierter Kopie von Python + Paketen — löschen und neu anlegen jederzeit gefahrlos möglich.

Kurzcheck

Warum blockiert Windows den activate-Befehl standardmäßig?

Gelernt: Git installieren, Repo klonen, venv mit py -3.12 -m venv .venv anlegen, Execution-Policy-Fehler beim Aktivieren erkennen und mit Weg A (venv-Python direkt) oder Weg B (Set-ExecutionPolicy) lösen.
