Fehlerbehebung — die 3 echten Fehler, real aufgetreten

Ziel: du erkennst die drei Fehlerbilder, die beim ersten Laptop-Setup tatsächlich passiert sind, sofort an ihrer Fehlermeldung und weißt, was zu tun ist — ohne nachzufragen.

① PowerShell blockiert das venv
File ...\Activate.ps1 cannot be loaded because running scripts is disabled on this system. CategoryInfo : SecurityError FullyQualifiedErrorId : UnauthorizedAccess
Ursache:

Windows/PowerShell blockiert .ps1-Skripte standardmäßig, unabhängig von Voice Flow.

Fix:

.venv\Scripts\python.exe statt python direkt verwenden, oder einmalig Set-ExecutionPolicy -Scope CurrentUser RemoteSigned ausführen. Details: Lektion 3.

② cublas-DLL fehlt (GPU-Pfad)
RuntimeError: Library cublas64_12.dll is not found or cannot be loaded
Ursache:

NVIDIA-Treiber ist da (CUDA wird erkannt), aber die Python-Pakete nvidia-cublas-cu12/nvidia-cudnn-cu12 fehlen im venv. Seit dem Fix vom 2026-07-22 (Commit 76682ff) fängt die App diesen Fall automatisch ab und rechnet stattdessen auf CPU weiter — aber nur, wenn whisper_device auf "auto" steht.

Fix:

GPU nutzen wollen: .venv\Scripts\python.exe -m pip install nvidia-cublas-cu12 nvidia-cudnn-cu12 (Lektion 4, Schritt 3). GPU egal: nichts tun, App läuft automatisch auf CPU weiter.

③ 401 Invalid authentication
API Error: 401 Invalid authentication credentials
Ursache:

Die Claude CLI auf diesem Gerät ist nicht (mehr) eingeloggt. STT lief bereits durch — nur der Cleanup-Aufruf (claude -p ...) scheitert.

Fix:

Terminal: claude eintippen, mit demselben Konto wie am Hauptgerät einloggen. Testen mit claude -p "sag hallo". Details: Lektion 5.

Bonus: harmlose Meldung, die wie ein Fehler aussieht

RuntimeError: Calling Tcl from different apartment (im Thread Thread-7 (mainloop)) — ein Bug im Taskleisten-Fenster, der mit demselben Fix wie ② behoben wurde. Falls das nach einem git pull immer noch auftaucht: die App läuft trotzdem weiter, nur das Taskleisten-Fenster fehlt dann, Tray-Icon und Diktat funktionieren unabhängig davon.

Genereller Diagnose-Ablauf

  1. Immer erst mit desktop\start.bat starten (nicht start_silent.vbs) — nur die Konsole zeigt Fehlertexte.
  2. Fehlermeldung mit den drei oben genannten vergleichen (Stichwörter: Activate.ps1, cublas64_12.dll, 401 Invalid authentication).
  3. Kein Treffer? Log-Dateien prüfen: ~/.voice-flow/update.log (Auto-Update) und ~/.voice-flow/timing.log (Laufzeiten).

Kurzcheck — welcher Fehler ist das?

Konsole zeigt: cannot be loaded because running scripts is disabled on this system

Gelernt: Die drei real aufgetretenen Fehlerbilder anhand ihrer Fehlermeldung erkennen (Execution Policy, cublas-DLL, 401 Auth), plus die harmlose Tk-Apartment-Meldung. Damit ist die Installation auf einem neuen Windows-Gerät eigenständig durchführbar — von Voraussetzungen bis Troubleshooting.
