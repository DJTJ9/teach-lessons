Lektion 02

Notion-Datenbanken anlegen: Projekte & Epics

Lektion 2 von ~6 · Dauer: ~25 min · 2026-06-10

Was du heute lernst

Du kennst das 3-Ebenen-Modell. Jetzt bauen wir die erste Hälfte des Systems in Notion: eine Projekte-Datenbank und eine Epics-Datenbank, verknüpft über eine Notion-Relation.

Designprinzip: Properties bewusst wählen

Jede Property kostet Pflege. Nur Properties anlegen, die du aktiv nutzt — lieber drei gezielte Felder als zehn halbgare.

Goldene Regel: Eine Datenbank, die zu komplex ist, wird nicht genutzt. Start simple — Properties können später immer noch hinzugefügt werden.

Projekte-Datenbank: Properties

Was braucht eine Projekte-DB wirklich? Fünf Properties reichen:

Property Typ Werte / Zweck
Name Pflicht title z.B. "Dart-Trainings-App"
Status Pflicht status Idee · Aktiv · Pausiert · Abgeschlossen
Bereich Pflicht select Arbeit · Privat · Lernen
Gleiche Werte wie im Tagesorganizer → später einfacher filtern
Zieldatum optional date Wann soll das Projekt fertig sein?
Beschreibung optional rich_text Ein Satz "Warum" — hilfreich bei späterem Rückblick

Projekte-DB anlegen: Prompt für Claude Code

Kopiere diesen Prompt in Claude Code (oder den Telegram-Bot):

Claude Code → eingebenErstelle in Notion eine neue Datenbank "Projekte" auf einer neuen Seite im Workspace. Properties: - Name (title) - Status (status): Idee, Aktiv, Pausiert, Abgeschlossen - Bereich (select): Arbeit, Privat, Lernen - Zieldatum (date) - Beschreibung (rich_text)

Was passiert danach

Claude nutzt das Notion-MCP um die Datenbank anzulegen. Du bekommst die database_id zurück — notiere sie, du brauchst sie später für den Bot.

Epics-Datenbank: Properties

Epics brauchen eine Relation zur Projekte-DB. Das ist der entscheidende Unterschied zu einer normalen Datenbank: Ein Epic gehört immer zu genau einem Projekt.

1 Projekt
↔
🧩 Epics
n Epics

Eine Relation in Notion ist ein Zeiger: Das Feld "Projekt" in einem Epic zeigt auf einen Eintrag in der Projekte-DB. Notion zeigt dann auf beiden Seiten die Verknüpfung.

Property Typ Werte / Zweck
Name Pflicht title z.B. "User Login", "Statistik"
Projekt Pflicht relation Zeigt auf Projekte-DB
Jedes Epic gehört zu einem Projekt
Status Pflicht status Offen · In Arbeit · Fertig
Priorität Pflicht select Hoch · Mittel · Niedrig

Epics-DB anlegen: Prompt für Claude Code

Claude Code → eingeben (nach Projekte-DB anlegen)Erstelle in Notion eine neue Datenbank "Projekt-Epics" auf einer neuen Seite im Workspace. Properties: - Name (title) - Projekt (relation zur "Projekte"-Datenbank) - Status (status): Offen, In Arbeit, Fertig - Priorität (select): Hoch, Mittel, Niedrig
Hinweis zur Relation: Claude Code sucht automatisch nach der Projekte-DB per Notion-MCP. Falls es mehrere Datenbanken mit ähnlichem Namen gibt, gib die database_id der Projekte-DB direkt an.

Das Resultat: deine neue Struktur

1
Projekte-DB — dein übergeordnetes Zielregister
z.B. "Dart-Trainings-App" (Aktiv, Arbeit)
2
Epics-DB — Themencluster, verknüpft mit Projekten
z.B. "User Login" (Projekt: Dart-App, Priorität: Hoch)
3
Verknüpfung — Epic kennt sein Projekt
In Notion siehst du beim Projekt alle zugehörigen Epics

Praxis-Aufgabe

Führe diese vier Schritte jetzt durch:

  • Schritt 1: Projekte-DB in Claude Code anlegen (Prompt oben nutzen)
  • Schritt 2: Epics-DB anlegen (zweiten Prompt nutzen)
  • Schritt 3: Dein erstes Projekt eintragen — z.B. "Dart-Trainings-App" oder ein eigenes
  • Schritt 4: 2–3 Epics für das Projekt anlegen und mit dem Projekt verknüpfen

Ziel: Beide Datenbanken existieren in Notion, mindestens ein Projekt mit zugehörigen Epics ist eingetragen.
