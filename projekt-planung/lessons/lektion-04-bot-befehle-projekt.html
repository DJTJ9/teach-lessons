Lektion 04

Bot-Befehle für Projekte

Lektion 4 von ~6 · Dauer: ~30 min · 2026-06-10

Was du heute lernst

Der Tagesorganizer hat jetzt die Properties Projekt und Epic. Aber der Bot kennt sie noch nicht wirklich — dart: ... leitet nur ins richtige Verzeichnis um. Heute bringst du dem Bot bei, gezielt nach Projekt-Tasks zu filtern und Tasks mit Projektkontext anzulegen.

Das aktuelle Problem

Die Lücke im Routing

Wenn du dart: zeig meine Tasks schickst, passiert folgendes:

dart: zeig Tasks
→
prefix erkannt
→
cwd = DartApp
→
Claude allgemein

Claude antwortet aus dem DartApp-Ordner — aber ohne den Tagesorganizer nach Projekt zu filtern.

× JETZT
dart: zeig Tasks

→ Claude im DartApp-Ordner, freie Antwort ohne Notion-Filter

✓ ZIEL
dart: tasks

→ Tagesorganizer: Projekt="Dart-App", offen, sortiert — präzise Liste

Die Lösung: Zwei neue Befehle

  • dart: tasks — zeige offene Tasks im Tagesorganizer wo Projekt = "Dart-App"
  • dart: task: Text — lege Task an und setze Projekt automatisch auf "Dart-App"
Schlüsselerkenntnis: Das Präfix liefert nicht nur das Arbeitsverzeichnis — es liefert auch den Projektnamen für Notion. dart: → notion_name = "Dart-App".

Schritt 1: PROJECTS auf Dict-of-Dicts umstellen

# Vorher
PROJECTS = {
    "notion": r"C:\Projekte\telegram-notion-bot",
    "dart":   r"C:\Unity\Aktuelle Projekte\DartTrainingsApp",
}

# Nachher
PROJECTS = {
    "notion": {"path": r"C:\Projekte\telegram-notion-bot",         "notion_name": "Notion-Bot"},
    "dart":   {"path": r"C:\Unity\Aktuelle Projekte\DartTrainingsApp", "notion_name": "Dart-App"},
}
Wichtig: notion_name muss exakt mit dem Select-Wert im Tagesorganizer übereinstimmen — also "Dart-App", nicht "dart" oder "DartApp".

Der bestehende projekte-Befehl liest Pfade aus — er muss auf info["path"] angepasst werden:

if text.lower() == "projekte":
    lines = ["📁 Verfügbare Projekte:"]
    for name, info in PROJECTS.items():
        lines.append(f"  {name}: → {info['path']}")
    lines.append("\nNutzung: <name>: <frage>")
    send_message(chat_id, "\n".join(lines))
    continue

Schritt 2: Zwei neue System-Prompts

PROJEKT_TASKS_SYSTEM_PROMPT = """Du bist ein Notion-Projektassistent.
Lies den Tagesorganizer (data_source_id: c9d2abbe-5607-44c2-bbf4-9aa673e0c4a0).
Zeige alle Tasks wo Property "Projekt" = dem angegebenen Projektnamen,
Status Not started oder In progress.
Sortiere nach Priorität (Hoch zuerst), dann Datum.
Format:
Erste Zeile: "📁 [Projektname] – [N] offene Tasks"
Je Task: · [Epic falls gesetzt] [Priorität] [Name] — [Datum]
Falls keine Tasks: "📁 [Projektname] – keine offenen Tasks."
Kein Markdown."""

PROJEKT_TASK_SYSTEM_PROMPT = """Du bist ein Notion-Task-Assistent.
Lege einen Task im Tagesorganizer an (data_source_id: c9d2abbe-5607-44c2-bbf4-9aa673e0c4a0).
Der Projektname wird vorgegeben — setze ihn als Property "Projekt".
Leite aus dem Text ab: Name, Datum (heute falls nicht angegeben),
Priorität (Hoch/Mittel/Niedrig, Mittel falls nicht angegeben),
Bereich (Arbeit/Privat/Lernen/Gesundheit, Arbeit falls Projekt gesetzt).
Epic: setze es falls im Text erkennbar, sonst leer lassen.
Antworte NUR mit: ✓ Task angelegt: [Name] · [Projekt] · [Priorität] · [Datum]"""

Schritt 3: Routing anpassen

# Präfix-Erkennung — ersetzt bisherige Variante
project_cwd = None
project_notion_name = None
for name, info in PROJECTS.items():
    prefix = f"{name}:"
    if text.lower().startswith(prefix):
        project_cwd = info["path"]
        project_notion_name = info["notion_name"]
        text = text[len(prefix):].strip()
        break

# Neue Zweige VOR dem bisherigen "task:" check
if project_notion_name and text.lower() == "tasks":
    prompt = f"Heute ist {today}. Projektname: {project_notion_name}"
    response = run_claude(prompt, system_prompt=PROJEKT_TASKS_SYSTEM_PROMPT)
elif project_notion_name and text.lower().startswith("task:"):
    task_text = text[5:].strip()
    prompt = f"Heute ist {today}. Projektname: {project_notion_name}. Aufgabe: {task_text}"
    response = run_claude(prompt, system_prompt=PROJEKT_TASK_SYSTEM_PROMPT)
elif text.lower().startswith("task:"):
    # bisheriger Zweig ohne Projektkontext
    task_text = text[5:].strip()
    prompt = f"Heute ist {today}. Aufgabe: {task_text}"
    response = run_claude(prompt, system_prompt=TASK_SYSTEM_PROMPT, cwd=project_cwd)
elif text.lower() == "woche":
    response = run_claude(f"Heute ist {today}.", system_prompt=WOCHE_SYSTEM_PROMPT)
# ... weitere elif wie bisher ...
Warum die Reihenfolge wichtig ist: project_notion_name and text == "tasks" muss vor dem allgemeinen text.lower().startswith("task:") kommen — sonst trifft nie der neue Zweig.

Wie die Befehle aussehen

📱
Du schreibst:
dart: tasks
🤖
Bot antwortet:
📁 Dart-App – 3 offene Tasks
· User Login · Hoch · Login-Screen implementieren — 2026-06-10
· Statistik · Mittel · Würfe-Daten speichern — 2026-06-11
· UI · Niedrig · Hauptmenü polieren — 2026-06-12
📱
Du schreibst:
dart: task: Highscore-Anzeige, Priorität Hoch
🤖
Bot antwortet:
✓ Task angelegt: Highscore-Anzeige · Dart-App · Hoch · 2026-06-10

Warum dieser Aufbau skaliert

  • 1
    Neues Projekt = eine Zeile — einfach "unity": {"path": ..., "notion_name": "Unity-Game"} in PROJECTS ergänzen. Alle Befehle funktionieren automatisch.
  • 2
    Präzise Notion-Filter — System-Prompts geben Claude den genauen Property-Namen und Wert. Kein Raten, kein Halluzinieren.
  • 3
    Morgen-Workflow bleibt unverändert — moin zeigt weiter alle Tasks inkl. Projekt-Tasks, weil sie im Tagesorganizer liegen.

Praxis-Aufgabe

  • Schritt 1: PROJECTS-Dict auf path + notion_name umstellen
  • Schritt 2: projekte-Befehl auf info["path"] anpassen
  • Schritt 3: Beide neue System-Prompts hinzufügen
  • Schritt 4: Routing-Block wie oben ändern (neue elif-Zweige)
  • Schritt 5: Bot neu starten → dart: tasks testen
  • Schritt 6: dart: task: Test-Task Lektion 4 senden → in Notion prüfen ob Projekt = "Dart-App" gesetzt ist

Ziel: Beide Befehle antworten korrekt, Task in Notion hat das richtige Projektfeld.
