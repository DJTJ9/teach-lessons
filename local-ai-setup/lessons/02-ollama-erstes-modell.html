Ollama — Erstes Modell lokal starten

Von Zero zu laufendem Qwen2.5-Coder in 15 Minuten auf Windows

1. Was ist Ollama?

Ollama ist der einfachste Weg, lokale LLMs auf Windows zu betreiben. Es bündelt die llama.cpp Engine, einen Modell-Manager und eine REST-API in einem einzigen Tool — ohne Docker, ohne kompliziertes Setup.

Ollamavs. Alternative
1 Zeile Befehl — Modell läuftLM Studio: GUI, mehr Klicks nötig
REST-API auf localhost:11434llama.cpp: mehr Kontrolle, aber komplexer Setup
Automatische GPU-Erkennung (CUDA)Manuelles GGUF-Laden entfällt
Modelfiles für Custom PromptsKein Vendor Lock-in

2. Installation (Windows, 4 Schritte)

1
Ollama herunterladen
ollama.com/download → Windows-Installer (.exe) herunterladen und ausführen. Ollama installiert sich als Hintergrunddienst (kein manuelles Starten nötig).
2
Installation prüfen (PowerShell) 
ollama --version
# → ollama version 0.x.x
3
Qwen2.5-Coder herunterladen 
ollama pull qwen2.5-coder:7b-instruct-q4_K_M
# ~4,5 GB Download — 5–15 Min je nach Verbindung
# Ollama weist GPU/VRAM automatisch zu (CUDA wird erkannt)
4
Modell starten & testen 
ollama run qwen2.5-coder:7b-instruct-q4_K_M
# Interaktiver Chat im Terminal
# >>> Wie implementiere ich ein Object Pooling System in Unity?
Erster Test: Frag das Modell nach einem Unity MonoBehaviour oder einer s&box Component. Wenn es flüssig antwortet (sichtbar tippend, kein langes Warten), läuft alles korrekt auf der GPU.

BefehlFunktion

Empfohlene Modelle für das komplette Setup

# Primär: Coding (C#, Unity, s&box)
ollama pull qwen2.5-coder:7b-instruct-q4_K_M

# Sekundär: Reasoning, Design-Diskussionen
ollama pull llama3.1:8b-instruct-q4_K_M

# Optional: Schnelle Quick-Lookups
ollama pull phi3.5:mini-instruct-q4_K_M

# Für RAG (Lektion 6): Embedding-Modell
ollama pull nomic-embed-text

4. Die REST-API

Sobald Ollama läuft, ist automatisch eine REST-API auf http://localhost:11434 verfügbar. Das ist die Grundlage für IDE-Integration (Lektion 4), eigene Skripte und RAG (Lektion 6).

EndpunktMethodeFunktion
/api/chatPOSTChat mit Verlauf (OpenAI-kompatibel)
/api/generatePOSTEinzel-Completion, Streaming möglich
/api/embeddingsPOSTText → Embedding-Vektor (für RAG)
/api/tagsGETAlle lokal verfügbaren Modelle
/v1/chat/completionsPOSTOpenAI-kompatibles Format (für externe Tools)

Schnelltest mit PowerShell

Invoke-RestMethod -Uri "http://localhost:11434/api/generate" `
  -Method POST -ContentType "application/json" `
  -Body '{"model":"qwen2.5-coder:7b-instruct-q4_K_M",
          "prompt":"Erkläre async/await in C# in einem Satz.",
          "stream":false}'
OpenAI-kompatible API: Ollama unterstützt /v1/chat/completions — exakt das Format der OpenAI-API. Damit funktionieren alle Tools, die ursprünglich für OpenAI gebaut wurden, direkt mit dem lokalen Modell. Das wird in Lektion 4 (Continue.dev) wichtig.

5. Modelfiles — Modell für Unity/s&box anpassen

Ein Modelfile konfiguriert ein bestehendes Modell mit einem System-Prompt und Parametern. Einmal erstellt, erscheint es als eigenes Modell in Ollama.

# Datei: Modelfile-unity (irgendwo auf deinem System speichern)
FROM qwen2.5-coder:7b-instruct-q4_K_M

SYSTEM """
Du bist ein erfahrener C#-Entwickler spezialisiert auf Unity und die s&box Engine.
Antworte immer auf Deutsch. Nutze modernes C# (C# 9+).
Bei Unity: bevorzuge den neuen Input System und Unity Events statt alter Alternativen.
Bei s&box: nutze das GameResource-System und Component-basierte Architektur.
Wenn du Code schreibst, füge kurze Kommentare zu nicht-offensichtlichen Stellen hinzu.
"""

PARAMETER temperature 0.3
PARAMETER num_ctx 8192
# Modell aus Modelfile erstellen
ollama create unity-assistant -f Modelfile-unity

# Starten
ollama run unity-assistant
Tipp: temperature 0.3 macht Code-Ausgaben deterministischer und präziser. Für kreatives Brainstorming (Game-Design-Ideen) besser 0.7–0.9 verwenden.

Quiz

1. Du willst sehen, wie viel VRAM dein geladenes Modell aktuell belegt. Welcher Befehl?

A) ollama list
B) ollama ps
C) ollama show qwen2.5-coder:7b
ollama ps zeigt aktuell geladene Modelle mit VRAM-Nutzung in Echtzeit. ollama list zeigt alle installierten Modelle (auch wenn gerade nicht geladen). ollama show zeigt statische Modell-Metadaten.

2. Auf welchem Port läuft die Ollama REST-API standardmäßig?

A) 8080
B) 11434
C) 3000
http://localhost:11434 ist der Standard-Port von Ollama. Port 3000 ist Open-WebUI (Lektion 3). Port 8080 ist ein generischer HTTP-Port ohne spezielle Bedeutung hier.

3. Was bewirkt PARAMETER temperature 0.3 im Modelfile?

A) Begrenzt Antwortlänge auf 30% des Kontexts
B) Macht Ausgaben deterministischer — weniger zufällig, besser für Code-Generierung
C) Setzt GPU-Temperaturgrenze in Celsius
Temperature steuert die "Zufälligkeit" der Ausgaben. 0.0 = immer dieselbe Antwort. 1.0 = sehr variabel/kreativ. Für Code: 0.1–0.4 ideal (präzise, reproduzierbar). Für Game-Design-Ideen: 0.6–0.9 besser.

4. Wozu dient der /api/embeddings Endpunkt (wird in Lektion 6 wichtig)?

A) Lädt eingebettete Systemdateien
B) Zeigt die Modell-Gewichte als JSON
C) Wandelt Text in einen numerischen Vektor um — Basis für semantische Suche in RAG
Embeddings sind numerische Repräsentationen von Text. Ähnliche Texte → ähnliche Vektoren → semantische Suche möglich. Das ist die Grundlage des RAG-Systems: statt Texte nach Stichwörtern zu durchsuchen, sucht man nach Bedeutung.
