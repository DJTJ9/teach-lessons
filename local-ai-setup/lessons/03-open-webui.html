Open-WebUI — Browser-Interface für lokale Modelle

ChatGPT-ähnliche Oberfläche lokal betreiben — kein Cloud-Zugriff, volle Kontrolle

1. Warum Open-WebUI?

Ollama im Terminal ist mächtig, aber für längere Gespräche und tägliche Nutzung unpraktisch. Open-WebUI gibt eine vollwertige Chat-Oberfläche im Browser — mit Gesprächsverlauf, Datei-Upload und mehr.

FeatureOllama TerminalOpen-WebUI
GesprächsverlaufNur aktuelle SessionDauerhaft gespeichert
Modell wechselnNeues Terminal nötigDropdown im Chat
Dateien hochladenNicht möglichPDFs, .cs-Dateien direkt
System-PromptsNur via ModelfilePer Chat konfigurierbar
Mehrere NutzerNeinMulti-User mit Logins

2. Installation — zwei Wege

Weg A: Python pip (schnellster Einstieg)

# Python 3.11+ vorausgesetzt
pip install open-webui
open-webui serve
# → http://localhost:8080

Weg B: Docker (empfohlen für Dauerbetrieb)

docker run -d -p 3000:8080 `
  --add-host=host.docker.internal:host-gateway `
  -v open-webui:/app/backend/data `
  --name open-webui --restart always `
  ghcr.io/open-webui/open-webui:main
# → http://localhost:3000
# --add-host: wichtig auf Windows, damit Docker Ollama auf dem Host erreicht
# --restart always: startet automatisch nach Windows-Neustart
Empfehlung: Weg A (pip) zum schnellen Ausprobieren. Weg B (Docker) für Dauerbetrieb — startet automatisch mit Windows und läuft im Hintergrund.

6. Dateien in den Chat laden — C#-Code-Review

Eines der nützlichsten Features: eigene .cs Dateien direkt in den Chat hochladen und Code-Review oder Erklärungen anfordern.

Workflow: .cs Datei per 📎 hochladen → "Erkläre diesen Code" oder "Wo sind mögliche Memory Leaks?" → Modell liest den gesamten Dateiinhalt und antwortet gezielt.
Ressourcen: Open-WebUI selbst braucht ~300–500 MB RAM. Das Modell läuft weiterhin in Ollama getrennt. Ollama muss laufen, damit Open-WebUI Modelle nutzen kann.

Quiz

1. Warum ist --add-host=host.docker.internal:host-gateway bei der Docker-Installation auf Windows wichtig?

A) Es öffnet Ollama-Port 11434 ins Internet
B) Innerhalb Docker ist localhost der Container selbst — dieses Flag erstellt einen Alias für den Windows-Host, damit Open-WebUI Ollama erreichen kann
C) Es aktiviert GPU-Unterstützung für den Container
In Docker zeigt localhost auf den Container, nicht den Windows-Host. Ohne dieses Flag kann Open-WebUI das Ollama auf dem Host (Port 11434) nicht finden. Das Flag erstellt den DNS-Namen host.docker.internal, der auf den Host-Rechner zeigt.

2. Ein "Modell" in Open-WebUI (Workspace → Models) ist dasselbe wie…?

A) Neue KI-Gewichte, eigenständig trainiert
B) Funktional ähnlich wie ein Ollama Modelfile — System-Prompt + Parameter über einem Basis-Modell, aber in der UI-Datenbank gespeichert statt als Datei
C) Ein OpenAI-API-Profil, das in der Cloud läuft
Beide — Open-WebUI Modell-Presets und Ollama Modelfiles — sind Konfigurationsebenen über echten Modell-Gewichten. Sie ändern nichts an den Gewichten, nur System-Prompt, Temperature und Context-Länge. Welches man nutzt, ist Geschmackssache.

3. Was ist der Vorteil von --restart always im Docker-Befehl?

A) Startet das Modell bei jeder Anfrage neu (frischer Cache)
B) Open-WebUI startet automatisch nach Windows-Neustart — kein manuelles Starten nötig
C) Aktualisiert Open-WebUI automatisch auf neue Versionen
--restart always bedeutet: Docker startet diesen Container automatisch wenn Docker Desktop startet — und Docker Desktop startet automatisch mit Windows. Effekt: Open-WebUI läuft immer im Hintergrund, ohne Aktion des Nutzers.
