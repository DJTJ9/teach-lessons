Hugging Face & GGUF — Weitere Modelle finden

Hugging Face navigieren, GGUF-Dateien verstehen, beliebige Modelle in Ollama laden

1. Was ist Hugging Face?

Hugging Face (huggingface.co) ist das GitHub der KI-Welt — eine Plattform mit Hunderttausenden Modellen, Datensätzen und Tools. Praktisch jedes relevante LLM wird dort veröffentlicht. Für lokale KI ist es die wichtigste Quelle für neue Modelle.

Wichtigste Sektion: Models → Filter: Text Generation + GGUF. GGUF ist das Format, das Ollama direkt einlesen kann — quantisierte Modelle, optimiert für lokale Inferenz.

2. GGUF-Dateinamen lesen

│              │  │         │
│              │  │         └─ Quantisierungs-Format
│              │  └─────────── Instruct = Chat/Anweisungen feinabgestimmt
│              └────────────── Parameteranzahl (7 Milliarden)
└───────────────────────────── Modell-Familie
SuffixBedeutung
-InstructFür Chat/Anweisungen trainiert — das willst du für interaktive Nutzung
-BaseNur Pretrained — kein Chat, nur Text-Vervollständigung
-ChatWie Instruct, ältere Bezeichnung
-CodeAuf Code spezialisiert (ohne Instruct: Base-Modell)

4. Modell von Hugging Face in Ollama laden

Methode A: Direkt via Ollama (einfachste)

# Schema: ollama pull hf.co/[nutzer]/[repo]:[quantisierung]
ollama pull hf.co/bartowski/Qwen2.5-Coder-7B-Instruct-GGUF:Q4_K_M

Methode B: Manuell herunterladen + Modelfile

# 1. Datei herunterladen (PowerShell)
Invoke-WebRequest `
  -Uri "https://huggingface.co/bartowski/Qwen2.5-Coder-7B-Instruct-GGUF/resolve/main/Qwen2.5-Coder-7B-Instruct-Q4_K_M.gguf" `
  -OutFile "C:\Models\qwen2.5-7b-q4.gguf"

# 2. Modelfile erstellen (Textdatei: C:\Models\Modelfile)
FROM C:\Models\qwen2.5-7b-q4.gguf
SYSTEM "Du bist ein C#-Experte für Unity und s&box."
PARAMETER temperature 0.3

# 3. In Ollama registrieren
ollama create mein-coder -f C:\Models\Modelfile
ollama run mein-coder

Methode C: huggingface-cli (für große Modelle mit Resume-Support)

pip install huggingface_hub
huggingface-cli download bartowski/Qwen2.5-Coder-7B-Instruct-GGUF `
  --include "*Q4_K_M*" `
  --local-dir C:\Models\
Empfehlung: Methode A für bekannte Modelle (Qwen, Llama, Mistral). Methode B/C für spezielle Modelle ohne Ollama-Eintrag oder bei langsamer Verbindung (CLI hat Resume-Support).

Quiz

1. Du siehst Qwen2.5-Coder-7B-Instruct-Q4_K_M.gguf. Was bedeutet "Instruct"?

A) Das Modell gibt nur Anweisungen, stellt keine Fragen
B) Nach dem Pretrain auf Instruction-Following feinabgestimmt — geeignet für interaktiven Chat und Anweisungen
C) Enthält Instruktions-Kommentare im Modell-Code
"Instruct"-Modelle wurden nach dem Pretraining mit RLHF oder DPO auf Instruction-Following feinabgestimmt. Sie erwarten Chat-Format-Eingaben. "Base"-Modelle sind nur pretrained — gut für Text-Completion, nicht für Dialog.

2. Du hast eine GGUF-Datei manuell heruntergeladen. Wie bringt man sie in Ollama?

A) ollama import datei.gguf
B) Modelfile mit FROM /pfad/datei.gguf erstellen → ollama create name -f Modelfile
C) GGUF-Datei in den Ollama-Installationsordner kopieren
Ollama hat keinen direkten import-Befehl für lokale Dateien. Weg: 1) Modelfile mit FROM /absoluter/pfad/zur/datei.gguf erstellen. 2) ollama create meinmodell -f Modelfile. Danach erscheint es in ollama list.

3. Warum sind HumanEval-Benchmarks für die C#/Unity-Entscheidung begrenzt nützlich?

A) HumanEval ist veraltet und wird nicht mehr gepflegt
B) HumanEval misst Python-Code-Completion — C# und Unity-spezifische Performance ist davon nicht direkt ableitbar
C) HumanEval funktioniert nur mit Base-Modellen, nicht mit Instruct
Benchmarks werden fast ausschließlich in Python evaluiert. Ein Modell das gut in Python ist, wird auch in C# gut sein — aber die relative Rangfolge kann sich verschieben. Eigene Tests mit typischen Unity-Aufgaben sind aussagekräftiger als reine Benchmark-Zahlen.
