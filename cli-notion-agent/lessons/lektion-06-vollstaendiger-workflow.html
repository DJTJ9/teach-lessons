Das Konzept: Agenten-Prompt

Ein guter Workflow-Prompt hat drei Teile:

  1. Quelle — Wo liegt der Task? (Notion-Datenbank, Filter)
  2. Aktion — Was soll der Agent tun? (lesen, verarbeiten, entscheiden)
  3. Rückschreiben — Was soll zurück nach Notion? (Status, Kommentar, Ergebnis)
Beispiel-Prompt (Vorlage)

„Lies alle Tasks aus meiner Notion-Datenbank 'Tasks' mit Status 'Not started'.
Für jeden Task: schreib den Task-Namen auf und setze den Status auf 'In progress'.
Zeig mir am Ende eine Liste aller Tasks, die du verändert hast."

Warum ein Prompt reicht

Wie Claude Code das intern zerlegt

Claude Code sieht den Prompt und plant automatisch die Tool-Kette:

1. notion_find("Tasks")           → data_source_id
2. notion_query_data_source(      → Liste aller "Not started"-Pages
     filter: Status = "Not started"
   )
3. für jede page_id:
   notion_update_page_properties( → Status = "In progress"
     page_id, Status: "In progress"
   )
4. Zusammenfassung ausgeben

Du denkst in Absichten, Claude Code denkt in Tool-Calls.

Filter in Queries

Statt alle Tasks zu laden, kannst du gezielt filtern. Beispiel: nur "Not started"-Tasks:

// notion_query_data_source mit filter:
{
  "data_source_id": "...",
  "filter": {
    "property": "Status",
    "status": {
      "equals": "Not started"
    }
  }
}
⚠ Vorsicht bei "für alle Tasks"

Wenn du sagst „setze alle Tasks auf Done" — macht Claude Code genau das. Kein Undo-Button. Nutze Filter und lass dir vor dem Schreiben eine Liste zeigen (zweistufiger Prompt).
