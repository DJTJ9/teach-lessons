Lektion 07 — Tasks in Notion anlegen

Kurs: CLI × Notion × Coding Agent  |  Heute: Neue Einträge erzeugen

🎯 Lernziel

Du kannst per Claude Code einen neuen Task in deiner Notion-Datenbank anlegen — Name und Status direkt beim Erstellen setzen.

Warum das die Mission abschließt

Bisher: Agent liest Tasks und schreibt Status zurück.
Mit diesem Schritt kann der Agent auch selbst Tasks erzeugen — z.B. Subtasks aus einem großen Task, oder Ergebnisse als neue Einträge festhalten.

Zwei Tools — welches wann?

notion_create_data_source_item_from_values

Einfachste Variante. Felder als flaches Key-Value-Objekt. Ideal für schnelles Anlegen mit bekannten Properties.

→ empfohlen für diesen Workflow

notion_create_data_source_item

Volle Notion-API-Syntax. Nötig für komplexe Property-Typen (Relations, Files, People). Mehr Tipparbeit.

→ nur wenn from_values nicht reicht

Neuen Task anlegen — from_values

Konzept

Du gibst data_source_id und ein values-Objekt mit Property-Namen als Keys an. Notion legt einen neuen Eintrag an und gibt die neue page_id zurück.

// Tool: notion_create_data_source_item_from_values
{
  "data_source_id": "37a4bba2-9c55-80e3-9102-000b4b86ec1d",
  "values": {
    "Name": "Mein neuer Task",
    "Status": "Not started"
  }
}
⚠ data_source_id ≠ database_id

Du brauchst die data_source_id (aus notion_find), nicht die database_id. Beides sieht ähnlich aus — falsche ID → Fehler.

Der komplette Ablauf

1. notion_find("Tasks")                    → data_source_id ermitteln
2. notion_create_data_source_item_from_values(
     data_source_id,
     values: { Name: "...", Status: "Not started" }
   )                                        → neue page_id zurück
3. Optional: sofort Status ändern           → notion_update_page_properties

🔬 Live-Aufgabe

Aufgabe

Sag Claude Code:

„Lege einen neuen Task in der Tasks-Datenbank an mit dem Namen 'Ersten echten Workflow bauen' und Status 'Not started'."

Prüfe danach in Notion Desktop — der neue Eintrag sollte erscheinen.

Bonus-Aufgabe

Wenn die erste Aufgabe klappt

Sag Claude Code:

„Lege 3 neue Tasks an: 'Notion API erkunden', 'Workflow dokumentieren', 'Workflow testen' — alle mit Status 'Not started'."

Claude Code führt 3 create-Calls automatisch nacheinander aus.
