Lektion 10 — CLAUDE.md als Workflow-Gedächtnis

Kurs: CLI × Notion × Coding Agent  |  Heute: Kontext einmal speichern, nie mehr erklären

🎯 Lernziel

Du speicherst deine Workflow-Prompts und Datenbank-IDs in einer CLAUDE.md-Datei — so kennt Claude Code deinen Organizer in jeder neuen Session ohne Einleitung.

Was ist CLAUDE.md?

Eine Markdown-Datei, die Claude Code beim Start automatisch liest. Alles darin ist persistenter Kontext — du musst es nie wiederholen. Ideal für:

  • Datenbank-IDs die sich nie ändern
  • Workflow-Prompt-Vorlagen zum Copy-Paste
  • Regeln ("immer Datum setzen", "Priorität Hoch = heute erledigen")

Vorher / Nachher

❌ Ohne CLAUDE.md

Jede Session: „Mein Tagesorganizer heißt X, die data_source_id ist Y, Morgen-Workflow macht Z..."

Claude Code sucht DB jedes Mal neu.

✅ Mit CLAUDE.md

Jede Session: „Morgen-Workflow starten."

Claude Code weiß sofort: welche DB, welche IDs, welcher Filter.

Deine CLAUDE.md — Vorlage

Speichern unter: C:\Users\tjark\.claude\CLAUDE.md (global, immer aktiv).

# Notion Tagesorganizer

## Datenbank
- Name: Tagesorganizer
- data_source_id: c9d2abbe-5607-44c2-bbf4-9aa673e0c4a0
- Eltern-Seite: Organizer (Workspace-Level)

## Properties
- Name (title)
- Status (status): Not started | In progress | Done
- Priorität (select): Hoch | Mittel | Niedrig
- Datum (date): ISO 8601, z.B. 2026-06-09
- Bereich (select): Arbeit | Privat | Lernen | Gesundheit
- Notiz (rich_text)

## Regeln
- Immer Datum setzen wenn Task angelegt wird
- Priorität Hoch = muss heute erledigt werden
- Notiz-Feld für Ergebnisse und Kommentare nutzen

## Morgen-Workflow
Lies den Tagesorganizer. Zeig alle Tasks mit Datum heute oder ohne Datum,
die nicht Done sind. Sortiere nach Priorität (Hoch zuerst). Gruppiere nach Bereich.

## Abend-Workflow
Lies den Tagesorganizer. Zeige:
1) Alle heute erledigten Tasks (Status = Done) — Erfolgsliste
2) Alle offenen Tasks (Status ≠ Done) — offene Punkte
Frage dann: Soll ich offene Tasks auf morgen verschieben?
💡 Warum die data_source_id reinschreiben?

Ohne ID muss Claude Code bei jedem Start notion_find aufrufen — kostet Zeit und einen Tool-Call. Mit ID geht es direkt los. Die ID ändert sich nie solange die DB existiert.

Wo speichern?

Zwei Optionen

Global C:\Users\tjark\.claude\CLAUDE.md — gilt für alle Projekte, immer aktiv.
Projekt-lokal CLAUDE.md im Projektordner — nur aktiv wenn du dort arbeitest.

Für den Tagesorganizer: global empfohlen — du willst ihn überall nutzen können.

🔬 Live-Aufgabe

Aufgabe

Sag Claude Code:

„Füge meiner globalen CLAUDE.md die Tagesorganizer-Konfiguration hinzu: data_source_id, Properties, Morgen- und Abend-Workflow als Vorlage."

Claude Code schreibt die CLAUDE.md — dann neue Session starten und einfach „Morgen-Workflow" tippen. Kein Kontext nötig.
