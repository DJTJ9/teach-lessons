Lektion 09 — Morgen- & Abend-Workflow

Kurs: CLI × Notion × Coding Agent  |  Heute: Wiederverwendbare Tagesplanung

🎯 Lernziel

Du hast zwei fertige Workflow-Prompts — einen für den Morgen, einen für den Abend — die du täglich nutzen kannst.

Das Prinzip: Prompt-Vorlagen

Ein Workflow-Prompt ist ein wiederverwendbarer Text, den du Claude Code gibst. Er kombiniert immer dieselben Bausteine: Filter + Aktion + Ausgabe. Je klarer der Prompt, desto weniger Rückfragen.

☀️ Morgen-Workflow

Ziel: Fokus setzen — was muss heute erledigt werden, in welcher Reihenfolge?

☀️ Morgen-Prompt (Vorlage)
„Lies meinen Tagesorganizer in Notion. Zeig mir alle Tasks mit Datum heute oder ohne Datum, die nicht 'Done' sind. Sortiere nach Priorität (Hoch zuerst). Gruppiere nach Bereich. Gib mir eine klare Liste für den Tag."

Variante mit Datum heute: füge „und Datum = [heutiges Datum]" hinzu wenn du nur datierte Tasks willst.

🌙 Abend-Workflow

Ziel: Tag abschließen — was ist erledigt, was bleibt, was kommt morgen?

🌙 Abend-Prompt (Vorlage)
„Lies meinen Tagesorganizer in Notion. Zeige mir: 1) Alle Tasks die heute 'Done' wurden — als kurze Erfolgsliste. 2) Alle Tasks die noch 'Not started' oder 'In progress' sind — als offene Punkte. Frage mich dann: Soll ich offene Tasks auf morgen verschieben?"

Warum diese Struktur funktioniert

Morgen

Filter: Status ≠ Done
Sort: Priorität desc
Group: Bereich
Output: Fokus-Liste

Abend

Filter 1: Status = Done
Filter 2: Status ≠ Done
Output: Rückblick + Vorschau
Aktion: Datum verschieben

⚠ Datum-Filter-Realität

Notion filtert Datum exakt — „heute" ist immer das konkrete Datum (z.B. 2026-06-09). Wenn du kein Datum setzt, erscheint der Task im Morgen-Workflow nur wenn du "oder ohne Datum" im Prompt hast. Empfehlung: Immer Datum setzen wenn du einen Task anlegst.
