Lektion 08 — Organizer-Datenbankstruktur designen

Kurs: CLI × Notion × Coding Agent  |  Heute: Das richtige Schema für tägliche Planung

🎯 Lernziel

Du entwirfst die Property-Struktur deines persönlichen Tagesorganizers — so, dass Claude Code damit sinnvoll filtern, sortieren und updaten kann.

Warum Schema zuerst?

Ein schlechtes Schema macht jeden Workflow kompliziert. Ein gutes Schema macht Prompts wie „Zeig mir alle dringenden Tasks von heute" trivial — weil Notion exakt die richtigen Filter-Felder hat.

Kernfrage

Was muss Claude Code über einen Task wissen, damit er nützlich handeln kann?
Nicht: was ist theoretisch interessant zu speichern.

Empfohlenes Schema: Tagesorganizer

Name title Was ist zu tun? MUSS
Status status Not started / In progress / Done — Workflow-Steuerung MUSS
Priorität select Hoch / Mittel / Niedrig — für „zeig mir was wichtig ist heute" MUSS
Datum date Fälligkeitsdatum — für Tagesfilter „alles für heute" MUSS
Bereich select Arbeit / Privat / Lernen / Gesundheit — Kontext für Fokusblöcke NICE
Notiz rich_text Kurze Zusatzinfo — was Claude Code als Ergebnis reinschreiben kann NICE
⚠ Weniger ist mehr

Jedes Property, das du nie filterst oder updatest, ist Lärm. Starte mit den 4 MUSS-Properties. Füge NICE-Properties erst hinzu wenn du sie brauchst.

Warum diese Properties mit Claude Code harmonieren

PropertyClaude Code kann damit...
Statusfiltern (nur "Not started"), updaten nach Erledigung
Prioritätsortieren, nur "Hoch" zeigen, Morgenplan priorisieren
Datumfiltern auf heute, überfällige Tasks finden
BereichFokusblöcke: "nur Arbeit-Tasks für heute"
NotizErgebnis oder Kommentar zurückschreiben

Bestehende DB erweitern oder neue anlegen?

Empfehlung

Neue Datenbank — "Tagesorganizer" — anlegen. Die bestehende "Tasks"-DB war für den Kurs. Der echte Organizer bekommt ein sauberes Schema von Anfang an.

Alternativ: "Tasks"-DB erweitern (schneller, aber Altlasten bleiben).

🔬 Live-Aufgabe — Entscheidung treffen

Aufgabe

Schritt 1: Entscheide dich — neue DB oder bestehende erweitern?

Option A (neu, empfohlen): Sag Claude Code:

„Lege in Notion eine neue Datenbank namens 'Tagesorganizer' an mit den Properties: Name (title), Status (status: Not started / In progress / Done), Priorität (select: Hoch / Mittel / Niedrig), Datum (date), Bereich (select: Arbeit / Privat / Lernen / Gesundheit), Notiz (rich_text)."

Option B (erweitern):

„Füge zur Tasks-Datenbank folgende Properties hinzu: Priorität (select: Hoch/Mittel/Niedrig), Datum (date), Bereich (select: Arbeit/Privat/Lernen/Gesundheit), Notiz (rich_text)."

Fragen
