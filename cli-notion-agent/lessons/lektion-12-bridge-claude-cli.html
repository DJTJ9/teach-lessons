Lektion 12 — Bridge: Telegram → Claude Code CLI

Kurs: CLI × Notion × Coding Agent  |  Heute: Bot-Nachricht ruft claude -p auf, Antwort zurück an Telegram

🎯 Lernziel

Du erweiterst bot.py so, dass jede Telegram-Nachricht an Claude Code weitergeleitet wird und die Antwort zurückkommt. Das ist die fertige Bridge — danach braucht nur noch der Prompt zu funktionieren.

Das Schlüsselkonzept: claude -p

Claude Code hat einen nicht-interaktiven Modus: claude -p "dein Prompt" führt den Prompt aus, gibt die Antwort auf stdout aus und beendet sich. Genau das brauchen wir.

# Im Terminal testen:
claude -p "Was ist 2+2?"
# Ausgabe: 4
Warum subprocess?

Python kann keine Claude-Code-Plugins direkt importieren. Aber es kann jeden CLI-Befehl als Kindprozess starten, Output einlesen und weiterverwenden. subprocess.run ist dafür gebaut.

bot.py — erweiterte Version

import os, subprocess, requests

TOKEN = os.environ["TELEGRAM_TOKEN"]
MY_CHAT_ID = 8896609541
BASE = f"https://api.telegram.org/bot{TOKEN}"
WORK_DIR = r"C:\Unity\Aktuelle Projekte\DartTrainingsApp"

def get_updates(offset=None):
    params = {"timeout": 30, "offset": offset}
    r = requests.get(f"{BASE}/getUpdates", params=params, timeout=35)
    return r.json().get("result", [])

def send_message(chat_id, text):
    requests.post(f"{BASE}/sendMessage", json={"chat_id": chat_id, "text": text})

def run_claude(prompt):
    result = subprocess.run(
        ["claude", "--dangerously-skip-permissions", "-p", prompt],
        capture_output=True,
        text=True,
        timeout=120,
        cwd=WORK_DIR
    )
    return result.stdout.strip() or result.stderr.strip() or "(keine Antwort)"

offset = None
print("Bridge läuft. Strg+C zum Beenden.")

while True:
    updates = get_updates(offset)
    for update in updates:
        offset = update["update_id"] + 1
        msg = update.get("message", {})
        chat_id = msg.get("chat", {}).get("id")
        text = msg.get("text", "").strip()

        if chat_id != MY_CHAT_ID or not text:
            continue

        send_message(chat_id, "⏳ Denke nach...")
        print(f"→ Claude: {text}")
        response = run_claude(text)
        print(f"← Antwort: {response[:80]}...")
        send_message(chat_id, response)
--dangerously-skip-permissions

Im nicht-interaktiven Modus kann Claude Code keine Bestätigungen einholen. Das Flag überspringt Permission-Prompts automatisch. Nur für vertrauenswürdige Prompts von deiner Chat-ID verwenden — daher ist der Chat-ID-Filter so wichtig.

cwd=WORK_DIR erklärt

Claude Code liest CLAUDE.md aus dem Arbeitsverzeichnis. Mit cwd zeigen wir auf den Projektordner — so kennt Claude beim Aufruf automatisch den Tagesorganizer, die data_source_id und alle Workflow-Regeln.
