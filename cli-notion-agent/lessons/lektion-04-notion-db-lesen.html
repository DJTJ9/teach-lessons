Lektion 4 – Notion-Datenbank lesen

Workspace: CLI × Notion × Agent  |  Datum: 2026-06-09

Konzept dieser Lektion

Heute eines: Claude Code eine Notion-Datenbank abfragen lassen und die Ergebnisse lesen. Kein Copy-Paste von IDs in APIs — Claude Code übernimmt den Abruf via MCP-Tool.

Warum der Flow so aussieht

Du (Prompt) → Claude Code → Notion MCP Server → Notion API → Daten zurück

Claude Code ruft nie direkt die Notion API auf. Stattdessen nutzt es MCP-Tools — der MCP-Server kümmert sich um Auth (Integration Token) und HTTP.

Die relevanten MCP-Tools (Notion)

Lesen
  • API-retrieve-a-database — Datenbankschema + Properties
  • API-query-data-source — Seiten in DB abfragen (mit Filter/Sort)
  • API-retrieve-a-page — eine einzelne Seite holen (per ID)
  • API-get-block-children — Inhaltsblöcke einer Seite
  • API-post-search — Volltext-Suche über alle zugänglichen Seiten

Für den Missions-Workflow brauchst du vor allem API-query-data-source: es gibt dir alle Tasks einer Datenbank zurück, optional gefiltert nach Status.

Schritt 1 – Datenbank-ID finden (Desktop-App)

Du nutzt die Notion Desktop-App — kein Browser-Link sichtbar. Zwei Wege:

Weg A — empfohlen: ohne ID

Kein Kopieren nötig. Claude Code sucht die Datenbank selbst per Namen.

Weg B — ID aus Desktop-App

Rechtsklick auf die Datenbank-Seite in der Sidebar → "Link kopieren" → in einen Texteditor einfügen → die ID ist der 32-Zeichen-Block am Ende der URL:

https://www.notion.so/meinWorkspace/abcdef1234567890abcdef1234567890?v=...

Schritt 2 – Claude Code prompton

Zwei Varianten:

Variante A — ohne ID (Desktop-App-freundlich)
Such in Notion nach einer Datenbank mit dem Namen "Tasks"
und zeig mir alle Einträge mit ihren Properties.
Variante B — mit ID
Frage meine Notion-Datenbank (ID: abc123...) ab
und zeig mir alle Tasks mit Status "In Progress".

Was Claude Code dann macht

  1. Bei Variante A: ruft API-post-search auf → findet die Datenbank-ID automatisch
  2. Ruft API-query-data-source auf → bekommt JSON mit allen Pages
  3. Fasst die Ergebnisse für dich lesbar zusammen

Du siehst das Ergebnis direkt im Terminal — kein Browser-Tab, kein Copy-Paste.

Hands-on Aufgabe

Aufgabe
  1. Öffne Notion Desktop und erstelle eine kleine Test-Datenbank (2-3 Einträge, z.B. "Aufgabe", "Status")
  2. Tippe in Claude Code: 
    Such in Notion nach einer Datenbank namens "[dein DB-Name]"
und zeig mir alle Einträge mit ihren Properties.
  3. Berichte: Was kommt zurück? Welche Properties siehst du?

Quiz

1. Claude Code ruft die Notion API direkt auf. Richtig oder falsch?

Antwort Falsch. Claude Code nutzt MCP-Tools. Der MCP-Server ruft die Notion API auf.

2. Welches MCP-Tool brauchst du, um alle Seiten einer Datenbank abzufragen?

Antwort API-query-data-source

3. Wo findest du die Datenbank-ID?

Antwort Im Browser-Link, zwischen dem letzten / und dem ?v=.
