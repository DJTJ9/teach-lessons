1 Was bauen wir, und warum?

Der Bot kennt bisher zwei Befehle: task: … (Task anlegen) und freien Text (Claude-Frage). Jetzt kommen drei neue Workflow-Befehle hinzu:

📅

woche

Alle Tasks der letzten 7 Tage — was wurde erledigt? Was blieb offen?

🎯

fokus: Bereich

Nur Tasks aus einem Bereich (Arbeit / Privat / Lernen / Gesundheit) für konzentriertes Arbeiten.

📆

verschieben: Datum

Alle offenen Tasks auf ein neues Datum setzen — z.B. morgen oder einen konkreten Tag.

Das Muster dahinter

Alle drei Befehle folgen demselben Aufbau wie task::

1
Befehl erkennen — z.B. text.lower().startswith("woche")
2
System-Prompt wählen — gibt Claude klare Anweisungen was zu tun ist
3
run_claude() aufrufen — Claude nutzt Notion-MCP-Tools intern
4
Antwort senden — fertig formatierter Text zurück an Telegram
Warum System-Prompt statt direktem Python-API-Call? Claude hat via MCP direkten Zugriff auf Notion. System-Prompts sind einfacher zu schreiben als eigener API-Code — und Claude kann mehrere Schritte ausführen (lesen + schreiben + zusammenfassen).

2 Befehl: woche

Was macht dieser Befehl?

Telegram-Befehl: woche
Bot fragt Claude: "Zeig mir alle Tasks aus den letzten 7 Tagen, gruppiert nach Done / Offen, dann nach Bereich."

Beispiel-Ausgabe

woche
📅 Wochenrückblick (03.06 – 09.06) ✅ Erledigt (5): · Arbeit · Hoch · Sprint-Review vorbereiten · Lernen · Mittel · Notion API Lektion durchgehen · Privat · Niedrig · Wohnung aufräumen ⏳ Offen (3): · Arbeit · Hoch · PR reviewen · Lernen · Mittel · Dart-Tests schreiben

Der System-Prompt

WOCHE_SYSTEM_PROMPT = """Du bist ein Notion-Wochenassistent.
Lies den Tagesorganizer (data_source_id: c9d2abbe-5607-44c2-bbf4-9aa673e0c4a0).
Zeige alle Tasks der letzten 7 Tage (inkl. heute).
Format:
1) Erste Zeile: "📅 Wochenrückblick (DD.MM – DD.MM)"
2) Abschnitt "✅ Erledigt (N):" — Tasks mit Status Done
   je Zeile: · Bereich · Priorität · Name
3) Abschnitt "⏳ Offen (N):" — Tasks Not started / In progress
   je Zeile: · Bereich · Priorität · Name
Sortiere Offen nach Priorität (Hoch zuerst). Kein Markdown."""

Warum "letzte 7 Tage" statt "diese Woche"?

"Diese Woche" ist relativ (beginnt Montag? Sonntag?). Letzte 7 Tage ist absolut — Claude berechnet das aus dem heutigen Datum, das wir im Prompt übergeben.

Trick: Wir übergeben immer f"Heute ist {today}." als Präfix. Claude rechnet dann selbst aus, welche Daten in den Bereich fallen.

Code

if text.lower() == "woche":
    prompt = f"Heute ist {today}."
    response = run_claude(prompt, system_prompt=WOCHE_SYSTEM_PROMPT)

Der System-Prompt

FOKUS_SYSTEM_PROMPT = """Du bist ein Notion-Fokusassistent.
Lies den Tagesorganizer (data_source_id: c9d2abbe-5607-44c2-bbf4-9aa673e0c4a0).
Zeige alle Tasks mit dem genannten Bereich, Datum heute,
Status Not started oder In progress. Sortiere nach Priorität (Hoch zuerst).
Format pro Task: · [Priorität] [Name] — [Notiz falls vorhanden]
Erste Zeile: "🎯 Fokus: [Bereich] – [N] Tasks heute"
Falls keine Tasks: "🎯 Fokus: [Bereich] – nichts geplant für heute."
Kein Markdown."""

Bereich extrahieren und validieren

BEREICHE = {"arbeit", "privat", "lernen", "gesundheit"}

elif text.lower().startswith("fokus:"):
    bereich_raw = text[6:].strip()        # "Arbeit" aus "fokus: Arbeit"
    bereich = bereich_raw.capitalize()    # "arbeit" → "Arbeit"
    if bereich.lower() not in BEREICHE:
        response = (f"Unbekannter Bereich: {bereich_raw}\n"
                    "Gültig: Arbeit, Privat, Lernen, Gesundheit")
    else:
        prompt = f"Heute ist {today}. Bereich: {bereich}"
        response = run_claude(prompt, system_prompt=FOKUS_SYSTEM_PROMPT)

Warum .capitalize()?

Nutzer tippen oft Kleinbuchstaben: fokus: arbeit. Notion erwartet "Arbeit" (wie in der Select-Property definiert). .capitalize() macht den ersten Buchstaben groß.

Der System-Prompt

VERSCHIEBEN_SYSTEM_PROMPT = """Du bist ein Notion-Planungsassistent.
Lies den Tagesorganizer (data_source_id: c9d2abbe-5607-44c2-bbf4-9aa673e0c4a0).
Finde alle Tasks mit Status Not started oder In progress.
Setze ihr Datum-Feld auf das vom Nutzer angegebene Datum.
"morgen" = heute + 1 Tag. Wochentage relativ zu heute berechnen.
Antworte NUR mit: "📆 N Tasks verschoben auf [Datum]."
Falls keine offenen Tasks: "Keine offenen Tasks zum Verschieben."
Kein Markdown."""

Code

elif text.lower().startswith("verschieben:"):
    ziel_datum = text[12:].strip()   # "morgen" / "2026-06-15" / "Montag"
    if not ziel_datum:
        response = "Nutzung: verschieben: morgen  oder  verschieben: 2026-06-15"
    else:
        prompt = f"Heute ist {today}. Zieldatum: {ziel_datum}"
        response = run_claude(prompt, system_prompt=VERSCHIEBEN_SYSTEM_PROMPT)

Was Claude intern macht

1
notion_query_data_source — alle Tasks mit Status ≠ Done lesen
2
Datum berechnen — "morgen" = today + 1 Tag als ISO-8601-String
3
notion_update_page_properties — für jeden Task das Datum-Feld setzen
4
Bestätigung — "📆 3 Tasks verschoben auf 2026-06-10."
Hinweis: Claude führt mehrere MCP-Calls nacheinander aus. Bei vielen Tasks kann das 30–60 Sekunden dauern. Die "⏳ Denke nach…"-Nachricht ist korrekt — kein Bug.

Reihenfolge der elif-Kette

Python prüft elif-Bedingungen von oben nach unten und nimmt den ersten Treffer.

1. task: (spezifischster Präfix)
2. woche (exakter Match)
3. fokus: (Präfix)
4. verschieben: (Präfix)
5. else — freier Text an Claude

6 Quiz

Überprüfe dein Verständnis

1. Warum übergeben wir immer f"Heute ist {today}." an den Prompt?

Claude hat kein eingebautes aktuelles Datum — wir müssen es explizit angeben
Telegram verlangt ein Datum in jedem Prompt
Notion-MCP kann ohne Datum keine Abfragen machen

2. "arbeit".capitalize() ergibt:

ARBEIT
Arbeit
arbeit (unverändert)

3. Warum kommt else: response = run_claude(text) ans Ende?

Python-Konvention — else muss immer zuletzt stehen
Damit alle spezifischen Befehle zuerst geprüft werden und unbekannter Text als freie Frage an Claude geht
run_claude() ist langsamer als die anderen Branches

4. Wer berechnet das konkrete Datum bei verschieben: Montag?

Python mit datetime.weekday()
Notion-MCP intern
Claude — wir geben "Heute ist …" + "Zieldatum: Montag", Claude rechnet das ISO-Datum selbst aus
