Wie Telegram-Bots funktionieren

Jeder Bot ist ein Telegram-Account mit einem API-Token. Über die Telegram Bot API kann ein Script diesen Account steuern — Nachrichten lesen, antworten, Buttons senden.

Polling vs. Webhook

Polling: Dein Script fragt alle paar Sekunden „Gibt es neue Nachrichten?" Läuft auf jedem PC, kein Server nötig. Genug für unseren Use-Case.

Webhook: Telegram schickt Nachrichten aktiv an eine öffentliche URL. Braucht einen Server (VPS / ngrok). Für 24/7-Betrieb später sinnvoll.

Schritt 1 — Bot via BotFather erstellen

  1. Telegram öffnen → nach @BotFather suchen → Chat starten
  2. /newbot schicken
  3. Name eingeben (z.B. Tjark Organizer)
  4. Username eingeben — muss auf bot enden (z.B. tjark_organizer_bot)
  5. BotFather gibt dir einen Token: 123456789:AAEh... — sofort sichern!
⚠️ Token-Sicherheit

Der Token ist wie ein Passwort. Wer ihn hat, kontrolliert den Bot. Niemals in Git committen. In einer .env-Datei oder als Umgebungsvariable speichern.

Schritt 2 — Chat-ID herausfinden

Du brauchst deine Chat-ID, damit der Bot nur auf deine Nachrichten reagiert. So bekommst du sie:

  1. Deinen neuen Bot in Telegram suchen → /start schicken
  2. Im Browser öffnen: https://api.telegram.org/bot<TOKEN>/getUpdates
  3. Im JSON nach "chat": {"id": ...} suchen — das ist deine numerische Chat-ID
Warum Chat-ID filtern?

Ohne Filter antwortet der Bot auf Nachrichten von jedem der ihn findet. Mit Chat-ID-Check reagiert er nur auf deine Nachrichten — wichtig wenn er später Claude Code-Befehle ausführt.

Schritt 3 — Erstes Polling-Script

Minimales Python-Script — empfängt Nachrichten und gibt sie aus. Kein Framework, nur requests.

import requests, time

TOKEN = "DEIN_TOKEN_HIER"
MY_CHAT_ID = 123456789  # deine Chat-ID eintragen
BASE = f"https://api.telegram.org/bot{TOKEN}"

def get_updates(offset=None):
    params = {"timeout": 30, "offset": offset}
    r = requests.get(f"{BASE}/getUpdates", params=params, timeout=35)
    return r.json().get("result", [])

def send_message(chat_id, text):
    requests.post(f"{BASE}/sendMessage", json={"chat_id": chat_id, "text": text})

offset = None
print("Bot läuft. Strg+C zum Beenden.")

while True:
    updates = get_updates(offset)
    for update in updates:
        offset = update["update_id"] + 1
        msg = update.get("message", {})
        chat_id = msg.get("chat", {}).get("id")
        text = msg.get("text", "")

        if chat_id != MY_CHAT_ID:
            continue  # ignoriere fremde Nachrichten

        print(f"Nachricht: {text}")
        send_message(chat_id, f"Empfangen: {text}")
Long Polling erklärt

timeout=30 — Telegram hält die HTTP-Verbindung bis zu 30 Sekunden offen, falls keine Nachricht kommt. Effizienter als jede Sekunde eine neue Anfrage abschicken. offset markiert verarbeitete Updates damit sie nicht zweimal ankommen.
