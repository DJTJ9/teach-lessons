Lektion 11: /dev vision + Continuation Prompts

Neue Phase im Lifecycle: Feature-Triage mit Jetzt/Später-Gate + explizite Weiter-Prompts

Das Problem

Vor dieser Lektion gab es eine Lücke zwischen /dev idea (kein Dialog, schnelle Erfassung) und /dev discuss (voller Spec-Flow). Wenn eine neue Idee aufkam und du entscheiden wolltest ob und wann sie umgesetzt werden soll — gab es keinen Befehl dafür.

Außerdem endete jede Phase abrupt. Du musstest selbst wissen, was als nächstes kommt.

Die Lösung: /dev vision

Design-Rationale: /dev vision ist der Triage-Schritt — schneller als Discuss, aber mit echtem Dialog. Ziel: Feature verstehen, Abhängigkeiten klären, Platz in der Roadmap finden.

Aktualisierter Lifecycle:

/dev vision
→
/dev discuss
→
/dev plan
→
/dev implement
→
/dev finish

Der vision-Flow

1. Feature-Beschreibung
Auswahl aus [idea]-Items oder neue Beschreibung
2. Klärungsfragen (2–3)
Kernproblem, Abhängigkeiten, Scope-Grenze
3. "Wann umsetzen?"
A) Jetzt → STATUS.md active → direkt zu /dev discuss
B) Später → Mini-Spec + Roadmap-Platzierung → Commit

Pfad A: Jetzt

STATUS.md bekommt Active + Phase: discuss. Roadmap-Eintrag wird als [vision] markiert. Dann startet /dev discuss sofort — kein zweiter Befehl nötig.

Pfad B: Später

Eine Mini-Spec (max. 1 Seite) wird geschrieben: Idee, Abhängigkeiten, Scope-Grenze, Roadmap-Position. Eintrag in Roadmap als [vision] mit Priorität und "nach Feature X".

Roadmap-Status-Tags

[idea] Schnell erfasst   [vision] Besprochen, triage-reif   [discussed] Spec vorhanden   [done] Implementiert

Continuation Prompts

Jede Phase endet jetzt mit einem expliziten Weiter-Prompt:

## Nächster Schritt

A) Mit `/dev [NEXT]` weitermachen
B) Fortschritt speichern und beenden
init / vision / discuss
→ vision / discuss / plan
plan
→ implement
implement (Sonderfall)
→ Teach Lessons oder /dev finish
Warum Continuation statt Auto-Trigger? Auto-Trigger springt ohne Rückfrage weiter — schwer zu debuggen. Explizite Prompts geben Kontrolle. Gleiches Ergebnis, aber transparent.

Quiz

1. Du hast eine neue Feature-Idee und weißt noch nicht, wann du sie umsetzen willst. Welcher Befehl ist richtig?

  • /dev idea "text"
  • /dev vision
  • /dev discuss
  • /dev plan

2. Was passiert bei /dev vision Pfad A ("Jetzt")?

  • Eine Mini-Spec wird geschrieben
  • STATUS.md → Phase: discuss, /dev discuss startet sofort
  • Das Feature kommt als [idea] in die Roadmap
  • Der Plan wird direkt geschrieben

Zusammenfassung

  • /dev vision füllt die Lücke zwischen /dev idea und /dev discuss
  • Jetzt-Pfad → sofort zu /dev discuss. Später-Pfad → Mini-Spec + Roadmap
  • Tags: [idea] → [vision] → [discussed] → [planned] → [done]
  • Continuation Prompts ersetzen Auto-Trigger — explizit und kontrollierbar
