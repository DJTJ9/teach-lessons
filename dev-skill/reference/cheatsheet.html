Alle Befehle

/dev-initsetup
Neues Projekt einrichten. Einmalig pro Projekt. Fragt Name + Slug, schreibt alle Basis-Dateien im Hub.
Erzeugt: VISION.md, STATUS.md, DECISIONS.md, LEARNINGS.md, TIMELINE.md + Registry
Pfad: $HUB_DIR/topics/<slug>/
/dev-idea "text"capture
Idee sofort sichern — kein Dialog, läuft in Sekunden. Landet als [idea] in STATUS.md + VISION.md.
Argument: Text in Anführungszeichen
Commit: chore(<slug>): capture idea
/devstatus
Zeigt aktives Feature, Phase, nächsten Roadmap-Punkt. Bietet direkten Weiter-Link zur aktuellen Phase. Alias: /dev-status.
Liest: STATUS.md
/dev-discussphase 1
Feature-Design: Clarifying Questions → UI-Check → Ansätze → Sektionsweise Präsentation → Spec schreiben.
Erzeugt: specs/YYYY-MM-DD-<feature>-design.md
STATUS danach: Phase: plan
/dev-planphase 2
Implementierungsplan schreiben: File Map → TDD-Tasks (Test schreiben → FAIL → Impl → PASS → Commit). Kein Placeholder.
Erzeugt: plans/YYYY-MM-DD-<feature>.md
STATUS danach: Phase: implement
/dev-implementphase 3
Plan ausführen: Modus wählen (Subagent/Inline), Zeitplan eintragen, implementation_mode: true setzen.
Modus: Subagent-Driven (empfohlen) oder Inline
STATUS danach: Phase: finish
/dev-finishphase 4
Abschluss: VISION.md aktualisieren, Learnings + Timeline extrahieren, Hub pushen, nächstes Item zeigen.
Schreibt: VISION.md, LEARNINGS.md, TIMELINE.md, STATUS.md
Endet mit: git push auf Hub
/dev-refinecorrection
Bestehende Spec überarbeiten. Gezielter Dialog, Spec wird in-place aktualisiert. Phase zurück auf plan, Plan-Feld gelöscht.
Wann: Nach discuss, wenn Spec nicht stimmt
