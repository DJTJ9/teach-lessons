HTTP & APIs mit requests

GET/POST, JSON-Responses, Header — das Rückgrat von bot.py

GET und POST

C# HttpClient

var client = new HttpClient();
var resp = await client.GetAsync(url);
string body = await resp.Content
    .ReadAsStringAsync();

Python requests

import requests

resp = requests.get(url, timeout=30)
body = resp.text         # String
data = resp.json()       # dict/list
code = resp.status_code  # 200, 404...
# POST mit JSON-Body (setzt Content-Type: application/json automatisch):
resp = requests.post(url, json={"key": "value"}, timeout=30)

# Mit Headers (Auth-Token):
resp = requests.get(url, headers={"Authorization": f"Bearer {token}"}, timeout=30)
Immer timeout setzen — ohne Timeout blockiert der Call unbegrenzt. Kein Äquivalent zu C#s async hier — requests ist synchron (das ist gewollt, s.u.).

Fehlerbehandlung

try:
    resp = requests.get(url, timeout=10)
    resp.raise_for_status()  # wirft HTTPError bei 4xx/5xx
    data = resp.json()
except requests.exceptions.Timeout:
    print("Timeout")
except requests.exceptions.HTTPError as e:
    print(f"HTTP {e.response.status_code}")
except requests.exceptions.RequestException as e:
    print(f"Failed: {e}")

raise_for_status() = idiomatisches Pattern. Entspricht C#s response.EnsureSuccessStatusCode().

🤖 bot.py erklärt — Telegram-API-Calls

get_updates() — Long Polling:

BASE = f"https://api.telegram.org/bot{TOKEN}"

def get_updates(offset: int) -> list:
    try:
        resp = requests.get(
            f"{BASE}/getUpdates",
            params={"offset": offset, "timeout": 30},
            timeout=35,
        )
        return resp.json().get("result", [])
    except Exception:
        return []

Long Polling: Telegram hält die Verbindung bis zu 30s offen wenn keine Updates vorliegen. Deshalb timeout=35 (5s Puffer). params= URL-encoded automatisch.

send_message() — POST mit JSON:

def send_message(chat_id, text, reply_markup=None):
    payload = {"chat_id": chat_id, "text": text[:4096]}
    if reply_markup:
        payload["reply_markup"] = reply_markup
    try:
        requests.post(f"{BASE}/sendMessage", json=payload, timeout=30)
    except Exception as e:
        print(f"send_message failed: {e}")

text[:4096]: Telegram-Nachrichten haben 4096-Zeichen-Limit. Slicing kürzt automatisch.

Warum kein async? bot.py nutzt synchrones requests statt asyncio/aiohttp. Das reicht weil nur 1 User sendet und run_claude() sowieso blockiert (~5-30s). In Produktions-App mit 1000 gleichzeitigen Nutzern wäre async nötig.
