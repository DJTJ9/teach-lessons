Kontrollfluss & Collections

if/elif/else, Listen, Dicts, Sets, List Comprehensions — und der große Dispatcher in bot.py

if / elif / else

C#

if (x == 1) { ... }
else if (x == 2) { ... }
else { ... }

Python

if x == 1:
    ...
elif x == 2:
    ...
else:
    ...

Kein switch in Python — dafür elif-Ketten. Truthy/Falsy: None, 0, "", [], {} sind alle falsy — if wert: reicht statt if wert != null && wert != "".

Listen

befehle = ["moin", "abend", "hilfe"]
befehle.append("woche")     # Add()
befehle[0]                  # "moin"
befehle[-1]                 # "hilfe" — letztes Element
befehle[1:3]                # ["abend", "hilfe"] — Slicing
len(befehle)                 # 4
"moin" in befehle          # True — Contains()

Dictionaries

projekt = {
    "name": "Notion-Bot",
    "path": r"C:\Projekte\bot",
}
projekt["name"]               # "Notion-Bot"
projekt.get("version", "1.0") # Default wenn Key fehlt
projekt["autor"] = "Tjark"   # neuer Eintrag
"name" in projekt              # True — ContainsKey()
for key, val in projekt.items(): print(f"{key}: {val}")

Sets — Mengen ohne Duplikate

bereiche = {"arbeit", "privat", "lernen", "gesundheit"}
"arbeit" in bereiche   # True — O(1), wie HashSet<T>
"sport" in bereiche    # False
Sets haben O(1) Lookup — ideal wenn du nur "ist X enthalten?" fragst, ohne Reihenfolge oder Duplikate zu brauchen.

List Comprehensions — LINQ in einer Zeile

C# LINQ

var groß = befehle
    .Where(b => b.Length > 4)
    .Select(b => b.ToUpper())
    .ToList();

Python

groß = [
    b.upper()
    for b in befehle
    if len(b) > 4
]

🤖 bot.py erklärt — Dispatcher, BEREICHE, PROJECTS

BEREICHE — Set für schnellen Lookup:

BEREICHE = {"arbeit", "privat", "lernen", "gesundheit"}

if bereich.lower() not in BEREICHE:
    response = f"Unbekannter Bereich: {bereich_raw}"

Set statt Liste: Die Intention ist klarer ("das ist eine Menge gültiger Werte") und Lookup ist O(1).

PROJECTS — Dict als Konfigurations-Registry:

PROJECTS = {
    "notion": {"path": r"C:\Projekte\bot", "notion_name": "Notion-Bot"},
    "dart":   {"path": r"C:\Unity\DartApp",  "notion_name": "Dart-App"},
}
for name, info in PROJECTS.items():
    if text.lower().startswith(f"{name}:"):
        project_cwd = info["path"]
        break

Neues Projekt = ein Dict-Eintrag, kein Code-Change. Das ist das "Data drives behavior"-Pattern.

new_lessons = [
