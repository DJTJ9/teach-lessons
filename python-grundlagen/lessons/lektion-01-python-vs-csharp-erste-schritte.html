Python vs. C# — Erste Schritte

Wie Python läuft, wie man Code schreibt, und was if __name__ == "__main__" bedeutet

Kein Compiler — Python ist ein Interpreter

In C# schreibst du Code, kompilierst zu einer Binary, führst die Binary aus. In Python liest der Interpreter deinen Code direkt Zeile für Zeile — kein Build-Schritt.

C# — Build then Run

// Erst bauen, dann starten:
dotnet build
dotnet run

Python — direkt starten

# Kein Build-Schritt:
python hello.py

Kehrseite: Python prüft nicht vorab ob ein Typ stimmt oder eine Variable existiert. Fehler erscheinen erst zur Laufzeit — deshalb sind Tests besonders wichtig (Lektion 10 zeigt pytest).

Einrückung statt geschweifte Klammern

Python verwendet 4 Leerzeichen Einrückung um Blöcke zu definieren. Falsche Einrückung ist ein IndentationError.

C#

if (x > 0)
{
    Console.WriteLine("positiv");
}

Python

if x > 0:
    print("positiv")
# kein } — Einrückung beendet den Block
Kommentare: # für Einzelzeilen. Mehrzeilige Kommentare: einfach mehrere #-Zeilen. Es gibt kein /* */.

Erstes Skript schreiben und starten

# hello.py
name = "Claude"
print(f"Hallo, {name}!")
print(f"Typ von name: {type(name)}")

Terminal: python hello.py → Hallo, Claude! und Typ von name: <class 'str'>

Das f"..." ist ein f-String — wie $"..." in C#. Ausdrücke in {} werden ausgewertet.

Der REPL — Python interaktiv

Tippe python ohne Argumente. Jede Zeile wird sofort ausgeführt. Ideal zum schnellen Ausprobieren — in C# brauchst du dafür dotnet-script oder das Interactive Window.

$ python
>>> x = 5 + 3
>>> x
8
>>> type(x)
<class 'int'>
>>> quit()

🤖 bot.py erklärt — Zeilen 1–11 und das Ende

Import-Block (Zeile 1–3):

import os, re, subprocess, requests, tempfile, sys, json, uuid, threading, time
from datetime import date, datetime
from pathlib import Path
from groq import Groq

import X entspricht C#-using X. from X import Y importiert einzelne Klassen — wie using static X.Y. Stdlib-Module (os, json, threading) sind immer verfügbar; externe (requests, groq) müssen per pip install installiert werden.

Ende von bot.py — if __name__ == "__main__":

if __name__ == "__main__":
    threading.Thread(target=_reminder_loop, daemon=True).start()
    # ... Bot-Polling-Loop startet

Warum? Python setzt __name__ auf "__main__" wenn die Datei direkt ausgeführt wird (python bot.py), und auf den Modulnamen wenn sie importiert wird. Die Testdatei test_bot.py importiert bot.py um Konstanten zu testen — ohne diesen Guard würde der Bot-Loop beim Testen sofort starten. Das ist das Python-Äquivalent zu C#s static void Main(), aber flexibler: jede Datei kann gleichzeitig Bibliothek und ausführbares Skript sein.
