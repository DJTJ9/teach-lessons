LLM-APIs & Prompting

Groq-SDK, System-Prompts, Konversations-History — eigenen AI-Bot bauen

LLM-API aufrufen — das Grundmuster

Alle modernen LLM-APIs (Groq, OpenAI, Anthropic) teilen dasselbe Chat-Completion-Muster: Liste von Messages mit Rollen schicken, LLM antwortet mit neuer Message.

from groq import Groq

client = Groq(api_key="...")

response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[
        {"role": "system", "content": "Du bist ein hilfreicher Assistent."},
        {"role": "user",   "content": "Was ist ein Daemon-Thread?"},
    ],
    temperature=0.7,
    max_tokens=1000,
)

antwort = response.choices[0].message.content
Groq, OpenAI und viele andere nutzen identische API-Strukturen (OpenAI-kompatibel). Code für Groq läuft fast unverändert mit OpenAI-SDK — nur base_url und Model-Name ändern sich.

Message-Rollen und Konversations-History

history = []

def chat(client, system: str, user_msg: str) -> str:
    history.append({"role": "user", "content": user_msg})

    resp = client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=[{"role": "system", "content": system}] + history[-20:],
        max_tokens=2000,
    )

    reply = resp.choices[0].message.content.strip()
    history.append({"role": "assistant", "content": reply})
    return reply

history[-20:]: History auf letzte 20 Messages begrenzen — sonst steigen API-Kosten und Latenz unbegrenzt.

🤖 bot.py erklärt — groq_chat() vs run_claude()

groq_chat() für direkte LLM-Calls:

GROQ_CLIENT = Groq(api_key=GROQ_API_KEY)

def groq_chat(user_msg: str, system: str = "") -> str:
    messages = []
    if system:
        messages.append({"role": "system", "content": system})
    messages.append({"role": "user", "content": user_msg})
    try:
        resp = GROQ_CLIENT.chat.completions.create(
            model="llama-3.3-70b-versatile",
            messages=messages,
            max_tokens=2000,
        )
        return resp.choices[0].message.content.strip()
    except Exception as e:
        return f"❌ Groq-Fehler: {e}"

Wann was nutzen:

  • groq_chat(): schnelle LLM-Antworten ohne Tool-Zugriff (Übersetzung, Gespräch). ~0.3-2s
  • run_claude(): alles was Notion/Filesystem-Zugriff braucht (Tasks anlegen, Daten lesen). ~10-60s

Whisper-Transcription für Sprachnachrichten:

def transcribe_voice(audio_path: str) -> str:
    with open(audio_path, "rb") as f:
        result = GROQ_CLIENT.audio.transcriptions.create(
            model="whisper-large-v3-turbo",
            file=f,
            language="de",
        )
    return result.text

Sprachnachrichten → Datei herunterladen → transkribieren → normalisieren → gleicher Dispatcher wie Textnachrichten. Der Bot "hört" dieselbe if/elif-Logik.

🚀 Lernprojekt — CLI-Chatbot

Vollständiger CLI-Bot mit History in chatbot.py:

import os
from groq import Groq

client = Groq(api_key=os.environ["GROQ_API_KEY"])

SYSTEM = """Du bist ein Python-Tutor.
Antworte kurz, immer mit Code-Beispielen, Vergleich zu C#."""

history = []

def chat(msg: str) -> str:
    history.append({"role": "user", "content": msg})
    resp = client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=[{"role": "system", "content": SYSTEM}] + history[-20:],
        max_tokens=1000,
    )
    reply = resp.choices[0].message.content.strip()
    history.append({"role": "assistant", "content": reply})
    return reply

if __name__ == "__main__":
    print("Python-Tutor. Ctrl+C zum Beenden.\n")
    while True:
        msg = input("Du: ").strip()
        if msg == "/clear":
            history.clear(); print("History geleert.\n"); continue
        if not msg: continue
        print(f"\nBot: {chat(msg)}\n")

✏️ Abschluss-Aufgabe

  1. pip install groq, setze GROQ_API_KEY Umgebungsvariable, baue den CLI-Chatbot oben
  2. Erweitere: /save schreibt History als JSON in Datei (Lektion 7), /load lädt sie zurück
  3. Passe den System-Prompt an: Lass den Bot nur über Python und C# reden, alles andere ablehnen
