Variablen & Dynamic Typing

Warum Python keine Typdeklarationen braucht — und wann Type Hints trotzdem helfen

Dynamic Typing — der größte Unterschied zu C#

In C# bestimmst du den Typ bei der Deklaration — er ändert sich nie. In Python hat die Variable selbst keinen Typ, nur der Wert hat einen. Eine Variable ist nur ein Name, der auf ein Objekt zeigt.

C# — statisch typisiert

string name = "Claude";
// name = 42; → Compilerfehler!
int alter = 30;

Python — dynamisch typisiert

name = "Claude"
# name = 42 → kein Fehler
alter = 30

Das heißt nicht dass Python typlos ist. Typen existieren zur Laufzeit. type(42) gibt <class 'int'>. Fehler entstehen wenn du einen Wert falsch verwendest, z.B. "hallo" + 42 → TypeError.

Basistypen im Vergleich zu C#

# Zahlen
x = 42         # int — beliebig groß, kein int32/int64
y = 3.14       # float — 64-bit IEEE 754

# Text — immer Unicode, kein char[]
s = "hallo"

# Wahrheitswerte — Großschreibung beachten!
t = True        # nicht: true
f = False

# Nichts — Äquivalent zu null/nil
n = None        # nicht: null oder nil
Häufige Falle: true/false (kleingeschrieben) sind in Python undefinierte Variablen → NameError. Immer True/False schreiben.

Type Hints — optional, aber empfohlen

Seit Python 3.5 kannst du Typen annotieren. Python erzwingt sie nicht — aber IDEs und Tools wie mypy prüfen sie. In AI-Engineering-Code ist das heute Standard.

Ohne Hint

def begruesse(name):
    return f"Hallo, {name}!"

Mit Type Hints

def begruesse(name: str) -> str:
    return f"Hallo, {name}!"

Konstanten — Konvention statt Compiler

Python hat kein const. Konstanten werden per Konvention in GROSSBUCHSTABEN geschrieben. Der Interpreter hindert dich nicht an Änderungen — es ist eine Vereinbarung unter Entwicklern.

MAX_RETRIES = 3
BASE_URL = "https://api.example.com"
DEBUG = False

🤖 bot.py erklärt — Konstanten (Zeilen 6–17)

TOKEN = os.environ["TELEGRAM_TOKEN"]
GROQ_API_KEY = os.environ["GROQ_API_KEY"]
MY_CHAT_ID = 8896609541
BASE = f"https://api.telegram.org/bot{TOKEN}"
WORK_DIR = r"C:\Projekte\telegram-notion-bot"

GROSSBUCHSTABEN: alle Konstanten folgen der Python-Konvention — einmal beim Start gesetzt, dann nie geändert.

os.environ["KEY"]: liest Umgebungsvariable — wie C#s Environment.GetEnvironmentVariable("KEY"). Unterschied: wirft KeyError wenn nicht vorhanden (C# gibt null). Alternative: os.environ.get("KEY") gibt None zurück statt Fehler.

r"C:\Pfad": Raw-String — Backslashes werden nicht als Escape-Sequenzen interpretiert. Äquivalent zu C#s @"C:\Pfad". Ohne r wäre \t ein Tab-Zeichen.

Warum Secrets in Umgebungsvariablen? Tokens im Code sind ein Sicherheitsrisiko — ein versehentlicher Git-Commit macht sie öffentlich. Umgebungsvariablen werden nicht committed. Das ist Standard in AI Engineering und Cloud-Deployments.
