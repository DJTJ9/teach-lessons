Threading & Hintergrundprozesse

threading.Thread, daemon-Threads, subprocess — wie bot.py mehrere Dinge gleichzeitig tut

Thread starten

C# Thread

var t = new Thread(() => Work());
t.IsBackground = true;
t.Start();

Python threading

import threading

t = threading.Thread(
    target=work_func,
    daemon=True
)
t.start()

daemon=True: Thread stirbt automatisch wenn der Haupt-Thread endet — kein t.join() nötig. Ideal für Hintergrundschleifen. Wie C#s IsBackground = true.

# Mit Argumenten:
t = threading.Thread(target=send_reminder, args=(chat_id, text), daemon=True)
t.start()

# Auf Abschluss warten (für nicht-daemon):
t.join()  # blockiert bis t fertig ist

Python GIL — die wichtigste Threading-Besonderheit

Global Interpreter Lock (GIL): Python-Threads laufen nie wirklich parallel für CPU-intensive Arbeit. Der GIL erlaubt immer nur einem Thread gleichzeitig Python-Bytecode. Für I/O (sleep, requests, subprocess) gibt der GIL den Lock frei — deshalb funktioniert Threading für I/O-bound Tasks wie Bots gut. Für CPU-Parallelität: multiprocessing-Modul.

subprocess — externe Programme starten

import subprocess

result = subprocess.run(
    ["git", "status", "--porcelain"],
    capture_output=True,   # stdout/stderr einfangen
    text=True,             # als String, nicht bytes
    encoding="utf-8",
    timeout=30,
)
print(result.stdout)
print(result.returncode)   # 0 = Erfolg
Argumente als Liste ["git", "status"], nie als String "git status". Als String bräuchtest du shell=True — öffnet Command-Injection-Risiken (wie C# Process.Start("cmd", "/c " + userInput)).

🤖 bot.py erklärt — Reminder-Loop, Plan-Loop, parallele Verarbeitung

Drei gleichzeitige Aktivitäten beim Start:

if __name__ == "__main__":
    threading.Thread(target=_reminder_loop, daemon=True).start()
    threading.Thread(target=_plan_loop, daemon=True).start()
    polling_loop()  # Haupt-Thread blockiert hier

(1) Telegram-Polling im Haupt-Thread, (2) Erinnerungen prüfen, (3) geplante Plans prüfen. Alle laufen unabhängig.

_reminder_loop() — periodisch prüfen:

def _reminder_loop():
    while True:
        try:
            jetzt = datetime.now().strftime("%H:%M")
            check_reminders(jetzt)
        except Exception as e:
            print(f"reminder_loop error: {e}")
        time.sleep(60)  # jede Minute

Warum eigener Thread? Polling-Loop blockiert bis zu 35s auf Telegram. Im selben Thread kämen Erinnerungen bis zu 35s zu spät. Eigener Thread prüft exakt jede Minute.

run_claude() im eigenen Thread für lange Tasks:

threading.Thread(
    target=handle_claude_task,
    args=(text, chat_id),
    daemon=True
).start()

run_claude() kann 30-120s dauern. Im Haupt-Thread würde das den Polling-Loop blockieren. Eigener Thread lässt den Bot weiter neue Nachrichten empfangen während Claude arbeitet.
