Dateien, JSON & Pfade

open(), json-Modul, pathlib — und wie bot.py scheduled_plans.json verwaltet

Dateien lesen und schreiben

C#

// Lesen
string text = File.ReadAllText("file.txt");

// Schreiben
File.WriteAllText("file.txt", text);

Python

# Lesen
with open("file.txt", encoding="utf-8") as f:
    text = f.read()

# Schreiben
with open("file.txt", "w", encoding="utf-8") as f:
    f.write(text)

with-Statement schließt die Datei automatisch — wie C#s using (var f = ...). Modi: "r" (read, default), "w" (write, überschreibt), "a" (append). Immer encoding="utf-8" angeben — sonst plattformabhängig.

JSON — serialisieren und parsen

import json

# Python-Objekt → JSON-String
daten = {"name": "Sport", "prio": "Hoch", "done": False}
text = json.dumps(daten, ensure_ascii=False, indent=2)

# JSON-String → Python-Objekt
daten = json.loads(text)

# Direkt aus/in Datei:
with open("data.json", encoding="utf-8") as f:
    daten = json.load(f)

with open("data.json", "w", encoding="utf-8") as f:
    json.dump(daten, f, ensure_ascii=False, indent=2)
ensure_ascii=False: behält Umlaute (ä, ö, ü) als echte Zeichen. indent=2: lesbare Formatierung. C#-Äquivalent: JsonSerializer.Serialize/Deserialize aus System.Text.Json.

pathlib — Pfade als Objekte

from pathlib import Path

p = Path(r"C:\Projekte\bot")
p / "settings.json"   # C:\Projekte\bot\settings.json
p.exists()            # bool
p.is_file()           # bool
p.stem                # "bot" (ohne Extension)
p.suffix              # leer (ist ein Verzeichnis)
p.parent              # C:\Projekte

# Alle .json-Dateien rekursiv:
for f in Path("docs").glob("**/*.json"):
    print(f)

Path-Objekte sind plattformübergreifend — kein manuelles String-Konkatenieren mit \\. C#-Äquivalent: System.IO.Path und FileInfo.

🤖 bot.py erklärt — scheduled_plans.json und tempfile

scheduled_plans.json lesen:

PLANS_FILE = Path(WORK_DIR) / "scheduled_plans.json"

def load_plans() -> list:
    if not PLANS_FILE.exists():
        return []
    with open(PLANS_FILE, encoding="utf-8") as f:
        return json.load(f)

# Eintrag-Struktur:
# {"slug": "suche-command",
#  "plan_path": "docs/superpowers/plans/2026-06-12-suche-command.md",
#  "scheduled_time": "02:00",
#  "status": "pending"}

Path-Objekte direkt an open() übergeben — keine String-Konversion. Path(WORK_DIR) / "file.json" ist idiomatischer als os.path.join().

Status-Update schreiben:

def mark_plan_done(slug: str) -> None:
    plans = load_plans()
    for plan in plans:
        if plan["slug"] == slug:
            plan["status"] = "done"
    with open(PLANS_FILE, "w", encoding="utf-8") as f:
        json.dump(plans, f, ensure_ascii=False, indent=2)

Pattern: lesen → in-memory mutieren → zurückschreiben. Für kleine JSON-Dateien ausreichend.

Tempfile für große Claude-Übergaben:

import tempfile

with tempfile.NamedTemporaryFile(
    mode="w", suffix=".txt", delete=False, encoding="utf-8"
) as tmp:
    tmp.write(großer_text)
    tmp_path = tmp.name

# Aufräumen:
Path(tmp_path).unlink(missing_ok=True)

Kommandozeilenargumente haben eine Längenbeschränkung. bot.py schreibt lange Inhalte in Tempfiles und übergibt den Pfad an Claude. delete=False weil der Prozess die Datei noch nach dem with-Block lesen soll.
