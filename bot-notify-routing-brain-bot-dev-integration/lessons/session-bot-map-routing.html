SESSION_BOT_MAP: Multi-Bot Routing

Ziel: Verstehen, wie eine Map session_type → bot_name → TOKEN_KEY eleganter ist als if/elif-Ketten.

Das Problem

Die TelegramBotArmy hat 4 Bots. Wenn telegram_ask.py eine Frage stellt, muss sie wissen: an welchen Bot?

Die naive Lösung:

session = settings.get("active_session")
if session == "brainstorming":
    token = os.environ["TOKEN_BRAIN"]
elif session == "teach":
    token = os.environ["TOKEN_TEACH"]
# ... jedes Mal erweitern wenn neuer Session-Typ kommt
Problem: Jeder neue Bot oder Session-Typ erfordert Änderungen an mehreren Stellen. Fragil und fehleranfällig.

Die Lösung: SESSION_BOT_MAP

Kernkonzept: Eine Dict-Map trennt Routing-Logik von Ausführungslogik. Alle Routing-Entscheidungen stehen an einem Ort. 
# core/routing.py

SESSION_BOT_MAP = {
    "brainstorming": "brain",
    "vision":        "brain",
    "teach":         "teach",
    "organizer":     "organizer",
    "dev":           "brain",
}

def get_notify_token(settings: dict, env=None) -> str | None:
    if env is None:
        env = os.environ
    bot_name = SESSION_BOT_MAP.get(settings.get("active_session"), "permissions")
    token_key = f"TOKEN_{bot_name.upper()}"
    return env.get(token_key)

Ablauf

  1. settings.json lesen → active_session = "dev"
  2. SESSION_BOT_MAP["dev"] → "brain"
  3. f"TOKEN_{'brain'.upper()}" → "TOKEN_BRAIN"
  4. os.environ.get("TOKEN_BRAIN") → Token-String

Erweitern

Neuen Session-Typ hinzufügen — nur die Map:

SESSION_BOT_MAP = {
    # bestehend ...
    "recap": "brain",  # nur diese Zeile
}

Quiz

Frage: Welchen Token-Key sucht get_notify_token() wenn active_session = "teach"?

Antwort

TOKEN_TEACH — via Map → "teach" → .upper() → TOKEN_TEACH

Aufgabe: Füge "recap" → Brain Bot hinzu. Welche Zeile ändert sich?

Antwort
"recap": "brain",  # in SESSION_BOT_MAP

Zusammenfassung

  • SESSION_BOT_MAP: Single-Source-of-Truth für session_type → bot_name
  • get_notify_token() baut daraus TOKEN_XXX Env-Var-Key
  • Neue Session-Typen: nur Map erweitern
  • Fallback: unbekannte Sessions → Permissions Bot
