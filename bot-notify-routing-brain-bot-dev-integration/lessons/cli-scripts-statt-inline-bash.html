CLI-Scripts statt Inline-Bash für Markdown-Phasen

Ziel: Wann ein dediziertes Python-Script besser ist als Shell-Oneliner in Markdown.

Das Problem: JSON-Manipulation in Bash

Fragiler Inline-Ansatz: 
python3 -c "
import json, pathlib
p = pathlib.Path('$WORK_DIR/settings.json')
s = json.loads(p.read_text())
s['active_session'] = 'dev'
s['active_dev_slug'] = '$SLUG'
p.write_text(json.dumps(s, indent=2))
"
Probleme: Quote-Escaping komplex, kein Fehler-Handling, in 5 Dateien dupliziert, nicht testbar.

Die Lösung: set_session.py

# scripts/set_session.py
import os, sys, json
from pathlib import Path

WORK_DIR = Path(os.environ.get("WORK_DIR", str(Path(__file__).parent.parent)))
SETTINGS_PATH = WORK_DIR / "settings.json"

def main():
    s = json.loads(SETTINGS_PATH.read_text())
    if sys.argv[1] == "dev":
        if len(sys.argv) < 3:
            print("Usage: set_session.py dev <slug>", file=sys.stderr)
            sys.exit(1)
        s["active_session"] = "dev"
        s["active_dev_slug"] = sys.argv[2]
    elif sys.argv[1] == "clear":
        s["active_session"] = None
        s["active_dev_slug"] = None
    SETTINGS_PATH.write_text(json.dumps(s, indent=2))

if __name__ == "__main__":
    main()

Aufruf aus Markdown

# Anfang der Phase:
python3 "$WORK_DIR/scripts/set_session.py" dev telegram-bot-army

# Ende der Phase:
python3 "$WORK_DIR/scripts/set_session.py" clear

Wann CLI-Script statt Inline-Bash?

  • JSON lesen/schreiben → immer Script
  • An 3+ Stellen wiederholt → immer Script
  • Testbar sein muss → immer Script
  • Simples ls / git add → Inline OK

Quiz

Frage: Warum ist python3 -c "s['x'] = '$VAR'" in Shell gefährlich?

Antwort

Einfache Quotes verhindern Variable-Expansion in Bash; doppelte Quotes lösen das, brechen aber das Python-Dict-Literal. Escaping wird schnell unkontrollierbar.

Aufgabe: Wie erweitert man set_session.py um "organizer"?

Antwort
elif sys.argv[1] == "organizer":
    s["active_session"] = "organizer"

Zusammenfassung

  • JSON-Manipulation in Bash = Escaping-Komplexität + keine Tests
  • CLI-Script: testbar, lesbar, kein Duplikat in mehreren Dateien
  • Path(__file__).parent.parent als Fallback wenn WORK_DIR fehlt
  • Faustregel: wenn der Bash-Oneliner länger als 1 Zeile wird → Script
