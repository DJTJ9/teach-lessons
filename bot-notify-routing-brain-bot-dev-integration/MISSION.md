# Mission

## Why
Die Bot-Notify Routing Implementierung hat 5 Kernkonzepte sichtbar gemacht, die in zukünftigen Bot-Sessions direkt anwendbar sind — ohne nochmal debuggen zu müssen.

## Success looks like
- SESSION_BOT_MAP-Muster auf neue Bots / Sessions erweitern können
- Stille Env-Var-Fehler sofort erkennen und fixen
- try/finally State-Tracking korrekt implementieren
- CLI-Scripts für Markdown-Skill-Phasen schreiben
- Telegram Callback-Handler für neue Button-Typen hinzufügen

## Constraints
- Vorwissen: Telegram-Bot-Basics bekannt, Python bekannt
- Fokus: Implementierungsdetails + Warum-Entscheidungen, keine Grundlagen

## Out of scope
- Telegram Bot API Basics (getUpdates, sendMessage)
- Python Grundlagen
- systemd Service-Setup
