🟢 Lektion 01 — Was ist Node.js?

Ziel: Verstehen was Node.js ist, warum es existiert, und was es von anderen Lösungen unterscheidet — bevor du eine Zeile Code schreibst.

Die kurze Antwort

Node.js ist eine Laufzeitumgebung (Runtime) für JavaScript außerhalb des Browsers.

JavaScript war ursprünglich nur für Browser gedacht. Node.js (2009) hat JavaScript aus dem Browser befreit — jetzt kann JavaScript auch auf Servern, in der Kommandozeile, und überall dort laufen wo früher nur Python, Java, oder C# liefen.

Analogie: JavaScript ist wie eine Sprache (z.B. Englisch). Ein Browser ist ein Land in dem diese Sprache gesprochen wird. Node.js ist ein weiteres Land — gleiche Sprache, andere Regeln und Umgebung.

Was ist die V8 Engine?

V8 ist Googles JavaScript-Engine — der Teil von Chrome der JavaScript-Code ausführt. Node.js hat diese Engine genommen und sie aus dem Browser befreit. Das bedeutet:

  • JavaScript läuft sehr schnell (V8 kompiliert zu nativem Maschinencode)
  • Gleiche JS-Engine wie im Browser → konsistentes Verhalten
  • Node.js kann auf Windows, Mac, Linux laufen

Der Event Loop — was Node.js besonders macht

Dein Python-Bot macht etwas Interessantes: Er wartet auf Telegram-Nachrichten. Wie wartet ein Server auf Anfragen?

Traditioneller Server (Threading-Modell):
Anfrage 1 → Thread 1 → warten... → antworten
Anfrage 2 → Thread 2 → warten... → antworten
Anfrage 3 → Thread 3 → warten... → antworten
(viele Threads = hoher Speicherverbrauch)

Node.js (Event Loop Modell):
Anfrage 1 → registriere Callback → weiter
Anfrage 2 → registriere Callback → weiter
Anfrage 3 → registriere Callback → weiter
Ergebnis 1 fertig → Callback aufrufen
(ein Thread, viele gleichzeitige Anfragen)

Node.js ist non-blocking und asynchron. Statt auf eine langsame Operation zu warten (Datenbankabfrage, API-Call), registriert Node.js eine Callback-Funktion und macht andere Arbeit. Wenn die Operation fertig ist, wird der Callback aufgerufen.

Analogie: Stell dir einen Kellner vor. Blocking (Python-Standard): Der Kellner steht bei Tisch 1 und wartet bis das Essen fertig ist, bevor er zu Tisch 2 geht. Non-blocking (Node.js): Der Kellner nimmt die Bestellung auf, geht zu Tisch 2, Tisch 3 — und bringt das Essen wenn es fertig ist.

Wofür nutzt man Node.js?

  • REST APIs und Web-Server
  • Echtzeit-Apps (Chat, Notifications — ähnlich deinem Telegram-Bot)
  • CLI-Tools (Command Line Interfaces)
  • Webhook-Server (empfängt HTTP-Anfragen von externen Diensten)
  • Backend für Webanwendungen

🧠 Quiz

Frage 1: Was ist Node.js?

A) Ein JavaScript-Framework für Webseiten
❌ Falsch. Node.js ist eine Runtime — sie lässt JavaScript außerhalb des Browsers laufen.
B) Eine Laufzeitumgebung die JavaScript außerhalb des Browsers ausführt
✅ Richtig! Node.js = V8 Engine + System-APIs (Dateisystem, Netzwerk) außerhalb des Browsers.

Frage 2: Was ist der npm-Äquivalent zu Pythons pip?

A) yarn
❌ Yarn ist auch ein Package Manager, aber npm kommt mit Node.js — das ist das Standard.
B) npm
✅ Richtig! npm (Node Package Manager) ist Pythons pip für Node.js.

Frage 3: Warum ist Node.js gut für Server mit vielen gleichzeitigen Anfragen?

A) Weil es viele Threads benutzt
❌ Falsch. Node.js nutzt einen einzigen Thread mit dem Event Loop — nicht viele Threads.
B) Weil der Event Loop asynchrone Operationen non-blocking verarbeitet
✅ Richtig! Non-blocking I/O = ein Thread kann tausende Anfragen gleichzeitig handhaben.

📋 Zusammenfassung

  • Node.js = JavaScript-Runtime außerhalb des Browsers
  • Powered by V8 — Googles Chrome JavaScript-Engine
  • npm = Package Manager (wie pip für Python)
  • Event Loop = non-blocking, asynchrones I/O-Modell
  • Gut für: APIs, Webhooks, Echtzeit-Apps, CLI-Tools
  • node script.js = python script.py
