⏳ Lektion 03 — async/await & Promises

Ziel: Das wichtigste Node.js-Konzept verstehen: asynchrone Operationen mit async/await schreiben — ohne Callback-Hölle.

Warum Asynchronität?

Dein Chatbot macht viel: HTTP-Anfragen an Telegram, Notion, Claude CLI. Diese Operationen dauern Millisekunden bis Sekunden. Node.js wartet dabei nicht — es macht andere Arbeit und kommt zurück wenn die Operation fertig ist.

Evolution der Asynchronität in Node.js:
1. Callbacks (alt, unleserlich) → 2005-2015
2. Promises (.then/.catch) → 2015
3. async/await (modern, lesbar) → 2017+
Du lernst direkt async/await — das ist der Standard heute.

async/await — Vergleich mit Python

🟡 Node.js async/await

// async macht eine Funktion asynchron:
async function dateiLesen() {
    const fs = require('fs/promises');
    const inhalt = await fs.readFile('daten.txt', 'utf-8');
    return inhalt;
}

// Aufrufen:
async function main() {
    const text = await dateiLesen();
    console.log(text);
}

main();

🐍 Python async/await

import asyncio

async def datei_lesen():
    with open('daten.txt', 'r') as f:
        return f.read()

async def main():
    text = await datei_lesen()
    print(text)

asyncio.run(main())

Syntax fast identisch zu Python! Beide nutzen async und await.

Promises — was async/await darunter tut

Jede async-Funktion gibt ein Promise zurück. Ein Promise ist wie ein Versprechen: "Ich gebe dir das Ergebnis, wenn ich fertig bin."

// Promise hat drei Zustände:
// pending  → läuft noch
// resolved → erfolgreich, Ergebnis vorhanden
// rejected → Fehler aufgetreten

// async/await ist "syntactic sugar" über Promises:
const ergebnis = await machEtwas();
// ist gleichbedeutend mit:
machEtwas().then(ergebnis => { ... });

HTTP-Anfragen mit fetch (Node.js 18+)

// Telegram API anfragen (wie dein Bot das macht):
async function sendeTelegramNachricht(token, chatId, text) {
    const url = `https://api.telegram.org/bot${token}/sendMessage`;
    
    const response = await fetch(url, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ chat_id: chatId, text: text })
    });
    
    const daten = await response.json();
    return daten;
}

// Aufrufen:
await sendeTelegramNachricht(TOKEN, CHAT_ID, "Hallo!");

Fehlerbehandlung mit try/catch

async function sicherLaden(pfad) {
    try {
        const fs = require('fs/promises');
        const inhalt = await fs.readFile(pfad, 'utf-8');
        return inhalt;
    } catch (fehler) {
        console.error(`Datei nicht gefunden: ${fehler.message}`);
        return null;
    }
}

// Mehrere async-Operationen parallel:
const [telegram, notion] = await Promise.all([
    fetchTelegram(),
    fetchNotion()
]);
// Beide laufen gleichzeitig!
Promise.all(): Wenn du mehrere unabhängige async-Operationen brauchst, starte sie parallel. Das ist wie Python's asyncio.gather().

Das Callback-Muster (historisch, manchmal noch zu sehen)

// ALT — Callback-Stil (vor async/await):
const fs = require('fs');
fs.readFile('daten.txt', 'utf-8', (fehler, inhalt) => {
    if (fehler) {
        console.error(fehler);
        return;
    }
    console.log(inhalt);
});

// NEU — async/await (bevorzugt):
const { readFile } = require('fs/promises');
const inhalt = await readFile('daten.txt', 'utf-8');

🧠 Quiz

Frage 1: Was gibt eine async function immer zurück?

A) Den direkten Return-Wert
❌ Falsch. async functions wrappen den Return-Wert immer in ein Promise.
B) Ein Promise (das den Return-Wert enthält)
✅ Richtig! async function f() { return 42; } gibt ein Promise zurück das zu 42 resolved.

Frage 2: Du musst zwei unabhängige API-Calls machen. Was ist schneller?

A) await call1(); await call2(); (nacheinander)
❌ Langsamer. Call2 startet erst nach Call1. Gesamtzeit = Zeit1 + Zeit2.
B) await Promise.all([call1(), call2()]); (parallel)
✅ Richtig! Promise.all startet beide gleichzeitig. Gesamtzeit = max(Zeit1, Zeit2).

Aufgabe: Schreibe eine async Funktion die eine URL abruft und die Antwort als JSON zurückgibt. Nutze fetch und await.

Zeig mir die Lösung
async function holeJson(url) {
    const response = await fetch(url);
    return await response.json();
}

// Nutzen:
const daten = await holeJson('https://api.example.com/data');

📋 Zusammenfassung

  • async/await: gleiche Syntax wie Python asyncio — macht asynchronen Code lesbar
  • Jede async-Funktion gibt ein Promise zurück
  • await pausiert die Funktion bis das Promise resolved ist
  • Fehlerbehandlung: try/catch wie synchroner Code
  • Promise.all(): mehrere Operationen parallel starten
  • fetch(): HTTP-Anfragen in Node.js 18+ eingebaut
