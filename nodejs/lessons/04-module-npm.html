📦 Lektion 04 — Module & npm

Ziel: Eigene Module erstellen, npm nutzen, package.json verstehen — das Fundament jedes Node.js-Projekts.

Module — Vergleich mit Python

KonzeptPythonNode.js (CommonJS)Node.js (ESM, modern)
Importierenimport mathconst math = require('math')import math from 'math'
Exportierendef fn(): (direkt)module.exports = fnexport function fn()
Eigenes Modulfrom utils import fnconst { fn } = require('./utils')import { fn } from './utils.js'
Welches System nutzen? Für neue Projekte: ESM (import/export). In älterem Code und vielen Tutorials siehst du CommonJS (require). Beide funktionieren.

Eigenes Modul erstellen (CommonJS)

🟡 utils.js

// utils.js — exportiert Funktionen
function grossen(name) {
    return `Hallo, ${name}!`;
}

function addieren(a, b) {
    return a + b;
}

// Exportieren:
module.exports = { grossen, addieren };

🟡 app.js — importiert

// Importieren (mit ./ für lokale Dateien):
const { grossen, addieren } = require('./utils');

console.log(grossen("Tjark"));    // Hallo, Tjark!
console.log(addieren(3, 4));      // 7

Eingebaute Node.js Module

// Kein npm install nötig — eingebaut:
const path = require('path');          // Dateipfade
const fs = require('fs/promises');     // Dateisystem (async)
const http = require('http');          // HTTP-Server
const os = require('os');              // Betriebssystem-Info
const crypto = require('crypto');      // Kryptographie

// Beispiele:
path.join('/home', 'user', 'file.txt');    // '/home/user/file.txt'
path.extname('bild.png');                  // '.png'
os.homedir();                              // 'C:\\Users\\Tjark'

npm — Node Package Manager

🐍 Python pip

pip install requests
pip install flask

# Abhängigkeiten:
requirements.txt

# Installieren:
pip install -r requirements.txt

🟡 Node.js npm

npm install axios
npm install express

# Abhängigkeiten:
package.json

# Installieren:
npm install

package.json — das Herz des Projekts

# Neues Projekt erstellen:
mkdir mein-projekt
cd mein-projekt
npm init -y   # erstellt package.json automatisch
// package.json:
{
  "name": "mein-projekt",
  "version": "1.0.0",
  "description": "Mein erstes Node.js Projekt",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "dev": "node --watch index.js"
  },
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^3.0.0"
  }
}
# Paket installieren:
npm install express          # wird in package.json eingetragen
npm install nodemon --save-dev  # nur für Entwicklung

# Scripts ausführen:
npm start    # führt "node index.js" aus
npm run dev  # führt "node --watch index.js" aus

# Alle Abhängigkeiten installieren (nach git clone):
npm install

.gitignore für Node.js

# Immer in .gitignore aufnehmen:
node_modules/   # kann riesig werden, wird via npm install regeneriert
.env            # Secrets!

🧠 Quiz

Frage 1: Was ist das Node.js-Äquivalent zu Pythons requirements.txt?

A) modules.json
❌ Falsch. Es heißt package.json.
B) package.json
✅ Richtig! package.json definiert alle Abhängigkeiten und Metadaten.

Frage 2: Du klonst ein Node.js-Projekt. Was ist dein erster Befehl?

A) npm start
❌ Noch nicht. Erst Abhängigkeiten installieren!
B) npm install
✅ Richtig! npm install liest package.json und lädt alle Pakete in node_modules/.

Aufgabe: Erstelle utils.js mit einer Funktion gruss(name) und importiere sie in index.js.

Zeig mir die Lösung
// utils.js
module.exports = { gruss: (name) => `Hallo, ${name}!` };

// index.js
const { gruss } = require('./utils');
console.log(gruss("Tjark"));
Ausführen: node index.js

📋 Zusammenfassung

  • require('./datei') = Pythons from datei import ...
  • module.exports = {...} = exportieren aus einem Modul
  • Eingebaute Module: fs, path, http, os — kein npm install nötig
  • package.json = Pythons requirements.txt + Projektmetadaten
  • npm install = pip install -r requirements.txt
  • node_modules/ immer in .gitignore
