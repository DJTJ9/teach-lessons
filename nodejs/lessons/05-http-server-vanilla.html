🌐 Lektion 05 — HTTP-Server

Ziel: Einen einfachen HTTP-Server mit Node.js bauen der auf GET-Anfragen antwortet — ohne externes Framework.

HTTP-Basics: Request und Response

Jede Web-Kommunikation folgt demselben Muster:

Client (Browser/App)          Server (Node.js)
       ─────────────────────────────────────
       GET /hallo HTTP/1.1   →
       ←  200 OK
          "Hallo, Welt!"
HTTP MethodWofür
GETDaten abrufen
POSTDaten senden (z.B. Webhook-Empfang)
PUTDaten ersetzen
DELETEDaten löschen

Einfachster Node.js HTTP-Server

// server.js
const http = require('http');

const server = http.createServer((req, res) => {
    // req = Request (Anfrage)
    // res = Response (Antwort)
    
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Hallo, Welt!');
});

server.listen(3000, () => {
    console.log('Server läuft auf http://localhost:3000');
});

// Ausführen: node server.js
// Dann im Browser: http://localhost:3000

URL-Routing manuell

const http = require('http');

const server = http.createServer((req, res) => {
    const url = req.url;
    const methode = req.method;
    
    // Route: GET /
    if (url === '/' && methode === 'GET') {
        res.writeHead(200, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({ nachricht: 'Willkommen!' }));
        return;
    }
    
    // Route: GET /status
    if (url === '/status' && methode === 'GET') {
        res.writeHead(200, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({ status: 'ok', zeit: new Date().toISOString() }));
        return;
    }
    
    // 404 — nicht gefunden
    res.writeHead(404, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ fehler: 'Seite nicht gefunden' }));
});

server.listen(3000, () => console.log('Server läuft'));

POST-Request empfangen (Webhook-Basis)

const http = require('http');

const server = http.createServer((req, res) => {
    if (req.url === '/webhook' && req.method === 'POST') {
        
        // POST-Body einlesen (kommt in Chunks):
        let body = '';
        req.on('data', chunk => { body += chunk.toString(); });
        
        req.on('end', () => {
            try {
                const daten = JSON.parse(body);
                console.log('Empfangen:', daten);
                
                res.writeHead(200, { 'Content-Type': 'application/json' });
                res.end(JSON.stringify({ ok: true }));
            } catch (e) {
                res.writeHead(400);
                res.end(JSON.stringify({ fehler: 'Ungültiges JSON' }));
            }
        });
        return;
    }
    
    res.writeHead(404);
    res.end('Not found');
});

server.listen(3000);
Beziehung zum Chatbot: Dein Python-Bot nutzt Telegram-Polling (Bot fragt Telegram ab). Webhooks drehen das um: Telegram schickt Nachrichten direkt an deinen Server via POST. Das ist das Muster das wir im Praxisprojekt bauen.

Warum vanilla HTTP für Lehre?

Das eingebaute http-Modul zeigt dir was wirklich passiert. Frameworks wie Express.js bauen darauf auf und verstecken diese Details. Wer versteht was darunter liegt, versteht auch Express.

🧠 Quiz

Frage 1: Was passiert wenn kein Status Code mit res.writeHead() gesetzt wird?

A) Node.js wirft einen Fehler
❌ Falsch. Node.js sendet dann automatisch 200 OK — aber explizit ist besser.
B) Node.js sendet automatisch 200 OK
✅ Richtig! Aber setze immer explizit einen Status Code — es macht den Code lesbarer.

Frage 2: Warum kommt der POST-Body in "chunks"?

A) Weil JSON immer aufgeteilt wird
❌ Falsch. Chunks haben nichts mit JSON zu tun. HTTP ist ein Stream-Protokoll.
B) HTTP ist ein Stream-Protokoll — Daten kommen stückweise und werden gesammelt
✅ Richtig! req.on('data') sammelt Stücke, req.on('end') signalisiert dass alle da sind.

Aufgabe: Erweitere den Server um eine Route GET /hallo/:name die "Hallo, [name]!" zurückgibt. (Tipp: req.url parsen)

Zeig mir die Lösung
if (req.url.startsWith('/hallo/') && req.method === 'GET') {
    const name = req.url.split('/')[2];
    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ nachricht: `Hallo, ${name}!` }));
    return;
}

📋 Zusammenfassung

  • http.createServer(): Server erstellen der auf Anfragen reagiert
  • req.url und req.method: Route und HTTP-Methode prüfen
  • res.writeHead(statusCode, headers): Antwort-Header setzen
  • res.end(body): Antwort-Body senden und Verbindung schließen
  • POST-Body: via req.on('data') als Chunks sammeln, dann parsen
  • JSON.stringify() / JSON.parse(): zwischen Objekt und JSON-String
