Variablen: let, const, var

🟡 JavaScript (Node.js)

// const — unveränderlich (wie Python-Konvention UPPERCASE)
const name = "Tjark";
const maxGesundheit = 100;

// let — veränderlich (wie Python-Variable)
let punkte = 0;
punkte = 50;  // OK

// var — alt, nicht mehr benutzen!
// Benutze immer const oder let

🐍 Python

# Python hat kein const/let
# Konvention: UPPERCASE für Konstanten
NAME = "Tjark"
MAX_GESUNDHEIT = 100

# Normale Variable:
punkte = 0
punkte = 50
Regel: Nutze const wenn sich der Wert nicht ändert. let wenn er sich ändern wird. Niemals var in modernem JavaScript.

Funktionen

🟡 JavaScript

// Function Declaration:
function addieren(a, b) {
    return a + b;
}

// Function Expression:
const multiplizieren = function(a, b) {
    return a * b;
};

// Arrow Function (modern, kompakt):
const subtrahieren = (a, b) => a - b;
const verdoppeln = x => x * 2;  // ein Param, kein ()

// Aufrufen:
console.log(addieren(3, 4));      // 7
console.log(verdoppeln(5));       // 10

🐍 Python

def addieren(a, b):
    return a + b

# Lambda (wie Arrow Function):
subtrahieren = lambda a, b: a - b
verdoppeln = lambda x: x * 2

# Aufrufen:
print(addieren(3, 4))      # 7
print(verdoppeln(5))       # 10

Objects (= Python Dictionaries)

// Object — wie Python dict, aber mit Punkt-Zugriff:
const person = {
    name: "Tjark",
    alter: 25,
    aktiv: true
};

// Zugriff:
console.log(person.name);        // "Tjark"  — Punkt-Notation
console.log(person["alter"]);    // 25       — Bracket-Notation (wie Python)

// Hinzufügen/Ändern:
person.email = "t@example.com";  // neu
person.alter = 26;               // ändern

// Zerstören (Destructuring):
const { name, alter } = person;  // name = "Tjark", alter = 26

Arrays (= Python Listen)

const zahlen = [1, 2, 3, 4, 5];

// Zugriff:
console.log(zahlen[0]);          // 1

// Methoden:
zahlen.push(6);                  // anhängen (wie append)
zahlen.pop();                    // letztes entfernen
const laenge = zahlen.length;    // 5 (kein .length() — kein !)

// Iteration (wie for x in liste):
for (const z of zahlen) {
    console.log(z);
}

// Array-Methoden (sehr mächtig):
const verdoppelt = zahlen.map(x => x * 2);   // [2, 4, 6, 8, 10]
const gerade = zahlen.filter(x => x % 2 === 0);  // [2, 4]

Template Literals (= Python f-Strings)

const name = "Tjark";
const punkte = 95;

// JavaScript — Template Literal mit Backtick:
const nachricht = `Hallo ${name}, du hast ${punkte} Punkte!`;

// Python — f-String:
// nachricht = f"Hallo {name}, du hast {punkte} Punkte!"

Kontrollfluss

// if/else — fast wie Python (aber mit Klammern und Geschweifte Klammern):
if (punkte >= 90) {
    console.log("Sehr gut");
} else if (punkte >= 70) {
    console.log("Gut");
} else {
    console.log("Weiter üben");
}

// === statt == für Vergleiche!
console.log(1 === "1");   // false (Typ + Wert)
console.log(1 == "1");    // true  (nur Wert — NICHT verwenden!)
Immer === verwenden! In JavaScript macht == automatische Typkonvertierung und führt zu überraschenden Ergebnissen. === prüft Typ und Wert — das ist was du willst.

🧠 Quiz

Frage 1: Was ist der Unterschied zwischen const und let?

A) const ist schneller als let
❌ Falsch. const ist unveränderlich (kein Reassign möglich), let ist veränderlich.
B) const kann nicht neu zugewiesen werden, let schon
✅ Richtig! const x = 5; x = 6; → TypeError. let y = 5; y = 6; → OK.

Frage 2: Was gibt 1 == "1" in JavaScript zurück?

A) false
❌ Falsch. == macht Typkonvertierung: "1" wird zu 1, dann 1 == 1 → true. Nutze ===!
B) true (daher immer === benutzen!)
✅ Richtig! Genau deshalb ist === sicherer.

Aufgabe: Erstelle eine Datei hallo.js die eine Arrow Function definiert die einen Namen nimmt und "Hallo, [Name]!" ausgibt.

Zeig mir die Lösung
const begruessung = name => console.log(`Hallo, ${name}!`);
begruessung("Tjark");  // Hallo, Tjark!

📋 Zusammenfassung

  • const für unveränderliche Werte, let für veränderliche — nie var
  • Arrow Functions: (a, b) => a + b = Pythons lambda a, b: a + b
  • Objects = Python Dicts mit Punkt-Zugriff
  • Arrays = Python Listen, .length statt len()
  • Template Literals: `${variable}` = Python f"{variable}"
  • Immer === statt == für Vergleiche
  • console.log() = Python print()
