☕ Lektion 01 — Java vs. C#: Überblick & Setup

Ziel: Die Gemeinsamkeiten und Unterschiede von Java und C# verstehen, JDK + Maven installieren, und Hello World kompilieren.

Java und C# — mehr Geschwister als Fremde

Java kam 1995 (Sun Microsystems). C# kam 2002 (Microsoft) — und war stark von Java inspiriert. Beide sind:

  • Objektorientiert, statisch typisiert
  • Kompiliert zu Bytecode (Java → JVM, C# → .NET CLR)
  • Garbage Collected — kein manuelles Speichermanagement
  • Mit ähnlicher Syntax: Klassen, Interfaces, Vererbung, Generics

Die wichtigsten Unterschiede im Überblick

FeatureJavaC#
HerstellerOracle (ehem. Sun)Microsoft

Setup — JDK und Maven installieren

# JDK 21 (LTS) installieren:
# Windows: https://adoptium.net/ → JDK 21 → .msi herunterladen

# Prüfen:
java -version
javac -version

# Maven installieren:
# https://maven.apache.org/download.cgi → Binary zip → entpacken → PATH setzen

# Prüfen:
mvn -version

Erstes Maven-Projekt

# Projekt erstellen:
mvn archetype:generate -DgroupId=com.beispiel -DartifactId=hallo-java \
  -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false

cd hallo-java

Hello World — Java vs. C#

☕ Java

// src/main/java/com/beispiel/App.java
package com.beispiel;

public class App {
    public static void main(String[] args) {
        System.out.println("Hallo, Java!");
    }
}

💜 C#

// Program.cs
using System;

class Program {
    static void Main(string[] args) {
        Console.WriteLine("Hallo, C#!");
    }
}

Was du sofort erkennst

Kompilieren und Ausführen

# Mit Maven:
mvn compile
mvn exec:java -Dexec.mainClass="com.beispiel.App"

# Ausgabe:
# Hallo, Java!
Dateiname = Klassenname: In Java muss die Klasse App in der Datei App.java liegen. Das ist eine strikte Regel, nicht nur Konvention.

🧠 Quiz

Frage 1: Wie gibt man in Java Text auf der Konsole aus?

A) Console.WriteLine("Text")
❌ Das ist C#. In Java: System.out.println("Text")
B) System.out.println("Text")
✅ Richtig! System.out ist der Standard-Ausgabestream in Java.

Frage 2: Was ist in Java anders als in C# bezüglich Dateinamen?

A) In Java muss der Dateiname dem public class Namen entsprechen
✅ Richtig! public class PlayerHealth muss in PlayerHealth.java sein.
B) In Java ist der Dateiname beliebig
❌ Falsch. Java erzwingt dass der Dateiname = public class Name.

📋 Zusammenfassung

  • Java und C# sind Geschwister — ähnliche Syntax, ähnliche Konzepte
  • Hauptunterschiede: Properties-Syntax, Checked Exceptions, Primärplattform
  • System.out.println() = Console.WriteLine()
  • Dateiname muss Klassenname entsprechen (Java-Regel)
  • Maven: Build-Tool für Java (mvn compile, mvn exec:java)
  • package in Java entspricht namespace in C#
