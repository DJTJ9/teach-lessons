☕ Lektion 02 — Syntax-Vergleich: Typen, Variablen, Kontrollfluss

Ziel: Java-Syntax sicher lesen und schreiben — mit direktem Vergleich zu C#.

Primitive Datentypen

JavaC#Beschreibung
intint32-Bit Integer
longlong64-Bit Integer
doubledouble64-Bit Float
booleanbooltrue/false
charchareinzelnes Zeichen
StringstringText (Referenztyp!)
Unterschied: In Java ist String großgeschrieben (es ist eine Klasse, kein primitiver Typ). In C# ist string ein Alias für System.String, wird aber kleingeschrieben.

Variablen deklarieren

☕ Java

int alter = 25;
double preis = 9.99;
boolean aktiv = true;
String name = "Tjark";

// var (seit Java 10):
var anzahl = 42;  // Typ wird inferiert

💜 C#

int alter = 25;
double preis = 9.99;
bool aktiv = true;
string name = "Tjark";

// var:
var anzahl = 42;  // gleich wie Java

Kontrollfluss — if/else

☕ Java

int punkte = 75;

if (punkte >= 90) {
    System.out.println("Sehr gut");
} else if (punkte >= 70) {
    System.out.println("Gut");
} else {
    System.out.println("Verbesserungswürdig");
}

💜 C#

int punkte = 75;

if (punkte >= 90) {
    Console.WriteLine("Sehr gut");
} else if (punkte >= 70) {
    Console.WriteLine("Gut");
} else {
    Console.WriteLine("Verbesserungswürdig");
}

if/else-Syntax ist fast identisch — nur die Ausgabe-Methode unterscheidet sich.

Schleifen

☕ Java — for

for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

// Foreach (enhanced for):
String[] namen = {"Anna", "Ben"};
for (String n : namen) {
    System.out.println(n);
}

// while:
int x = 0;
while (x < 3) {
    x++;
}

💜 C# — for

for (int i = 0; i < 5; i++) {
    Console.WriteLine(i);
}

// Foreach:
string[] namen = {"Anna", "Ben"};
foreach (string n in namen) {
    Console.WriteLine(n);
}

// while:
int x = 0;
while (x < 3) {
    x++;
}
Unterschied Foreach: Java nutzt for (Type var : collection) mit Doppelpunkt. C# nutzt foreach (type var in collection) mit in. Ansonsten identisch.

String-Operationen

☕ Java

String s = "Hallo";
int len = s.length();        // length() mit ()
String upper = s.toUpperCase();
boolean hat = s.contains("all");
String[] teile = s.split("l");

// String-Vergleich: .equals() !
String a = "test";
if (a.equals("test")) { ... }  // NICHT ==

💜 C#

string s = "Hallo";
int len = s.Length;          // Length ohne ()
string upper = s.ToUpper();
bool hat = s.Contains("all");
string[] teile = s.Split('l');

// String-Vergleich: == funktioniert
string a = "test";
if (a == "test") { ... }     // OK in C#
Wichtiger Unterschied! In Java muss man Strings mit .equals() vergleichen, NICHT mit ==. Mit == vergleicht man Referenzen (Speicheradressen), nicht Inhalte. Das ist ein häufiger Bug für C#-Umsteiger!

String Interpolation

☕ Java — String.format()

String name = "Tjark";
int alter = 25;
String msg = String.format(
    "Hallo %s, du bist %d Jahre alt.", 
    name, alter);

// Seit Java 15: Text Blocks
String json = """
    {"name": "Tjark"}
    """;

💜 C# — $"..."

string name = "Tjark";
int alter = 25;
string msg = 
    $"Hallo {name}, du bist {alter} Jahre alt.";

// Multi-line strings:
string json = @"{""name"": ""Tjark""}";

🧠 Quiz

Frage 1: Was gibt folgender Java-Code aus?

String a = new String("test");
String b = new String("test");
System.out.println(a == b);
A) true
❌ Falsch! == vergleicht Referenzen. Beide sind verschiedene Objekte im Speicher.
B) false
✅ Richtig! Verschiedene Objekte → verschiedene Referenzen → false. Nutze a.equals(b) für Inhaltsvergleich.

Frage 2: Wie schreibt man foreach in Java?

A) foreach (String s in liste)
❌ Das ist C#. In Java: for (String s : liste)
B) for (String s : liste)
✅ Richtig! Doppelpunkt statt in.

Aufgabe: Schreibe Java-Code der prüft ob zwei Strings gleich sind und das Ergebnis ausgibt.

Zeig mir die Lösung
String s1 = "hallo";
String s2 = "hallo";
if (s1.equals(s2)) {
    System.out.println("Gleich!");
}

📋 Zusammenfassung

  • boolean (Java) statt bool (C#), String groß statt string
  • if/else, for, while: fast identisch
  • Foreach: Java for (Type x : list) statt C# foreach (type x in list)
  • Strings immer mit .equals() vergleichen, nicht ==
  • s.length() (Java, mit Klammern) statt s.Length (C#, ohne)
  • String.format() statt $"..." für String-Interpolation
