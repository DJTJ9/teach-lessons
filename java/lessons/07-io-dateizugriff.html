Konsoleneingabe — Scanner

☕ Java — Scanner

import java.util.Scanner;

Scanner scanner = new Scanner(System.in);

System.out.print("Dein Name: ");
String name = scanner.nextLine();

System.out.print("Dein Alter: ");
int alter = scanner.nextInt();

System.out.println("Hallo " + name + "!");
scanner.close();

💜 C#

Console.Write("Dein Name: ");
string name = Console.ReadLine();

Console.Write("Dein Alter: ");
int alter = int.Parse(Console.ReadLine());

Console.WriteLine($"Hallo {name}!");

Datei schreiben

☕ Java — Files API (modern)

import java.nio.file.Files;
import java.nio.file.Path;
import java.io.IOException;

// Ganzen String in Datei schreiben:
try {
    Files.writeString(Path.of("aufgaben.txt"), 
        "Task 1: Einkaufen\nTask 2: Sport");
} catch (IOException e) {
    System.out.println("Fehler: " + e.getMessage());
}

// Mehrere Zeilen:
List<String> zeilen = List.of("Zeile 1", "Zeile 2");
Files.write(Path.of("liste.txt"), zeilen);

💜 C#

using System.IO;

// Ganzen String schreiben:
File.WriteAllText("aufgaben.txt", 
    "Task 1: Einkaufen\nTask 2: Sport");

// Mehrere Zeilen:
string[] zeilen = {"Zeile 1", "Zeile 2"};
File.WriteAllLines("liste.txt", zeilen);

Datei lesen

☕ Java

// Ganzen Inhalt als String:
String inhalt = Files.readString(Path.of("aufgaben.txt"));
System.out.println(inhalt);

// Alle Zeilen als Liste:
List<String> zeilen = Files.readAllLines(Path.of("liste.txt"));
for (String zeile : zeilen) {
    System.out.println(zeile);
}

// Datei existiert?
boolean existiert = Files.exists(Path.of("aufgaben.txt"));

💜 C#

// Ganzen Inhalt:
string inhalt = File.ReadAllText("aufgaben.txt");
Console.WriteLine(inhalt);

// Alle Zeilen:
string[] zeilen = File.ReadAllLines("liste.txt");
foreach (string zeile in zeilen) {
    Console.WriteLine(zeile);
}

// Datei existiert?
bool existiert = File.Exists("aufgaben.txt");

Methodenvergleich

Java (Files API)C# (File API)

Files.writeString(path, text)File.WriteAllText(path, text)
Files.readAllLines(path)File.ReadAllLines(path)

JSON lesen/schreiben (mit Gson)

// pom.xml Abhängigkeit hinzufügen:
// <dependency>
//   <groupId>com.google.code.gson</groupId>
//   <artifactId>gson</artifactId>
//   <version>2.10.1</version>
// </dependency>

import com.google.gson.Gson;
import java.util.HashMap;

Gson gson = new Gson();

// Objekt → JSON-String:
HashMap<String, Object> daten = new HashMap<>();
daten.put("name", "Tjark");
daten.put("punkte", 95);
String json = gson.toJson(daten);  // {"name":"Tjark","punkte":95}
Files.writeString(Path.of("daten.json"), json);

// JSON-String → Objekt:
String gelesenerJson = Files.readString(Path.of("daten.json"));
HashMap datenZurueck = gson.fromJson(gelesenerJson, HashMap.class);

🧠 Quiz

Frage 1: Wie liest man Konsoleneingabe in Java?

A) Console.ReadLine()
❌ Das ist C#. Java nutzt new Scanner(System.in) und dann scanner.nextLine().
B) new Scanner(System.in).nextLine()
✅ Richtig! Scanner ist das Standard-Werkzeug für Konsoleneingabe.

Frage 2: Welche IOException-Behandlung ist für Files.readString() nötig?

A) Keine — IOException ist unchecked
❌ Falsch. IOException ist eine Checked Exception — try/catch oder throws nötig.
B) try/catch IOException oder throws IOException im Methodenkopf
✅ Richtig! Checked Exception — Compiler erzwingt die Behandlung.

📋 Zusammenfassung

  • Scanner(System.in) für Konsoleneingabe (statt Console.ReadLine())
  • Files.readString(Path.of(...)) für Dateizugriff (modernes API)
  • IOException ist Checked — immer try/catch oder throws nötig
  • Methodennamen ähnlich zu C# File-API, aber lowercase
  • Gson-Bibliothek für JSON (wie Newtonsoft.Json in C#)
