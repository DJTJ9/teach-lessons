☕ Lektion 03 — OOP: Klassen & Konstruktoren

Ziel: Klassen in Java erstellen, Felder und Methoden definieren, Konstruktoren schreiben — im Vergleich zu C#.

Klassen: fast identisch zu C#

☕ Java

public class Person {
    // Felder (private per Konvention)
    private String name;
    private int alter;

    // Konstruktor
    public Person(String name, int alter) {
        this.name = name;
        this.alter = alter;
    }

    // Getter-Methode
    public String getName() {
        return name;
    }

    // Setter-Methode
    public void setName(String name) {
        this.name = name;
    }

    // Methode
    public String vorstellen() {
        return "Ich bin " + name + ", " + alter;
    }
}

💜 C#

public class Person {
    // Felder
    private string name;
    private int alter;

    // Konstruktor
    public Person(string name, int alter) {
        this.name = name;
        this.alter = alter;
    }

    // Properties (statt Getter/Setter)
    public string Name {
        get => name;
        set => name = value;
    }

    // Methode
    public string Vorstellen() {
        return $"Ich bin {name}, {alter}";
    }
}

Der größte Unterschied: Properties vs. Getter/Setter

In C# hast du Properties mit get/set. Java kennt das nicht — stattdessen schreibt man explizite Getter- und Setter-Methoden:

C# PropertyJava Äquivalent
person.Nameperson.getName()
person.Name = "Anna"person.setName("Anna")
// Java — Nutzung der Klasse:
Person p = new Person("Tjark", 25);
System.out.println(p.getName());   // "Tjark"
p.setName("Anna");
System.out.println(p.vorstellen()); // "Ich bin Anna, 25"

this-Keyword

Gleich wie in C#: this verweist auf das aktuelle Objekt. Typisch im Konstruktor wenn Parameter gleich heißen wie Felder:

public Person(String name, int alter) {
    this.name = name;  // this.name = Feld, name = Parameter
    this.alter = alter;
}

Mehrere Konstruktoren (Overloading)

public class Person {
    private String name;
    private int alter;

    public Person(String name, int alter) {
        this.name = name;
        this.alter = alter;
    }

    // Zweiter Konstruktor — nur Name, Alter default 0
    public Person(String name) {
        this(name, 0);  // ruft den anderen Konstruktor auf
    }
}
Java-Konvention: Klassennamen = PascalCase (PlayerHealth), Methoden + Variablen = camelCase (getName, maxHealth). Konstanten = UPPER_SNAKE_CASE (MAX_LEVEL). Genau wie C#.

Vollständiges Beispiel — nutzen

public class App {
    public static void main(String[] args) {
        Person p1 = new Person("Tjark", 25);
        Person p2 = new Person("Anna");   // alter = 0

        System.out.println(p1.vorstellen());
        System.out.println(p2.getName());
        
        p2.setName("Lena");
        System.out.println(p2.getName());  // "Lena"
    }
}

🧠 Quiz

Frage 1: Wie liest man in Java das Feld name einer Person-Instanz?

A) person.Name
❌ Das ist C# Property-Syntax. In Java: person.getName()
B) person.getName()
✅ Richtig! Java nutzt explizite Getter-Methoden, kein Property-Syntax.

Frage 2: Was bedeutet kein Zugriffsmodifier (package-private) in Java?

A) Gleich wie private
❌ Falsch. Package-private ist weiter als private — sichtbar für alle Klassen im selben Package.
B) Sichtbar für alle Klassen im selben Package
✅ Richtig! In C# entspricht das ungefähr internal.

Aufgabe: Schreibe eine Java-Klasse Produkt mit Feldern name (String) und preis (double), Konstruktor, Getter und einer Methode istGuenstig() die true zurückgibt wenn preis unter 10 liegt.

Zeig mir die Lösung
public class Produkt {
    private String name;
    private double preis;

    public Produkt(String name, double preis) {
        this.name = name;
        this.preis = preis;
    }

    public String getName() { return name; }
    public double getPreis() { return preis; }

    public boolean istGuenstig() {
        return preis < 10.0;
    }
}

📋 Zusammenfassung

  • Klassen-Syntax: fast identisch zu C# — public class, Felder, Konstruktor, Methoden
  • Java hat keine Properties — stattdessen explizite getX()/setX()-Methoden
  • this: verweist auf aktuelle Instanz, gleich wie C#
  • Konstruktor-Overloading: this(args) ruft anderen Konstruktor auf
  • Zugriffsmodifier: public/private/protected gleich wie C#, dazu package-private (kein Modifier)
