☕ Lektion 04 — Vererbung & Interfaces

Ziel: extends und implements in Java verstehen und anwenden — im direkten Vergleich zur C#-Syntax.

Vererbung — extends vs. :

☕ Java — extends

public class Tier {
    protected String name;
    
    public Tier(String name) {
        this.name = name;
    }
    
    public void sprechen() {
        System.out.println("...");
    }
}

public class Hund extends Tier {
    public Hund(String name) {
        super(name);  // Basisklassen-Konstruktor
    }
    
    @Override
    public void sprechen() {
        System.out.println(name + " sagt: Wuff!");
    }
}

💜 C# — :

public class Tier {
    protected string name;
    
    public Tier(string name) {
        this.name = name;
    }
    
    public virtual void Sprechen() {
        Console.WriteLine("...");
    }
}

public class Hund : Tier {
    public Hund(string name) : base(name) { }
    
    public override void Sprechen() {
        Console.WriteLine($"{name} sagt: Wuff!");
    }
}

Wichtiger Unterschied: In Java sind alle Methoden standardmäßig überschreibbar (außer final). In C# muss eine Methode explizit virtual sein damit man sie überschreiben kann.

Interfaces — implements vs. :

☕ Java — implements

public interface Fahrbar {
    void fahren();
    int getGeschwindigkeit();
    
    // Default-Methoden (seit Java 8):
    default void hupen() {
        System.out.println("Beep!");
    }
}

public class Auto implements Fahrbar {
    @Override
    public void fahren() {
        System.out.println("Auto fährt");
    }
    
    @Override
    public int getGeschwindigkeit() {
        return 120;
    }
}

💜 C# — :

public interface IFahrbar {
    void Fahren();
    int GetGeschwindigkeit();
    
    // Default interface methods (C# 8+):
    void Hupen() {
        Console.WriteLine("Beep!");
    }
}

public class Auto : IFahrbar {
    public void Fahren() {
        Console.WriteLine("Auto fährt");
    }
    
    public int GetGeschwindigkeit() {
        return 120;
    }
}

Mehrere Interfaces implementieren

// Java — mehrere Interfaces, nur eine Basisklasse:
public class Amphibienfahrzeug extends Fahrzeug implements Fahrbar, Schwimmbar {
    // muss alle Methoden beider Interfaces implementieren
}
Konvention: C# benennt Interfaces mit "I" Prefix (IFahrbar). Java macht das nicht — Interface heißt einfach Fahrbar. Das ist nur Konvention, kein Zwang.

Abstrakte Klassen

☕ Java

public abstract class Form {
    // Abstrakte Methode — MUSS überschrieben werden
    public abstract double flaeche();
    
    // Konkrete Methode — kann benutzt werden
    public void drucken() {
        System.out.println("Fläche: " + flaeche());
    }
}

public class Kreis extends Form {
    private double radius;
    
    public Kreis(double radius) {
        this.radius = radius;
    }
    
    @Override
    public double flaeche() {
        return Math.PI * radius * radius;
    }
}

💜 C#

public abstract class Form {
    public abstract double Flaeche();
    
    public void Drucken() {
        Console.WriteLine($"Fläche: {Flaeche()}");
    }
}

public class Kreis : Form {
    private double radius;
    
    public Kreis(double radius) {
        this.radius = radius;
    }
    
    public override double Flaeche() {
        return Math.PI * radius * radius;
    }
}

🧠 Quiz

Frage 1: Wie implementiert man ein Interface in Java?

A) class Auto : Fahrbar
❌ Das ist C#. Java: class Auto implements Fahrbar
B) class Auto implements Fahrbar
✅ Richtig! implements für Interfaces, extends für Klassen.

Frage 2: Kann man in Java mehrere Klassen erben (multiple inheritance)?

A) Ja, mit extends Klasse1, Klasse2
❌ Falsch. Java erlaubt nur Single Inheritance bei Klassen. Mehrere Interfaces sind aber möglich.
B) Nein — nur eine Basisklasse, aber mehrere Interfaces
✅ Richtig! Gleich wie C#.

Aufgabe: Schreibe eine abstrakte Klasse Charakter mit Methode angreifen() und eine Unterklasse Krieger die diese implementiert.

Zeig mir die Lösung
public abstract class Charakter {
    public abstract void angreifen();
}

public class Krieger extends Charakter {
    @Override
    public void angreifen() {
        System.out.println("Krieger schlägt mit Schwert!");
    }
}

📋 Zusammenfassung

  • Vererbung: Java extends statt C# :
  • Interface: Java implements statt C# :
  • Basisklasse aufrufen: Java super() statt C# base()
  • Methode überschreiben: Java @Override statt C# override
  • In Java: alle Methoden überschreibbar außer final; in C# braucht man virtual
  • Java-Interfaces haben kein I-Prefix (C# Konvention)
