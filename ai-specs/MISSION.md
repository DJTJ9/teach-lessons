# Mission

## Why
Eigenständig professionelle Specs (requirement, design, task) für eigene Projekte schreiben — damit Projekte wie der Telegram-Bot und die Dart-App sauber dokumentiert, planbar und durch KI-Assistenten umsetzbar sind.

## Success looks like
- Kann für ein neues Feature selbstständig eine Requirement Spec schreiben
- Kann daraus eine Design Spec ableiten
- Kann atomare Task Specs aus einer Design Spec generieren
- Hat mindestens ein echtes Projekt (Telegram-Bot oder Dart-App) vollständig durchgespeced

## Constraints
- Vorkenntnisse: versteht Zweck von Specs, kennt grobes Aussehen — noch nie selbst geschrieben
- Fokus: requirement specs, design specs, task specs im AI-Engineering-Kontext
- Kontext: Claude Code, agentic workflows, eigene Projekte
- Abschluss: wenn erstes echtes Projekt eigenständig durchgespeced ist

## Out of scope
- Formelle Methoden (UML-Diagramme, ISO-Standards)
- Enterprise-Prozesse (SAFe, TOGAF, ITIL)
- Non-AI-Engineering-Kontext
