Android SDK + JDK 17

JDK 17 und Android SDK für headless Android-Builds auf dem Server.

Was ist JDK 17?

JDK = Java Development Kit. Gradle (Android Build-Tool) benötigt JDK als Basis. Version 17 ist der aktuelle LTS für Android-Entwicklung.

Was ist das Android SDK?

  • platform-tools: adb zum Gerät verbinden und debuggen
  • platforms;android-35: Android 15 API gegen die du entwickelst
  • build-tools;35.0.0: Compiler, APK-Signer, d8

Schritt 1: JDK 17

apt install -y openjdk-17-jdk

Prüfen: java -version sollte openjdk version "17.x" ausgeben.

Schritt 2: cmdline-tools herunterladen

mkdir -p /opt/android-sdk/cmdline-tools
wget "https://dl.google.com/android/repository/commandlinetools-linux-11076708_latest.zip" -O /tmp/cmdline-tools.zip

Schritt 3: Entpacken + umbenennen

unzip /tmp/cmdline-tools.zip -d /opt/android-sdk/cmdline-tools
mv /opt/android-sdk/cmdline-tools/cmdline-tools /opt/android-sdk/cmdline-tools/latest

sdkmanager erwartet Pfad cmdline-tools/latest/bin/ — ohne Umbenennung nicht gefunden.

Schritt 4: Umgebungsvariablen

echo 'export ANDROID_HOME=/opt/android-sdk' >> ~/.bashrc
echo 'export PATH=$PATH:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools' >> ~/.bashrc
source ~/.bashrc

Schritt 5: Lizenzen + Pakete

yes | sdkmanager --licenses
sdkmanager "platform-tools" "platforms;android-35" "build-tools;35.0.0"

Prüfen

adb --version && sdkmanager --list_installed

Quiz

Warum Ordner zu "latest" umbenennen?

Google nennt ihn immer latest
sdkmanager erwartet Pfad cmdline-tools/latest/bin
Kürzerer Ordnername

Zusammenfassung

  • JDK 17: Basis für Gradle und Android-Builds
  • SDK: platform-tools, platforms;android-35, build-tools;35.0.0
  • Ordner zu latest umbenennen: Anforderung von sdkmanager
  • ANDROID_HOME + PATH: Tools von überall aufrufbar
